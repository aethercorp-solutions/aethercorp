Turning math and data into pixels on your screen

Computers draw graphics by turning math and data into pixels on your screen. Here’s a clear, step-by-step explanation from the lowest level to what you see.

1. The Screen Is a Grid of Pixels

A display is a matrix of tiny dots called pixels.

  • Each pixel has a position: (x, y)
  • Each pixel has a color: (R, G, B)

    Example: (255, 0, 0) = red

📺 A 1920×1080 screen = 2,073,600 pixels

2. Graphics Are Represented as Data

Computers don’t “see” images — they store numbers.

Two main graphic types:

A. Raster Graphics (Pixels)

  • Photos, screenshots, textures
  • Stored as pixel color values
  • Example:

Pixel (10, 5) → RGB(0, 255, 0)

B. Vector Graphics (Math Shapes)

  • Lines, circles, polygons
  • Defined by equations
  • Example:

Circle(center=(100,100), radius=50)

Vectors must be converted into pixels to display.

3. CPU vs GPU

🧠 CPU (Central Processing Unit)

  • Handles logic, instructions
  • Tells what to draw

🚀 GPU (Graphics Processing Unit)

  • Specialized for millions of parallel calculations
  • Tells how to draw pixels fast

Modern graphics rely heavily on the GPU.

This is the most important part.

Step 1: Application Sends Data

Your app/game sends:

  • Vertices (points in 3D space)
  • Colors
  • Textures
  • Shaders (small programs)

Example (triangle):


(0,0,0)
(1,0,0)
(0,1,0)

Step 2: Vertex Processing

The GPU:

  • Transforms coordinates
  • Converts 3D → 2D (projection)
  • Applies camera and perspective

🧮 Math involved:

  • Matrix multiplication
  • Linear algebra

Step 3: Rasterization

This step converts shapes into pixels.

  • A triangle becomes thousands of pixels
  • GPU decides which pixels are inside the shape

This is where vector → raster happens.

Step 4: Fragment (Pixel) Shading

For each pixel:

  • Calculate color
  • Apply lighting
  • Apply texture
  • Apply shadows

Example:


FinalColor = TextureColor × LightIntensity

Step 5: Framebuffer Output

  • Final pixel colors are written to memory
  • The screen refreshes (usually 60–120 times per second)

🎞️ Each full image = one frame

5. 2D Graphics (UI, Apps, Web)

Even 2D graphics use the GPU:

  • Rectangles → two triangles
  • Text → textured quads
  • Icons → vector paths → pixels

Example:


Button → rectangle → triangles → pixels

Frameworks:

  • iOS: Core Graphics, Metal
  • Android: Canvas, Vulkan
  • Web: Canvas, WebGL
  • React Native: Skia, Fabric

6. 3D Graphics (Games, AR, VR)

Extra steps:

  • 3D models (meshes)
  • Lighting (Phong, PBR)
  • Depth buffer (Z-buffer)
  • Shaders (GLSL, HLSL, Metal)

Depth example:


Pixel closer to camera wins

7. Double Buffering (No Flicker)

Computers draw using two buffers:

  1. Back buffer → drawing happens here
  2. Front buffer → currently displayed

Then they swap instantly.

This prevents flickering.

8. Why GPUs Are So Fast

GPUs:

  • Have thousands of cores
  • Do the same math on many pixels at once
  • Perfect for graphics and AI

CPU: few powerful cores

GPU: many simple cores

9. Simple Mental Model

Think of it like this:

  1. Math defines shapes
  2. GPU breaks shapes into pixels
  3. Each pixel gets a color
  4. Pixels appear on screen
