React.Component — Detailed Explanation

React.Component is the base class for creating class-based components in React. Before React Hooks (introduced in 2019), this was the primary way to create stateful components.

When you create a class component, you extend React.Component:


import React from "react";

class MyComponent extends React.Component {
  render() {
    return <h1>Hello World</h1>
  }
}

1️⃣ What is React.Component?

React.Component is a JavaScript class that provides:

  • Component lifecycle methods
  • State management (this.state)
  • Props handling (this.props)
  • Rendering via render()
  • Built-in update mechanisms (setState())

2️⃣ Core Concepts

✅ A) render() Method (Required)

Every class component must implement render().


render() {
    return <div>Hello</div>
}
  • Returns JSX
  • Must return a single root element
  • Must be pure (no side effects)

✅ B) Props

Props are inputs passed from parent components.


class Welcome extends React.Component {
    render() {
        return <h1>Hello {this.props.name}</h1>
    }
}

Usage:


<Welcome name="John" />

Props are:

  • Read-only
  • Immutable inside the component

✅ C) State

State stores internal data that can change over time.


class Counter extends React.Component {
    constructor(props) {
        super(props);
        this.state = { count: 0 };
    }
    
    render() {
        return <h1>{this.state.count}</h1>
    }
}

3️⃣ setState()

You must use setState() to update state.


this.setState({ count: 1 });

Why?

  • It triggers re-render
  • It updates efficiently
  • It batches updates

Functional setState (Recommended)


this.setState((prevState) => ({
    count: prevState.count + 1
}));

Use this when new state depends on old state.

4️⃣ Lifecycle Methods

Lifecycle methods allow you to run code at different stages.

🔵 Mounting (Component Creation)

Method Purpose
constructor() Initialize state
render() Return JSX
componentDidMount() Runs after first render

Example:


componentDidMount() {
    console.log("Component mounted");
}

🟡 Updating (Re-rendering)

Method Purpose
shouldComponentUpdate() Performance control
componentDidUpdate() Runs after update

Example:


componentDidUpdate(prevProps, prevState) {
    if (prevState.count !== this.state.count) {
        console.log("Count changed");
    }
}

🔴 Unmounting (Component Removal)

Method Purpose
componentWillUnmount() Cleanup tasks

Example:


componentWillUnmount() {
    clearInterval(this.timer);
}

Important for:

  • Removing event listeners
  • Clearing timers
  • Cancelling API calls

5️⃣ Event Handling in Class Components


class Button extends React.Component {
    handleClick = () => {
        console.log("Clicked");
    };
    
    render() {
        return <button onClick={this.handleClick}>Click</button>;
    }
}

If not using arrow functions, you must bind:


this.handleClick = this.handleClick.bind(this);

6️⃣ PureComponent

React also provides:


class MyComp extends React.PureComponent {}

It automatically implements shouldComponentUpdate() with shallow comparison for better performance.

7️⃣ Class Component vs Functional Component

Modern React prefers functional components + Hooks.

Example using Hooks:


function Counter() {
    const [count, setCount] = React.useState(0);
    return <h1>{count}</h1>
}

Class components are:

  • Still supported
  • Used in legacy codebases
  • Required for error boundaries (before React 16.8)

8️⃣ When To Use React.Component Today?

Use class components if:

  • Working on old React projects
  • Maintaining legacy code
  • Using Error Boundaries (without Hooks alternative)

Otherwise → prefer functional components.

🔥 Summary

React.Component provides:

  • ✔ State management
  • ✔ Props access
  • ✔ Lifecycle methods
  • ✔ Re-rendering system
  • ✔ Structured component logic

Modern React → prefers Hooks

Legacy React → uses React.Component
