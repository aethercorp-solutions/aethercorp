useEffect

The useEffect hook in React lets your functional components handle side effects—things that happen outside the normal rendering process.

🔹 What counts as a “side effect”?

Anything like:

  • Fetching data from an API
  • Subscribing to events
  • Updating the DOM manually
  • Timers (setTimeout, setInterval)
  • Logging

🔹 Basic Syntax


useEffect(() => {
    // side effect code here
    
    return () => {
        // cleanup (optional)
    };
}, [dependencies]);

🔹 Simple Example


import React, { useEffect } from "react";

const Example = () => {
  useEffect(() => {
    console.log("Component mounted");
  }, []);

  return <h1>Hello</h1>
};

👉 Runs only once when the component mounts.

🔹 Dependency Array Behavior

1. Run once (on mount)


useEffect(() => {
    console.log("Run once");
}, []);

2. Run when dependency changes


useEffect(() => {
    console.log("Count changed");
}, [count]);

👉 Runs every time count changes.

3. Run on every render


useEffect(() => {
    console.log("Runs every render");
});

⚠️ Usually not recommended unless needed.

🔹 Cleanup Function

Used to prevent memory leaks (important in React Native too):


useEffect(() => {
    const interval = setInterval(() => {
        console.log("Running...");
    }, 1000);
    
    return () => {
        clearInterval(interval); // cleanup
    };
}, []);

👉 Cleanup runs when:

  • Component unmounts
  • Before the effect runs again

🔹 Real-World Example (API Call)


import React, { useState, useEffect } from "react";

const Users = () => {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch("https://jsonplaceholder.typicode.com/users")
      .then(res => res.json())
      .then(data => setUsers(data));
  }, []);

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
};

🔹 Common Mistakes

❌ Missing dependencies


useEffect(() => {
    console.log(count);
}, []); // wrong if count is used

❌ Infinite loop


useEffect(() => {
    setCount(count + 1);
}, [count]); // keeps updating forever

🔹 When to Use useEffect

Use it when:

  • You need to sync with external systems
  • You want to run code after render
  • You need lifecycle behavior (like componentDidMount)

Think of useEffect as:

👉 “Run this code after React renders”
