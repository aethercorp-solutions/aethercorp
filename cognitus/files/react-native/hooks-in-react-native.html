Hooks in React Native

In React Native, hooks are special functions that let you use state, lifecycle features, context, and more inside functional components — without writing class components.

React Native uses the same hooks system as React, because React Native is built on top of React.

Why Hooks?

Before hooks, you had to use class components to:

  • Store state
  • Run lifecycle methods
  • Access context
  • Optimize performance

Hooks allow you to do all of that inside simple functions.

Most Common React / React Native Hooks

1️⃣ useState

Adds state to a functional component.


const [count, setCount] = useState(0);

Used for:

  • Form values
  • Toggle states
  • Counters
  • Local UI state

2️⃣ useEffect

Runs side effects (like lifecycle methods).


useEffect(() => {
    console.log("Component mounted");
}, []);

Used for:

  • API calls
  • Subscriptions
  • Timers
  • Listening to events

3️⃣ useContext

Accesses context without nesting components.

4️⃣ useRef

Stores a mutable value that does not trigger re-renders.

5️⃣ useMemo / useCallback

Performance optimization hooks to prevent unnecessary recalculations or re-renders.

Hooks in Other Frameworks

The concept of “hooks” exists in other frameworks too, but it means slightly different things.

🔹 In Vue.js

Vue does not call them hooks the same way React does, but it has:

  • Composition API (similar concept to hooks)
  • ref() – reactive state
  • watch() – react to changes
  • onMounted() – lifecycle hook

Example:


onMounted(() => {
    console.log("Mounted");
});

Vue lifecycle hooks are more structured and built into the framework.

🔹 In Angular

Angular uses lifecycle hooks inside classes:

  • ngOnInit()
  • ngOnDestroy()
  • ngAfterViewInit()

These are methods inside components, not standalone functions like React hooks.

🔹 In Backend Frameworks

The word “hook” usually means:

A function that runs automatically when something happens.

Examples:

  • Database hooks (before save / after update)
  • Authentication hooks
  • Middleware hooks

For example, in Node.js frameworks, hooks can run:

  • Before a request
  • After a response
  • Before writing to database
