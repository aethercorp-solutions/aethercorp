Automating React Native iOS builds to TestFlight using GitHub Actions

Automating React Native iOS builds to TestFlight using GitHub Actions is a common CI/CD setup. The idea is:

  1. Push code to GitHub
  2. GitHub Actions builds the iOS app
  3. Upload automatically to TestFlight through App Store Connect

Below is a practical production setup used by many teams.

1. Requirements

You need the following:

  • Apple Developer account
  • App registered in App Store Connect
  • App Store Connect API Key
  • A Mac-based CI runner (GitHub provides macOS runners)

Technologies used:

  • GitHub Actions – CI/CD pipeline
  • Fastlane – handles iOS build + upload
  • TestFlight – distribution

2. Install Fastlane in Your React Native Project

Inside your iOS folder:


cd ios
bundle init
bundle add fastlane
bundle exec fastlane init

Choose:


2) Automate beta distribution to TestFlight

3. Create Fastlane Lane for TestFlight

Edit:


ios/fastlane/Fastfile

Example:


default_platform(:ios)

platform :ios do
  desc "Build and upload to TestFlight"
  lane :beta do
    build_app(
      scheme: "Castanet",
      export_method: "app-store"
    )

    upload_to_testflight(
      skip_waiting_for_build_processing: true
    )
  end
end

4. Create App Store Connect API Key

Go to:

Users and Access → Keys → App Store Connect API

Download:


AuthKey_XXXX.p8

Save these values:


KEY_ID
ISSUER_ID

5. Add GitHub Secrets

Go to:

GitHub Repo → Settings → Secrets

Add:


APP_STORE_CONNECT_API_KEY
APP_STORE_CONNECT_KEY_ID
APP_STORE_CONNECT_ISSUER_ID
MATCH_PASSWORD (if using match)

Paste the .p8 contents into the API key secret.

6. GitHub Actions Workflow

Create:


.github/workflows/ios-testflight.yml

Example pipeline:


name: iOS TestFlight Build

on:
  push:
    branches:
      - main

jobs:
  build:
    runs-on: macos-latest

    steps:
      - name: Checkout repository
        uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 18

      - name: Install dependencies
        run: |
          npm install

      - name: Install pods
        run: |
          cd ios
          pod install

      - name: Install Fastlane
        run: |
          cd ios
          bundle install

      - name: Build and upload to TestFlight
        env:
          APP_STORE_CONNECT_API_KEY: ${{ secrets.APP_STORE_CONNECT_API_KEY }}
          APP_STORE_CONNECT_KEY_ID: ${{ secrets.APP_STORE_CONNECT_KEY_ID }}
          APP_STORE_CONNECT_ISSUER_ID: ${{ secrets.APP_STORE_CONNECT_ISSUER_ID }}
        run: |
          cd ios
          bundle exec fastlane beta

7. What Happens After Push

When you push to main:


git push origin main

GitHub Actions will:

  1. Start macOS runner
  2. Install Node modules
  3. Install CocoaPods
  4. Build Xcode project
  5. Upload IPA to TestFlight
  6. TestFlight processes the build

Then testers receive the build automatically.

increment_build_number

slack(

9. Recommended Production Architecture


Developer
   ↓ push
GitHub Repo
   ↓
GitHub Actions (macOS runner)
   ↓
Fastlane
   ↓
Xcode build
   ↓
Upload IPA
   ↓
TestFlight
   ↓
Internal / External Testers

10. Pro Tips for React Native CI/CD

For React Native apps like your Castanet platform, add caching:


node_modules
Pods
DerivedData

Example:


actions/cache

This reduces build time from 15 min → ~6 min.
