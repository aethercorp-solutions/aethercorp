Expo

Expo is a toolchain and platform that makes it easier to build React Native apps—especially if you want to avoid dealing with complex native setup (iOS/Android configuration).

Think of it as a shortcut layer on top of React Native.

🔹 What Expo does

Expo simplifies mobile development by giving you:

  • A pre-configured environment (no need to install Xcode/Android Studio initially)
  • Ready-to-use APIs for common features:
    • Camera
    • Location
    • Notifications
    • Sensors
  • Fast development tools like live reload and QR code testing

🔹 How it works

Instead of writing native code, you:

  1. Write JavaScript/TypeScript (React)
  2. Run your app using the Expo Go app on your phone
  3. Scan a QR code → your app runs instantly

🔹 Two main workflows

  1. Managed Workflow (easy mode)
    • No native code
    • Expo handles everything
    • Best for:
      • MVPs
      • Prototypes
      • Standard apps
  2. Bare Workflow (advanced)
    • You can access native code (like normal React Native)
    • Still use Expo tools if needed

🔹 Example

Instead of manually linking native modules in React Native, with Expo you can just do:


npx create-expo-app myApp
cd myApp
npm start

🔹 Pros

  • Fast setup (minutes)
  • Great for beginners and rapid dev
  • Built-in APIs (no need for many third-party libs)
  • OTA (over-the-air) updates

🔹 Cons

  • Limited access to custom native code (in managed mode)
  • Some libraries not supported without ejecting
  • Slightly larger app size

🔹 Expo vs React Native CLI

Feature Expo React Native CLI
Setup Very easy Complex
Native code Limited (unless ejected) Full control
Speed of development Faster Slower initially
Flexibility Medium High

🔹 When YOU should use Expo (based on your background)

Since you're already working with:

  • React Native
  • AWS serverless
  • Advanced integrations (auth, gRPC, etc.)

👉 Use Expo if:

  • You want faster iteration / prototyping
  • Your app doesn’t need heavy native customization

👉 Stick to React Native CLI if:

  • You need deep native integrations
  • You’re doing complex SDK integrations (e.g., custom auth flows, low-level device APIs)

✅ How to force a blank template

Use the --template flag explicitly:

JavaScript blank app


npx create-expo-app myApp --template blank

TypeScript blank app


npx create-expo-app myApp --template blank-typescript
