MongoDB

MongoDB is a popular NoSQL database used to store and manage data in a flexible, scalable way.

Instead of organizing data into rows and tables like traditional relational databases (such as MySQL or PostgreSQL), MongoDB stores data in documents that look like JSON objects.

🧠 Simple idea

Think of MongoDB as a system where data is stored like this:


{
    "name": "Alice",
    "age": 25,
    "skills": ["JavaScript", "Python"]
}

Each of these records is called a document, and a group of documents is called a collection (similar to a table in SQL).

🔑 Key Features

  • Document-oriented

    Data is stored in flexible JSON-like format (called BSON)

  • Schema-less

    You don’t need to define a fixed structure beforehand

  • Scalable

    Designed to handle large amounts of data across multiple servers

  • Fast and flexible

    Great for applications that change frequently

📌 Example use case

A social media app might store user data like posts, comments, and likes in MongoDB because the structure can vary and evolve easily.

When people say “Mongoose MongoDB implementation”, they’re usually talking about using Mongoose as a layer on top of MongoDB in a Node.js app.

Mongoose is an ODM (Object Data Modeling) library—it gives structure, validation, and cleaner code when working with MongoDB.

🧠 Why use Mongoose instead of raw MongoDB?

Using the native MongoDB driver works, but Mongoose adds:

  • Schemas (define structure)
  • Validation (required fields, types, etc.)
  • Models (clean abstraction for queries)
  • Middleware/hooks (before/after save)
  • Better dev experience

⚙️ Basic Implementation (Node.js)

1. Install dependencies


npm install mongoose

2. Connect to MongoDB


const mongoose = require('mongoose');

mongoose.connect('mongodb://127.0.0.1:27017/myapp', {
  useNewUrlParser: true,
  useUnifiedTopology: true,
});

mongoose.connection.on('connected', () => {
  console.log('MongoDB connected');
});

3. Create a Schema


const userSchema = new mongoose.Schema({
    name: {
        type: String,
        required: true
    },
    email: {
        type: String,
        required: true,
        unique: true
    },
    age: Number,
    createdAt: {
        type: Date,
        default: Date.now
    }
});

4. Create a Model


const User = mongoose.model('User', userSchema);

5. CRUD Operations

➤ Create


const user = new User({
    name: 'Paul',
    email: 'paul@example.com',
    age: 30
});
    
await user.save();

➤ Read


const users = await User.find();
const singleUser = await User.findOne({ email: 'paul@example.com' });

➤ Update


await User.updateOne(
  { email: 'paul@example.com' },
  { age: 31 }
);

➤ Delete


await User.deleteOne({ email: 'paul@example.com' });

🔥 Real-world pattern (clean structure)

For a production Node.js API (like your AWS Lambda setup), you’d usually structure it like:


/models
  user.model.js
/controllers
  user.controller.js
/services
  user.service.js

🧩 Example Model File


// models/user.model.js
const mongoose = require('mongoose');

const schema = new mongoose.Schema({
  email: { type: String, required: true, unique: true },
  name: String,
}, { timestamps: true });

module.exports = mongoose.model('User', schema);

⚠️ Important (Serverless / AWS Lambda)

Since you're working with serverless:

  • DON’T reconnect on every request
  • Cache the connection

let isConnected = false;

module.exports.connectDB = async () => {
  if (isConnected) return;

  await mongoose.connect(process.env.MONGO_URI);
  isConnected = true;
};

💡 When to use Mongoose

Use it when:

  • You want structured data + validation
  • You prefer models over raw queries
  • You’re building APIs quickly

Avoid it when:

  • You need ultra-high performance tuning
  • You want full control of raw MongoDB queries
