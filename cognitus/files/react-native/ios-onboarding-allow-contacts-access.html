1️⃣ What is lottie-react-native?

Lottie renders After Effects animations (JSON) natively — smooth, tiny file size, no GIF lag.

Why it’s perfect for onboarding:

  • Explains why permission is needed (Apple likes this)
  • Reduces fear before the system permission popup
  • Feels premium and intentional

Typical use cases:

  • First-launch onboarding
  • Permissions (Contacts, Notifications, Calendar)
  • Empty states

2️⃣ iOS Contacts Permission (Important Reality Check)

⚠️ iOS does NOT allow “select all contacts” via the system picker UI

You either:

  • Request full contacts access
  • Or limited access (user manually picks contacts)

Once granted:


CNAuthorizationStatusAuthorized

Apple’s rule:

You must clearly explain why you need access before showing the permission popup.

That’s where Lottie onboarding comes in.

3️⃣ Recommended Onboarding Flow (Apple-Friendly)

UX Flow


App Launch
   ↓
Onboarding Screen (Lottie animation)
   ↓
Explain benefit (sync / balance / reminders)
   ↓
User taps “Continue”
   ↓
iOS Permission Popup

Key Copy Rules

✅ Say:

  • “Access your contacts to keep everyone in sync”
  • “We never modify or message contacts”

❌ Avoid:

  • “Grant all permissions”
  • “Select all contacts” (sounds forceful)

4️⃣ Example: Lottie Onboarding + Contacts Permission

📦 Install Dependencies


npm install lottie-react-native react-native-permissions react-native-contacts
cd ios && pod install

📁 Example Lottie File


assets/contacts-onboarding.json

(You can get free ones from LottieFiles like “contacts sync” or “address book”)

🧩 Onboarding Screen (Core Example)


import React from 'react';
import { View, Text, TouchableOpacity, StyleSheet } from 'react-native';
import LottieView from 'lottie-react-native';
import { request, PERMISSIONS, RESULTS } from 'react-native-permissions';

export default function ContactsOnboarding({ onDone }) {

    const requestContactsPermission = async () => {
    const result = await request(PERMISSIONS.IOS.CONTACTS);

    if (result === RESULTS.GRANTED) {
        onDone();
    }
    };

    return (
    <View style={styles.container}>
        <LottieView
        source={require('../assets/contacts-onboarding.json')}
        autoPlay
        loop
        style={styles.animation}
        />

        <Text style={styles.title}>
        Stay Connected Effortlessly
        </Text>

        <Text style={styles.description}>
        Allow access to your contacts so we can help you stay organized,
        balanced, and in sync — all in one place.
        </Text>

        <TouchableOpacity
        style={styles.button}
        onPress={requestContactsPermission}
        >
        <Text style={styles.buttonText}>Continue</Text>
        </TouchableOpacity>

        <Text style={styles.footer}>
        We never modify or message your contacts.
        </Text>
    </View>
    );
}

Without this, Apple will reject the app.


<key>NSContactsUsageDescription</key>

We use your contacts to help you stay organized and connected.

6️⃣ After Permission → Access Contacts


import Contacts from 'react-native-contacts';

Contacts.getAll()
  .then(contacts => {
    console.log('Total contacts:', contacts.length);
  })
  .catch(err => {
    console.warn(err);
  });

You now have full access (no picker UI).

7️⃣ Best Practices (From App Store Reviews 💡)

✅ Show onboarding before permission popup

✅ One clear CTA (“Continue”, not “Allow”)

✅ Friendly animation (no scary system icons)

✅ Reassure privacy in small text

❌ Don’t spam permission requests

❌ Don’t block the app if user denies (show retry later)

8️⃣ When to Re-Prompt Permission

If denied:


RESULTS.DENIED → explain again
RESULTS.BLOCKED → link to Settings
            

import { openSettings } from 'react-native-permissions';

openSettings();
