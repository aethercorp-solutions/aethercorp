JSX

JSX stands for JavaScript XML. It’s a syntax used in React and React Native that lets you write UI in a way that looks like HTML, but is actually JavaScript.

🔹 Simple Definition

JSX = JavaScript + HTML-like syntax

It allows you to describe your UI inside your JavaScript code.

🔹 Example

JSX version (what you wrote):


return (
  <View>
    <Text>Hello World</Text>
  </View>
);

What it actually becomes under the hood:


return React.createElement(
  View,
  null,
  React.createElement(Text, null, 'Hello World')
);

👉 JSX is just syntactic sugar for React.createElement(...).

🔹 Why JSX is useful

Without JSX, UI code becomes messy and hard to read. JSX makes it:

  • ✅ More readable
  • ✅ Easier to structure UI
  • ✅ Closer to how you think about layouts

🔹 Important Rules in JSX

1. Must return a single parent element

❌ Wrong:


return (
  <Text>Hello</Text>
  <Text>World</Text>
);

✅ Correct:


return (
  <View>
    <Text>Hello</Text>
    <Text>World</Text>
  </View>
);

2. Use {} to run JavaScript inside JSX


const name = "Paul";

return <Text>Hello {name}</Text>

👉 Output: Hello Paul

3. JSX is NOT exactly HTML

In React Native, you don’t use HTML tags like <div> or <p>.

Instead:

<View> → like </div>

<Text> → for <text/>

<Image> → for <images/>

4. Props look like HTML attributes


<Text style={{ color: 'blue' }}>Hi</Text>

🔹 Think of JSX Like This

Instead of doing:


buildUIManually();

You declare:


<View>
    <Text>Hello</Text>
</View>

👉 It’s declarative UI

🔹 In Your Case (React Native)

JSX is what makes this possible:


<View style={styles.container}>
    <Text<Open up App.tsx</Text>
</View>

You're describing what the UI should look like, not how to build it step-by-step.

🔹 Quick Analogy

  • JavaScript → logic (brain)
  • JSX → UI structure (body)
