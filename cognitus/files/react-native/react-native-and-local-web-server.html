React Native and Local Web Server

Ah, yes! This is a really interesting quirk of React Native. Let’s break it down clearly.

1. React Native and “Local Web Server”

When people say “React Native uses a local web server,” they usually refer to Metro, the JavaScript bundler and development server that ships with React Native.

  • Metro runs on your development machine (or emulator/device when using localhost/IP).
  • It serves your JavaScript bundle, assets, and transforms your JSX/ES6 code into standard JavaScript the mobile runtime can execute.

So, technically, React Native doesn’t render in a browser, but it does fetch the JS bundle from a local server.

2. Why it uses a local server

a) Dynamic bundling & hot reloading

  • Metro watches your project files.
  • When you change a file, it rebundles only the changed code and sends it to the app in real-time.
  • This enables fast refresh (you see changes instantly on the device/emulator).

Without a local server, you’d need to rebuild the entire app every time you change JS.

b) Asset management

  • Images, fonts, or other resources in React Native projects can be referenced via require('./image.png').
  • Metro serves these assets on-demand to the app, similar to how a web server serves static files.

c) Unified development workflow

  • The app itself runs native code (Objective-C/Swift for iOS, Java/Kotlin for Android).
  • Your JavaScript code is not compiled into native binaries.
  • The app fetches your JS bundle from Metro at runtime.
  • This allows a single JS bundle to drive both platforms without rebuilding the native code constantly.

3. How it works in practice

  1. Start the app with: 
    
npx react-native start
  2. Metro starts a local server, usually on http://localhost:8081.
  3. The app running on simulator/device requests the JavaScript bundle from this local server.
  4. If you change code, Metro hot-reloads the app with only the changes, without recompiling native code.

4. Key Points

Feature Why Local Server?
JS bundle delivery App fetches JS code at runtime.
Hot reload / Fast refresh Only changed code is sent instantly.
Asset serving Images, fonts, and other resources are served dynamically.
Development speed No need to rebuild native code every change.

✅ Summary:

React Native uses a “local web server” (Metro) not because the app runs in a browser, but to serve JS bundles and assets dynamically. This enables fast refresh, dynamic updates, and cross-platform JS execution without rebuilding the entire app.
