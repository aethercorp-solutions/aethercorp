React Native functional component

A React Native functional component is a JavaScript (or TypeScript) function that returns JSX to render UI in a React Native app. Since the introduction of Hooks in React (v16.8), functional components have become the standard way to build components instead of class components.

1️⃣ What Is a Functional Component?

A functional component is simply a function that:

  • Accepts props (input data)
  • Returns JSX (UI layout)
  • Can manage state and lifecycle using Hooks

Basic Example


import React from 'react';
import { View, Text } from 'react-native';

const Welcome = () => {
  return (
    <View>
        <Text>Hello World</Text>
    </View>
  );
};

export default Welcome;

This renders “Hello World” on the screen.

2️⃣ Functional Component with Props

Props allow passing data from parent to child.


const Welcome = ({ name }) => {
    return (
        <View>
            <Text>Hello {name}</Text>
        </View>
    );
};

Usage:


<Welcome name="John" />

Props are:

  • Read-only
  • Passed from parent component
  • Used to customize component behavior

3️⃣ Using State in Functional Components (useState)

Before Hooks, only class components could use state. Now we use useState.


import React, { useState } from 'react';
import { View, Text, Button } from 'react-native';

const Counter = () => {
    const [count, setCount] = useState(0);

    return (
        <View>
            <Text<Count: {count}</Text>
            <Button 
                title="Increase"
                onPress={() => setCount(count + 1)}
            />
        </View>
    );
};

How useState works:


const [stateVariable, setStateFunction] = useState(initialValue);
  • count → current state value
  • setCount → function to update state
  • 0 → initial value

4️⃣ Side Effects (useEffect)

useEffect replaces lifecycle methods like:

  • componentDidMount
  • componentDidUpdate
  • componentWillUnmount

import React, { useEffect } from 'react';

useEffect(() => {
  console.log("Component mounted");

  return () => {
    console.log("Component unmounted");
  };
}, []);

Dependency Array:

  • [] → runs once (on mount)
  • [count] → runs when count changes
  • no array → runs on every render

5️⃣ Full Example (Realistic Component)


import React, { useState, useEffect } from 'react';
import { View, Text, Button } from 'react-native';

const Timer = () => {
  const [seconds, setSeconds] = useState(0);

  useEffect(() => {
    const interval = setInterval(() => {
      setSeconds(prev => prev + 1);
    }, 1000);

    return () => clearInterval(interval);
  }, []);

  return (
    <View>
      <Text>Seconds: {seconds}</Text>
    </View>
  );
};

export default Timer;

This:

  • Starts a timer
  • Updates every second
  • Cleans up interval when component unmounts

6️⃣ Why Functional Components Are Preferred

Compared to class components:

Functional Components Class Components
Less boilerplate More verbose
Easier to read More complex
Use Hooks Use lifecycle methods
Better performance optimizations Slightly heavier

Modern React Native development strongly favors functional components.

7️⃣ Important Hooks in Functional Components

Besides useState and useEffect, common hooks include:

  • useContext → Access context
  • useRef → Access DOM/native elements or persist values
  • useMemo → Optimize expensive calculations
  • useCallback → Prevent unnecessary re-renders
  • useReducer → Manage complex state

8️⃣ Best Practices

✅ Keep components small

✅ Destructure props

✅ Use custom hooks for reusable logic

✅ Avoid inline functions in large lists

✅ Use React.memo() when needed

Example:


export default React.memo(MyComponent);

9️⃣ Functional vs Class Component (Quick Comparison)

Class Component


class Example extends React.Component {
    state = { count: 0 };
    
    render() {
        return <Text>{this.state.count}</Text>;
    }
}

Functional Component


const Example = () => {
    const [count] = useState(0);
    return <Text>{count}</Text>;
};

Cleaner and simpler ✅
