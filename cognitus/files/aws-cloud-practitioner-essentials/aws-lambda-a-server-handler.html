AWS Lambda, a server handler

In AWS Lambda, a server handler (commonly called a Lambda handler) is the main function AWS executes when your serverless function is triggered.

Think of it as:


Client Request/Event
        ↓
API Gateway / S3 / EventBridge / etc.
        ↓
Lambda Handler (your entry point)
        ↓
Business Logic
        ↓
Response

Basic Concept

The handler is:

  • the entry point of your Lambda function
  • the function AWS calls automatically
  • responsible for:
    • receiving the event/request
    • processing logic
    • returning a response

Example in Node.js


exports.handler = async (event) => {
    return {
        statusCode: 200,
        body: JSON.stringify({
            message: "Hello from Lambda!"
        })
    };
};

Breakdown

exports.handler

This exposes the function to AWS.

AWS looks for:


fileName.exportedFunctionName

Example:


index.handler

Means:

  • file = index.js
  • exported function = handler

event

Contains incoming request data.

Example from API Gateway:


{
    "httpMethod": "GET",
    "path": "/users",
    "headers": {},
    "body": null
}

You access it like:


event.httpMethod
event.body
event.path

Full Real Example

API Gateway → Lambda


exports.handler = async (event) => {

    console.log("Incoming event:", event);

    const users = [
        { id: 1, name: "Paul" },
        { id: 2, name: "John" }
    ];

    return {
        statusCode: 200,
        headers: {
            "Content-Type": "application/json"
        },
        body: JSON.stringify(users)
    };
};

Request Flow

1. Client calls API


GET /users

2. API Gateway receives request

API Gateway converts request into an event object.

3. Lambda handler executes


exports.handler = async (event) => {

AWS runs this function.

4. Business logic runs


const users = [...]

5. Lambda returns response


return {
    statusCode: 200,
    body: ...
}

6. API Gateway sends response back

Client receives JSON.

Event-Driven Architecture

Lambda handlers can be triggered by many AWS services:

Trigger Example
API Gateway REST API
S3 File upload
DynamoDB Streams Database change
EventBridge Scheduled cron
SQS Queue message
SNS Notifications

Example: S3 Upload Trigger


exports.handler = async (event) => {

    const fileName =
        event.Records[0].s3.object.key;

    console.log("Uploaded file:", fileName);

    return {
        statusCode: 200
    };
};

When a file uploads to S3:


S3 Upload
   ↓
Lambda Handler Executes
   ↓
Process File

Handler in Serverless Framework

Example serverless.yml:


functions:
  getUsers:
    handler: src/users.handler

Meaning:


src/users.js

Common Handler Structure (Production)


exports.handler = async (event) => {

    try {

        // 1. Validate request
        // 2. Authenticate user
        // 3. Execute business logic
        // 4. Return response

        return successResponse();

    } catch (error) {

        console.error(error);

        return errorResponse();
    }
};

Middleware Pattern

Advanced projects use middleware:


export const handler = middy(mainHandler)
    .use(cors())
    .use(authMiddleware())
    .use(errorHandler());

Flow:


Request
  ↓
Middleware
  ↓
Main Handler
  ↓
Middleware
  ↓
Response

Popular middleware library:

Middy.js

Important Lambda Concepts

Cold Start

First execution may be slower because AWS initializes runtime.

Stateless

Lambda should not rely on local memory between executions.

Use:

  • MongoDB
  • DynamoDB
  • Redis
  • S3

for persistence.

Best Practices

Keep handler thin

Good:


Handler
  ↓
Service Layer
  ↓
Database Layer

Bad:


Everything inside handler

Better Architecture


exports.handler = async (event) => {

    const user =
        await userService.getUser();

    return response(user);
};
