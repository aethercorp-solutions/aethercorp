step-by-step guide to deploy a Next.js app to AWS with full CI/CD

Here’s a step-by-step guide to deploy a Next.js app to AWS with full CI/CD, using:

  • Next.js
  • Amazon EC2
  • Amazon ECR
  • Amazon ECS
  • AWS CodePipeline
  • AWS CodeBuild
  • AWS CodeDeploy

This is a production-ready containerized setup.

✅ Architecture Overview


GitHub
   ↓
CodePipeline
   ↓
CodeBuild (Build Docker image)
   ↓
Amazon ECR (Store image)
   ↓
Amazon ECS (Deploy container)
   ↓
Load Balancer → Public URL

STEP 1 — Create a Next.js App


npx create-next-app@latest my-app
cd my-app

Update next.config.js (for standalone build)


module.exports = {
    output: "standalone",
};

STEP 2 — Add Docker Support

Create a Dockerfile:


# Install dependencies
FROM node:18-alpine AS deps
WORKDIR /app
COPY package.json package-lock.json ./
RUN npm ci

# Build app
FROM node:18-alpine AS builder
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY . .
RUN npm run build

# Production image
FROM node:18-alpine AS runner
WORKDIR /app
ENV NODE_ENV=production

COPY --from=builder /app/.next/standalone ./
COPY --from=builder /app/.next/static ./.next/static
COPY --from=builder /app/public ./public

EXPOSE 3000
CMD ["node", "server.js"]

Test locally:


docker build -t nextjs-app .
docker run -p 3000:3000 nextjs-app

STEP 3 — Create Amazon ECR Repository

Go to Amazon ECR → Create Repository

Example:


Repository name: nextjs-app

Authenticate Docker:


aws ecr get-login-password --region us-east-1 | \
docker login --username AWS --password-stdin <account-id>.dkr.ecr.us-east-1.amazonaws.com

Tag and push:


docker tag nextjs-app:latest <account-id>.dkr.ecr.us-east-1.amazonaws.com/nextjs-app
docker push <account-id>.dkr.ecr.us-east-1.amazonaws.com/nextjs-app

STEP 4 — Create ECS Cluster

Go to Amazon ECS → Create Cluster

Choose:

  • Networking only (Fargate)
  • Cluster name: nextjs-cluster

STEP 5 — Create Task Definition

  1. Launch type: Fargate
  2. Task size: 0.5 vCPU / 1GB
  3. Add container:
    • Image: ECR image URI
    • Port mapping: 3000

STEP 6 — Create Service

  1. Launch type: Fargate
  2. Select cluster
  3. Task definition: choose created one
  4. Desired tasks: 1
  5. Attach Application Load Balancer
  6. Create target group (port 3000)

After creation → you get a public Load Balancer URL 🎉

STEP 7 — Setup CI/CD with AWS CodePipeline

Now automate everything.

7.1 Create CodeBuild Project

Go to AWS CodeBuild → Create project

Environment:

  • Managed image
  • Ubuntu
  • Privileged mode: ENABLED (required for Docker)

Add buildspec.yml in your repo:


version: 0.2

phases:
  pre_build:
    commands:
      - echo Logging in to Amazon ECR...
      - aws ecr get-login-password --region $AWS_DEFAULT_REGION | docker login --username AWS --password-stdin $ACCOUNT_ID.dkr.ecr.$AWS_DEFAULT_REGION.amazonaws.com
      - REPOSITORY_URI=$ACCOUNT_ID.dkr.ecr.$AWS_DEFAULT_REGION.amazonaws.com/nextjs-app
      - IMAGE_TAG=latest

  build:
    commands:
      - echo Build started
      - docker build -t $REPOSITORY_URI:$IMAGE_TAG .
  
  post_build:
    commands:
      - docker push $REPOSITORY_URI:$IMAGE_TAG
      - printf '[{"name":"nextjs-container","imageUri":"%s"}]' $REPOSITORY_URI:$IMAGE_TAG > imagedefinitions.json

artifacts:
  files:
    - imagedefinitions.json

Add environment variables in CodeBuild:


ACCOUNT_ID = your-aws-account-id
AWS_DEFAULT_REGION = us-east-1

7.2 Create CodePipeline

Go to AWS CodePipeline → Create Pipeline

Steps:

1️⃣ Source

  • Connect GitHub repo
  • Branch: main

2️⃣ Build

  • Select CodeBuild project

3️⃣ Deploy

  • Deploy provider: Amazon ECS
  • Select:
    • Cluster
    • Service
    • imagedefinitions.json

Now:


Git Push → Auto Build → Auto Push to ECR → Auto Deploy to ECS

🎉 Fully automated CI/CD.

OPTIONAL: Use EC2 Instead of ECS

If you want simpler setup:

  1. Launch EC2 instance
  2. Install Docker
  3. Pull ECR image
  4. Run container
  5. Use CodeDeploy for automatic deployment

But ECS Fargate is recommended for production.
