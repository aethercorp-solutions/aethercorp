Production-ready GitHub Actions deployment pipeline

Since you’re already working with AWS (S3, CloudFront, API Gateway, Lambda) for your React + Node.js setup, I’ll show you a production-ready GitHub Actions deployment pipeline that fits your architecture.

🚀 Automate Deployment with GitHub Actions (AWS + React + Node.js)

🏗 Typical Architecture (Based on Your Setup)

  • Frontend (React Web) → Deploy to:
    • Amazon S3
    • Amazon CloudFront
  • Backend (Node.js API) → Deploy to:
    • AWS Lambda
    • Amazon API Gateway
  • DNS:
    • Amazon Route 53
  • CI/CD:
    • GitHub Actions

🔐 Step 1 — Create IAM User for GitHub

Create a dedicated IAM user:

  1. Go to AWS IAM
  2. Create user: github-deploy-user
  3. Attach policies:

For S3 + CloudFront:

  • AmazonS3FullAccess (or scoped bucket policy)
  • CloudFrontFullAccess

For Lambda:

  • AWSLambdaFullAccess
  • AmazonAPIGatewayAdministrator

⚠️ Production tip: Create a custom scoped policy instead of full access.

Then generate:

  • AWS_ACCESS_KEY_ID
  • AWS_SECRET_ACCESS_KEY

🔒 Step 2 — Add GitHub Secrets

In your GitHub repo:


Settings → Secrets and variables → Actions → New repository secret

Add:


AWS_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY
AWS_REGION
S3_BUCKET_NAME
CLOUDFRONT_DISTRIBUTION_ID

⚛️ Example 1 — Deploy React App to S3 + CloudFront

Create:


.github/workflows/deploy-frontend.yml
            

name: Deploy React App to S3

on:
  push:
    branches:
      - main

jobs:
  build-and-deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout repo
        uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 18

      - name: Install dependencies
        run: npm install

      - name: Build React app
        run: npm run build

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: ${{ secrets.AWS_REGION }}

      - name: Deploy to S3
        run: |
          aws s3 sync build/ s3://${{ secrets.S3_BUCKET_NAME }} --delete

      - name: Invalidate CloudFront
        run: |
          aws cloudfront create-invalidation \
            --distribution-id ${{ secrets.CLOUDFRONT_DISTRIBUTION_ID }} \
            --paths "/*"

What This Does

Every push to main:

  1. Builds React
  2. Uploads to S3
  3. Clears CloudFront cache
  4. Your custom domain updates automatically

🧠 Example 2 — Deploy Node.js Lambda (Serverless Framework)

Since you’re using serverless AWS setup, this is ideal.

Install Serverless in your project:


npm install serverless

Create:


.github/workflows/deploy-backend.yml
            

name: Deploy Backend to Lambda

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 18

      - name: Install dependencies
        run: npm install

      - name: Install Serverless
        run: npm install -g serverless

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v4
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: ${{ secrets.AWS_REGION }}

      - name: Deploy
        run: serverless deploy --stage production

🌎 Production-Level Upgrade (Recommended for You)

Since you're building real production systems (TryCastanet + AWS infra), here’s how to level it up:

1️⃣ Use Environment-Based Deployment

Branches:


dev → deploy to staging
main → deploy to production
            

on:
  push:
    branches:
      - dev
      - main

Then:


run: serverless deploy --stage ${{ github.ref_name }}

2️⃣ Use OIDC Instead of AWS Secrets (Best Practice 🔥)

Instead of storing AWS keys:

Use:

  • GitHub OIDC
  • IAM Role with trust policy
  • Temporary credentials

This is the most secure production method.

3️⃣ Add Automatic Testing Before Deploy


- name: Run tests
  run: npm test

Deployment only happens if tests pass.
