Mongoose model

A Mongoose model is how you define the structure (schema) of documents in a MongoDB collection when using Node.js.

Think of it like:

  • Schema → blueprint (fields, types, validation)
  • Model → actual interface to interact with the database (CRUD)

🔹 1. Install Mongoose


npm install mongoose

🔹 2. Connect to MongoDB


const mongoose = require("mongoose");

mongoose.connect("mongodb://127.0.0.1:27017/mydb", {
  useNewUrlParser: true,
  useUnifiedTopology: true,
});

🔹 3. Create a Schema


const { Schema } = mongoose;

const userSchema = new Schema({
  name: {
    type: String,
    required: true,
  },
  email: {
    type: String,
    required: true,
    unique: true,
    lowercase: true,
  },
  age: {
    type: Number,
    default: 18,
  },
  isActive: {
    type: Boolean,
    default: true,
  },
}, {
  timestamps: true, // adds createdAt & updatedAt
});

🔹 4. Create a Model


const User = mongoose.model("User", userSchema);

👉 This creates a collection:


users   // (Mongoose auto pluralizes "User")

🔹 5. Use the Model (CRUD)

✅ Create


const user = await User.create({
    name: "Paul",
    email: "paul@example.com",
});

✅ Read


const users = await User.find();
const oneUser = await User.findOne({ email: "paul@example.com" });

✅ Update


await User.updateOne(
  { email: "paul@example.com" },
  { age: 30 }
);

✅ Delete


await User.deleteOne({ email: "paul@example.com" });

🔹 6. Advanced Example (Methods + Index)


userSchema.methods.isAdult = function () {
    return this.age >= 18;
};
    
userSchema.index({ email: 1 }); // index for faster queries

🔥 Key Concepts

  • Schema → defines fields & rules
  • Model → interacts with MongoDB
  • Collection → actual stored data
  • Document → one record

⚡ Pro Tips (important for your AWS Lambda setup)

Since you're using serverless:

  • Reuse DB connection (avoid reconnect every invocation)
  • Put mongoose connection in a global scope
  • Disable buffering if needed:

mongoose.set("bufferCommands", false);
