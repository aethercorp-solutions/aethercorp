A modern MongoDB query

A modern MongoDB query usually refers to using the newer MongoDB query syntax and features introduced in recent versions (MongoDB 4.x–7.x), especially with the MongoDB Query API, aggregation pipeline, and modern operators instead of older patterns.

Here are common modern MongoDB query examples.

1. Basic Find Query (Modern Style)


db.users.find(
  { age: { $gte: 18 } },
  { name: 1, email: 1, _id: 0 }
);

Explanation:

  • $gte → greater than or equal
  • second parameter → projection
  • _id:0 removes _id from result

2. Using $in Operator


db.users.find({
    role: { $in: ["admin", "editor"] }
});

Find users where role is admin OR editor.

3. Modern Update Query


db.users.updateOne(
  { email: "user@email.com" },
  { $set: { verified: true } }
);

Or update multiple:


db.users.updateMany(
  { active: false },
  { $set: { archived: true } }
);

4. Aggregation Pipeline (Modern MongoDB)

Modern MongoDB heavily uses aggregation pipelines.


db.orders.aggregate([
  { $match: { status: "completed" } },
  { $group: { _id: "$userId", totalSpent: { $sum: "$amount" } } },
  { $sort: { totalSpent: -1 } }
]);

Steps:

  1. $match → filter
  2. $group → aggregate
  3. $sort → order

5. Lookup (Modern Join)

MongoDB now supports joins via $lookup.


db.orders.aggregate([
  {
    $lookup: {
      from: "users",
      localField: "userId",
      foreignField: "_id",
      as: "user"
    }
  }
]);

Equivalent to SQL JOIN.

6. Modern Text Search


db.posts.find({
    $text: { $search: "mongodb tutorial" }
});

7. Using $expr (Modern Dynamic Query)


db.orders.find({
    $expr: { $gt: ["$amount", "$discount"] }
});

Compares fields inside the document.

8. Modern Query with Date


db.events.find({
    createdAt: {
        $gte: new Date("2026-01-01"),
        $lte: new Date("2026-12-31")
    }
});

9. Projection with Computed Fields


db.users.aggregate([
  {
    $project: {
      name: 1,
      fullEmail: { $concat: ["User: ", "$email"] }
    }
  }
]);

10. Modern Pagination


db.users.find()
  .sort({ createdAt: -1 })
  .skip(20)
  .limit(10);

Example in Node.js (Mongoose or Native Driver)

Since you're working with Node.js + MongoDB in AWS Lambda, a typical modern query:


const users = await User.find({
    role: { $in: ["admin", "editor"] },
    active: true
})
.select("name email")
.sort({ createdAt: -1 })
.limit(20);

Modern MongoDB Best Practices

  1. Use Aggregation Pipeline
  2. Use Indexes
  3. Avoid large skip values
  4. Use projection
  5. Use lean() in Mongoose

Example:


User.find({ active: true }).lean();

Faster because it skips Mongoose document overhead.

2. MongoDB Repository (Modern Query Layer)

calendar.repository.js


const Schedule = require("../models/schedule.model");

class CalendarRepository {

  async getSchedules(filter, limit = 20, skip = 0) {

    return Schedule.aggregate([
      { $match: filter },

      {
        $lookup: {
          from: "users",
          localField: "userId",
          foreignField: "_id",
          as: "user"
        }
      },

      { $unwind: "$user" },

      {
        $project: {
          title: 1,
          startDate: 1,
          endDate: 1,
          "user.name": 1,
          "user.email": 1
        }
      },

      { $sort: { startDate: 1 } },

      { $skip: skip },

      { $limit: limit }

    ]);
  }

}

module.exports = new CalendarRepository();

3. Service Layer (Business Logic)

calendar.service.js


const CalendarRepo = require("../repositories/calendar.repository");

class CalendarService {

  async getCalendarSchedules(userId, start, end, limit, skip) {

    const filter = {
      userId,
      startDate: {
        $gte: start,
        $lte: end
      }
    };

    return CalendarRepo.getSchedules(filter, limit, skip);
  }

}

module.exports = new CalendarService();

4. Controller (AWS Lambda Handler)


const CalendarService = require("../services/calendar.service");

exports.getCalendarSchedules = async (req) => {

  const { userId, start, end, limit, skip } = req.query;

  const data = await CalendarService.getCalendarSchedules(
    userId,
    new Date(start),
    new Date(end),
    Number(limit),
    Number(skip)
  );

  return {
    statusCode: 200,
    body: JSON.stringify(data)
  };
};

5. MongoDB Schema (Mongoose)


const mongoose = require("mongoose");

const ScheduleSchema = new mongoose.Schema({

  userId: {
    type: mongoose.Schema.Types.ObjectId,
    ref: "User"
  },

  title: String,

  startDate: Date,

  endDate: Date

}, { timestamps: true });

module.exports = mongoose.model("Schedule", ScheduleSchema);

8. Performance Best Practices (Important for Lambda)

Use Indexes


db.schedules.createIndex({ userId: 1, startDate: 1 })

Use .lean() when using find()


Schedule.find(filter).lean()

Avoid large .skip()

Instead use cursor pagination

Example:


{ createdAt: { $lt: lastItemDate } }

10. Example Advanced Modern Query (Used by Large Apps)


db.schedules.aggregate([
  { $match: { userId: ObjectId(userId) } },

  {
    $facet: {
      data: [
        { $sort: { startDate: 1 } },
        { $skip: 0 },
        { $limit: 20 }
      ],

      total: [
        { $count: "count" }
      ]
    }
  }
])

Returns:


{
    data: [...],
    total: [{ count: 124 }]
}

Perfect for mobile pagination.

1. Pagination + Total Count ($facet) ✅

👉 Solves: “Get data + total count in one query”


db.schedules.aggregate([
  { $match: { userId: ObjectId(userId) } },

  {
    $facet: {
      data: [
        { $sort: { startDate: -1 } },
        { $skip: 0 },
        { $limit: 10 }
      ],
      total: [
        { $count: "count" }
      ]
    }
  }
