What is a JOIN in databases?

A JOIN is used in SQL to combine rows from two (or more) tables based on a related column between them.

You use a JOIN when data is split into multiple tables (to avoid duplication) but you want to view it together.

Why JOIN is used

  • Keeps data organized and normalized
  • Avoids repeating the same data
  • Allows meaningful queries across related tables

Simple real-world analogy

Think of:

  • Students table → student info
  • Courses table → course info

A JOIN lets you answer:

“Which students are enrolled in which courses?”

Simple example (very common scenario)

Table 1: customers

customer_id name
1 Alice
2 Bob

Table 2: orders

order_id customer_id product
101 1 Laptop
102 1 Phone
103 2 Tablet

The common column is customer_id.

INNER JOIN (most used)

Returns only rows that match in both tables.


SELECT customers.name, orders.product
FROM customers
INNER JOIN orders
ON customers.customer_id = orders.customer_id;

Result:

name product
Alice Laptop
Alice Phone
Bob Tablet

👉 Only customers with orders appear.

LEFT JOIN (very important)

Returns all rows from the left table, even if there is no match in the right table.


SELECT customers.name, orders.product
FROM customers
LEFT JOIN orders
ON customers.customer_id = orders.customer_id;

If Bob had no orders, he would still appear with NULL for product.

Visual explanation


customers        orders
    |               |
    | customer_id   |
    +---------------+
           JOIN

When to use JOIN

  • Show customer names with their orders
  • Show employees with their departments
  • Show students with their grades

Key takeaway

  • JOIN connects tables
  • INNER JOIN → only matching data
  • LEFT JOIN → keep all rows from left table
