What is Firebase Anonymous Authentication?

Firebase Anonymous Authentication is a feature of Firebase (part of Google) that allows users to sign in to your app without creating an account or providing credentials (no email, phone, or social login required).

It creates a temporary, unique user ID (UID) for the user so they can:

  • Access authenticated features
  • Read/write to Firestore or Realtime Database (based on rules)
  • Save user-specific data (cart, preferences, progress, etc.)
  • Upgrade later to a permanent account (email, Google, etc.)

🔎 Why Use Anonymous Authentication?

✅ Common Use Cases

  • Guest checkout in e-commerce apps
  • Try-before-signup experience
  • Saving game progress before registration
  • Temporary sessions for content apps

✅ Benefits

  • No friction for first-time users
  • You still get a Firebase UID
  • Can link to permanent account later
  • Works with Firestore security rules

⚠️ Important Things to Know

  • Anonymous accounts are real Firebase users
  • They persist until:
    • User logs out
    • App deletes account
    • You delete them manually
  • On paid Firebase plans, anonymous users count as MAU (Monthly Active Users)

🚀 Step-by-Step Setup in React Native

Assuming you're using React Native CLI (not Expo managed).

We will use:

  • @react-native-firebase/app
  • @react-native-firebase/auth

1️⃣ Create Firebase Project

  1. Go to: https://console.firebase.google.com
  2. Click Add Project
  3. Follow setup steps
  4. Add:
    • Android app (download google-services.json)
    • iOS app (download GoogleService-Info.plist)

2️⃣ Enable Anonymous Authentication

  1. In Firebase Console
  2. Go to Authentication
  3. Click Sign-in method
  4. Enable Anonymous
  5. Save

3️⃣ Install React Native Firebase


npm install @react-native-firebase/app
npm install @react-native-firebase/auth

iOS Only


cd ios
pod install

4️⃣ Configure Android

🔹 Add google-services.json

Place it inside:


android/app/google-services.json

🔹 Update android/build.gradle


buildscript {
    dependencies {
        classpath 'com.google.gms:google-services:4.3.15'
    }
}

🔹 Update android/app/build.gradle


apply plugin: 'com.google.gms.google-services'

5️⃣ Configure iOS

  • Add GoogleService-Info.plist into ios/YourProject
  • Make sure it's added to Xcode project
  • Run pod install

6️⃣ Sign In Anonymously (React Native Code)


import auth from '@react-native-firebase/auth';

const signInAnonymously = async () => {
  try {
    const userCredential = await auth().signInAnonymously();
    console.log('User ID:', userCredential.user.uid);
  } catch (error) {
    console.log('Error:', error);
  }
};

7️⃣ Check Current User


const user = auth().currentUser;

if (user) {
  console.log('User is signed in:', user.uid);
} else {
  console.log('No user signed in');
}

8️⃣ Listen for Auth State Changes (Recommended)


useEffect(() => {
    const subscriber = auth().onAuthStateChanged(user => {
        if (user) {
        console.log('User logged in:', user.uid);
        } else {
        console.log('User logged out');
        }
    });
    
    return subscriber; // unsubscribe on unmount
}, []);

🔄 Upgrade Anonymous User to Permanent Account

Very important feature!

Example: Link to Email & Password


import auth from '@react-native-firebase/auth';

const linkAccount = async () => {
  const credential = auth.EmailAuthProvider.credential(
    'test@email.com',
    'password123'
  );

  await auth().currentUser.linkWithCredential(credential);
};

This keeps:

  • Same UID
  • Same Firestore data
  • Same user identity

🗑 Delete Anonymous User


await auth().currentUser.delete();

🔐 Firestore Security Rule Example


rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId} {
      allow read, write: if request.auth != null && request.auth.uid == userId;
    }
  }
}

Anonymous users will pass request.auth != null.

Since you already have Google Sign-In working with Firebase Auth, adding Anonymous Sign-In is actually very simple. The important part is deciding how you want it to behave:

You have two common patterns:

  1. ✅ Allow users to continue as Guest (anonymous)
  2. ✅ Automatically start as anonymous → later upgrade to Google (best UX)

I’ll explain both clearly.

✅ Step 1: Enable Anonymous in Firebase Console

  1. Go to Firebase Console
  2. Authentication → Sign-in method
  3. Enable Anonymous
  4. Save

That’s it on the Firebase side.

✅ Step 2: Add Anonymous Sign-In in React Native

Since you're already using:


@react-native-firebase/auth

Just add:


import auth from '@react-native-firebase/auth';

const signInAsGuest = async () => {
  try {
    const userCredential = await auth().signInAnonymously();
    console.log('Guest UID:', userCredential.user.uid);
  } catch (error) {
    console.log(error);
  }
};

Done. 🎉

🚀 Recommended Architecture (Best Practice)

Instead of showing login immediately:

👉 On app start:

Automatically sign in anonymously if no user exists.


useEffect(() => {
    const subscriber = auth().onAuthStateChanged(async (user) => {
        if (!user) {
        await auth().signInAnonymously();
        }
    });
    
    return subscriber;
}, []);

Now every user has a UID from the beginning.

🔥 Very Important: Linking Anonymous to Google

If the user later chooses "Sign in with Google", you must LINK the account — not sign in normally.

Otherwise:

  • Anonymous account gets lost
  • New UID is created
  • Firestore data breaks

❌ Wrong Way (creates new user)


auth().signInWithCredential(googleCredential);

✅ Correct Way (link accounts)

If current user is anonymous:


import auth from '@react-native-firebase/auth';

const signInWithGoogleAndLink = async (googleCredential) => {
  const currentUser = auth().currentUser;

  if (currentUser?.isAnonymous) {
    await currentUser.linkWithCredential(googleCredential);
    console.log('Anonymous account upgraded to Google');
  } else {
    await auth().signInWithCredential(googleCredential);
  }
};

🔎 How to Detect Anonymous User


const user = auth().currentUser;

if (user?.isAnonymous) {
  console.log('User is guest');
} else {
  console.log('User is permanent');
}

🔐 What Happens Internally

Action Result
signInAnonymously() Creates temporary UID
linkWithCredential() Converts to permanent
signInWithCredential() Creates NEW user

🧠 Best UX Flow (Recommended)

  1. App opens
  2. Auto anonymous login
  3. User uses app normally
  4. When needed → show "Continue with Google"
  5. Link account
  6. Same UID, same data, no loss

This is how many production apps work.

⚠️ Important Edge Case

If user:

  • Deletes app
  • Clears storage
  • Logs out manually

Anonymous account is gone unless linked.

🏁 Final Summary

Since you already have Google Sign-In:

✔ Enable Anonymous in Firebase

✔ Add signInAnonymously()

✔ Link with linkWithCredential() when upgrading

✔ Never replace anonymous user with signInWithCredential()
