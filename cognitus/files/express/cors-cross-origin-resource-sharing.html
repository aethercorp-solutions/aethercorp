CORS (Cross-Origin Resource Sharing)

CORS (Cross-Origin Resource Sharing) in a Node.js Express app is a browser security mechanism that controls which domains (origins) are allowed to access your API.

🔒 Why CORS exists

Browsers enforce something called the Same-Origin Policy:

A frontend running on one origin (domain + protocol + port) cannot directly call another origin unless explicitly allowed.

Example:

  • Frontend: http://localhost:3000
  • Backend API: https://api.myapp.com

👉 These are different origins, so the browser blocks the request unless your server allows it via CORS.

⚙️ How CORS works in Express

In a Node.js Express app, you usually enable CORS using the cors middleware.

Basic example:


const express = require('express');
const cors = require('cors');

const app = express();

app.use(cors()); // allow all origins

app.get('/api', (req, res) => {
  res.json({ message: 'CORS enabled' });
});

👉 This allows any website to call your API.

🎯 Restricting access (more secure)

You can limit which origins are allowed:


const allowedOrigins = [
  'http://localhost:3000',
  'https://myapp.com'
];

app.use(
  cors({
    origin: (origin, callback) => {
      if (!origin || allowedOrigins.includes(origin)) {
        callback(null, true);
      } else {
        callback(new Error('Not allowed by CORS'));
      }
    }
  })
);

👉 Only specified domains can access your API.

🔁 What happens under the hood

When a browser makes a cross-origin request:

  1. It sends a preflight request (OPTIONS) if needed
  2. Your server responds with headers like:
    • Access-Control-Allow-Origin
    • Access-Control-Allow-Methods
    • Access-Control-Allow-Headers

Example response header:


Access-Control-Allow-Origin: https://myapp.com

👉 This tells the browser: “It’s safe to allow this request.”

🚨 Important clarifications

  • CORS is enforced by browsers, not by Node.js itself
  • Tools like Postman or curl ignore CORS
  • CORS does not secure your API — it only controls browser access

🧠 Simple analogy

Think of CORS like a bouncer at a club:

  • Your API = the club
  • Different websites = people trying to enter
  • CORS = the guest list

Only allowed origins get in.
