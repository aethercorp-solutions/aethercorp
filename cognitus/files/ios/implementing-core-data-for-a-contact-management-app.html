Implementing Core Data for a contact management app using Swift

Here’s a clean, practical example of implementing Core Data for a contact management app using Swift

+ UIKit. This walks through:

  • Core Data setup
  • Contact entity
  • CRUD operations
  • Simple UIKit integration (UITableView)

1. Core Data Setup

If you created your project with “Use Core Data” checked, you already have a PersistenceController or AppDelegate setup.

AppDelegate.swift


import UIKit
import CoreData

@main
class AppDelegate: UIResponder, UIApplicationDelegate {

    lazy var persistentContainer: NSPersistentContainer = {
        let container = NSPersistentContainer(name: "ContactsModel")
        container.loadPersistentStores { _, error in
            if let error = error {
                fatalError("Core Data failed: \(error)")
            }
        }
        return container
    }()

    func saveContext () {
        let context = persistentContainer.viewContext
        if context.hasChanges {
            try? context.save()
        }
    }
}

2. Create Core Data Entity

Open .xcdatamodeld file and create:

Entity: Contact

Attributes:

  • name (String)
  • phone (String)
  • email (String)

Then generate NSManagedObject:


import Foundation
import CoreData

@objc(Contact)
public class Contact: NSManagedObject {}

extension Contact {
    @nonobjc public class func fetchRequest() -> NSFetchRequest<Contact> {
        return NSFetchRequest<Contact>(entityName: "Contact")
    }

    @NSManaged public var name: String?
    @NSManaged public var phone: String?
    @NSManaged public var email: String?
}

3. Core Data Manager (Clean Architecture)

Create a helper class:


import UIKit
import CoreData

class CoreDataManager {

    static let shared = CoreDataManager()
    private let context = (UIApplication.shared.delegate as! AppDelegate).persistentContainer.viewContext

    // CREATE
    func createContact(name: String, phone: String, email: String) {
        let contact = Contact(context: context)
        contact.name = name
        contact.phone = phone
        contact.email = email

        save()
    }

    // READ
    func fetchContacts() -> [Contact] {
        let request: NSFetchRequest<Contact> = Contact.fetchRequest()
        return (try? context.fetch(request)) ?? []
    }

    // UPDATE
    func updateContact(contact: Contact, name: String, phone: String, email: String) {
        contact.name = name
        contact.phone = phone
        contact.email = email

        save()
    }

    // DELETE
    func deleteContact(contact: Contact) {
        context.delete(contact)
        save()
    }

    private func save() {
        try? context.save()
    }
}

4. UIKit TableView Example

ContactsViewController.swift


import UIKit

class ContactsViewController: UIViewController {

    @IBOutlet weak var tableView: UITableView!

    var contacts: [Contact] = []

    override func viewDidLoad() {
        super.viewDidLoad()
        tableView.dataSource = self
        loadData()
    }

    func loadData() {
        contacts = CoreDataManager.shared.fetchContacts()
        tableView.reloadData()
    }

    @IBAction func addContactTapped() {
        CoreDataManager.shared.createContact(
            name: "John Doe",
            phone: "123456789",
            email: "john@email.com"
        )
        loadData()
    }
}

UITableViewDataSource


extension ContactsViewController: UITableViewDataSource {

    func tableView(_ tableView: UITableView, numberOfRowsInSection section: Int) -> Int {
        contacts.count
    }

    func tableView(_ tableView: UITableView,
                    cellForRowAt indexPath: IndexPath) -> UITableViewCell {

        let cell = tableView.dequeueReusableCell(withIdentifier: "cell", for: indexPath)

        let contact = contacts[indexPath.row]
        cell.textLabel?.text = contact.name
        cell.detailTextLabel?.text = contact.phone

        return cell
    }
}

5. Delete Support (Swipe to Delete)


extension ContactsViewController: UITableViewDelegate {

    func tableView(_ tableView: UITableView,
                    commit editingStyle: UITableViewCell.EditingStyle,
                    forRowAt indexPath: IndexPath) {

        if editingStyle == .delete {
            let contact = contacts[indexPath.row]
            CoreDataManager.shared.deleteContact(contact: contact)
            loadData()
        }
    }
}

6. Best Practices

  • Use NSFetchedResultsController for large datasets (auto UI updates)
  • Avoid force casting AppDelegate in production → inject dependencies
  • Add validation (email/phone formatting)
  • Use background context for heavy operations

Core Data

In Core Data, “search functionality” usually refers to how you fetch, filter, and sort data stored in your persistent store (SQLite under the hood in most cases). The core mechanism for searching is the NSFetchRequest.

Here’s a clear breakdown of the main parts that make up Core Data search functionality:

1. NSFetchRequest (the core of searching)

This is the primary way to query data.


let request: NSFetchRequest<Person> = Person.fetchRequest()

It defines:

  • What entity you’re searching (e.g., Person)
  • Optional filters (predicate)
  • Sorting rules

2. NSPredicate (filtering/search conditions)

This is where the “search logic” lives.

Example: exact match


request.predicate = NSPredicate(format: "name == %@", "John")

Example: partial text search


request.predicate = NSPredicate(format: "name CONTAINS[cd] %@", "jo")
  • [c] → case-insensitive
  • [d] → diacritic-insensitive

Example: multiple conditions


NSPredicate(format: "name CONTAINS[cd] %@ AND age > %d", "jo", 18)

3. NSSortDescriptor (ordering results)

Not required for search, but commonly used with it.


request.sortDescriptors = [
    NSSortDescriptor(key: "name", ascending: true)
]

4. Executing the search


do {
    let results = try context.fetch(request)
} catch {
    print("Fetch failed")
}

5. Common search patterns

“Search bar” style filtering


let query = searchText
request.predicate = NSPredicate(format: "name CONTAINS[cd] %@", query)

Search across multiple fields


NSPredicate(format: "name CONTAINS[cd] %@ OR email CONTAINS[cd] %@", query, query)

Fetch everything (no filter)


request.predicate = nil

6. Performance considerations (important in Core Data search)

  • Use indexed attributes for frequently searched fields
  • Avoid overly complex predicates on large datasets
  • Prefer CONTAINS carefully (can be slower than exact matches)
  • Use batch size for large results:

request.fetchBatchSize = 20

7. Advanced: Compound predicates


let p1 = NSPredicate(format: "name CONTAINS[cd] %@", query)
let p2 = NSPredicate(format: "age > %d", 18)

request.predicate = NSCompoundPredicate(andPredicateWithSubpredicates: [p1, p2])

Summary

Core Data search functionality is mainly built from:

  • NSFetchRequest → defines the query
  • NSPredicate → filters data (the “search engine”)
  • NSSortDescriptor → orders results
  • context.fetch() → executes the search
