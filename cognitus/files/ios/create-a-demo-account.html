Create a Demo Account

Got it! You want to let Apple reviewers log in to your app without needing their own Apple ID or triggering 2FA/OTP, while still using your Firebase + Google Sign-In + React Native setup. I’ll break it step by step for a demo/test account.

Step 1: Create a Demo Account in Firebase

  1. Go to Firebase Console → Authentication → Users.
  2. Click Add User.
    • Email: apple-reviewer@example.com (any email you choose)
    • Password: DemoPassword123 (use something simple)
  3. Optionally, fill in display name, photo URL, etc., to make it realistic.
  4. Save the user.

✅ This is your “Apple reviewer” demo account. No OTP or Google Sign-In is needed for this account.

Step 2: Create a “Demo Login” API Endpoint in Node.js

Your AWS Lambda backend likely has an endpoint for authentication with Firebase. Add a special endpoint for the demo account:


// AWS Lambda / Node.js example
const admin = require("firebase-admin");

// Initialize Firebase Admin SDK if not already done
// admin.initializeApp();

exports.handler = async (event) => {
  try {
    const { demo } = JSON.parse(event.body);

    if (demo === true) {
      // Demo account credentials
      const email = "apple-reviewer@example.com";
      const password = "DemoPassword123";

      // Authenticate demo user using Firebase Admin
      const user = await admin.auth().getUserByEmail(email);

      // Create a custom token for client
      const token = await admin.auth().createCustomToken(user.uid);

      return {
        statusCode: 200,
        body: JSON.stringify({ token }),
      };
    }

    return {
      statusCode: 400,
      body: JSON.stringify({ error: "Not a demo login request" }),
    };
  } catch (error) {
    console.error(error);
    return { statusCode: 500, body: JSON.stringify({ error: error.message }) };
  }
};

✅ This returns a Firebase custom token for the demo account.

Step 3: Modify React Native App for Demo Login

In your app, provide a hidden button or special path that Apple reviewers can use.


import auth from "@react-native-firebase/auth";

async function loginDemoAccount() {
  try {
    const response = await fetch("https://your-api.com/demo-login", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ demo: true }),
    });

    const data = await response.json();

    // Sign in to Firebase using the custom token
    await auth().signInWithCustomToken(data.token);

    console.log("Logged in as demo user");
  } catch (error) {
    console.error("Demo login failed", error);
  }
}
  • You can hide this button in a debug/test screen so only Apple reviewers know about it.
  • No OTP or Apple ID required.

Step 4: Provide Demo Credentials to Apple Reviewer

  1. In App Store Connect → App Review Information → Demo Account, enter:
    • Username: apple-reviewer@example.com
    • Password: DemoPassword123
    • Optional notes: “Press ‘Demo Login’ button on login screen.”
  2. Ensure this account bypasses any 2FA or Google Sign-In flow.

Step 5: Optional: Skip Social Login for Demo

If your app forces Google Sign-In normally:

  • Add a conditional flag to skip social login for demo account.
  • Example:

if (isAppleReviewer) {
    // Show demo login button
} else {
    // Normal Google Sign-In flow
}

✅ Summary

  1. Create Firebase demo account.
  2. Add a secure demo-login API endpoint returning a Firebase custom token.
  3. In React Native, call that endpoint to sign in.
  4. Hide the demo login button from normal users.
  5. Provide credentials to Apple reviewer in App Store Connect.
