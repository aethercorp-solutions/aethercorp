guard

In Swift, guard is a control flow statement used to exit early from a scope unless certain conditions are met. It’s designed to make your code cleaner, safer, and easier to read—especially compared to nested if statements.

🔹 Basic Idea

A guard statement checks a condition.

  • If the condition is true, execution continues normally.
  • If the condition is false, the else block must exit the current scope (using return, break, continue, or throw).

🔹 Syntax


guard condition else {
    // must exit scope
    return
}

🔹 Simple Example


func greet(name: String?) {
    guard name != nil else {
        print("No name provided")
        return
    }

    print("Hello, \(name!)")
}

What’s happening:

  • If name is nil, the function exits early.
  • If not, execution continues and you can safely use name.

🔹 Optional Binding with guard let

This is one of the most common uses:


func greet(name: String?) {
    guard let unwrappedName = name else {
        print("No name")
        return
    }

    print("Hello, \(unwrappedName)")
}

Why this is powerful:

  • The unwrapped variable (unwrappedName) is available after the guard statement.
  • No need for force unwrapping (!), which avoids crashes.

🔹 Multiple Conditions


func login(username: String?, password: String?) {
    guard let user = username,
            let pass = password,
            pass.count >= 6 else {
        print("Invalid credentials")
        return
    }

    print("Welcome, \(user)")
}

You can combine:

  • Optional binding
  • Boolean conditions

🔹 Why Use guard Instead of if?

❌ Without guard (nested ifs)


func process(value: Int?) {
    if let value = value {
        if value > 0 {
            print("Valid: \(value)")
        } else {
            print("Not positive")
        }
    } else {
        print("Nil value")
    }
}

✅ With guard (cleaner)


func process(value: Int?) {
    guard let value = value else {
        print("Nil value")
        return
    }

    guard value > 0 else {
        print("Not positive")
        return
    }

    print("Valid: \(value)")
}

Key benefit:

  • Reduces nesting
  • Improves readability
  • Makes the “happy path” clearer

🔹 Where guard Must Exit

Inside the else block, you must use:

  • return → exit function
  • throw → throw error
  • break → exit loop/switch
  • continue → skip loop iteration

Example in a loop:


for number in [1, -1, 3] {
    guard number > 0 else {
        continue
    }
    print(number)
}

🔹 Common Use Cases

1. Input validation


guard !username.isEmpty else { return }

2. Early exit in functions


guard isLoggedIn else { return }

3. Safe optional handling


guard let data = data else { return }

4. Prevent invalid states


guard index < array.count else { return }

🔹 Mental Model

Think of guard as:

“Make sure this condition is true, or stop everything right here.”

🔹 When NOT to Use guard

Avoid guard when:

  • You don’t need to exit early
  • The logic is better expressed as a normal conditional branch

Example where if is better:


if isPremiumUser {
    showPremiumFeatures()
} else {
    showBasicFeatures()
}

🔹 Summary

  • guard enforces early exit
  • Helps avoid deep nesting
  • Keeps code clean and readable
  • Essential for safe optional unwrapping
