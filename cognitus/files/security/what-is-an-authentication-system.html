Authentication System

An authentication system is a way to verify that a person (or system) is who they claim to be before granting access to something—like an app, website, or network.

In simple terms:

👉 Authentication = proving your identity

🔑 Common examples

  • Entering a username and password
  • Logging in with Google or Facebook
  • Using fingerprint or face recognition
  • Receiving a one-time code (OTP) on your phone

🧠 How it works (basic idea)

  1. You provide some form of identity (e.g., username or email).
  2. You provide proof (e.g., password, fingerprint, code).
  3. The system checks if the proof matches its records.
  4. If it matches → access granted

    If not → access denied

🔒 Types of authentication

  • Single-factor authentication (SFA):

    One method only (e.g., password)

  • Two-factor authentication (2FA):

    Two methods (e.g., password + OTP)

  • Multi-factor authentication (MFA):

    Two or more methods, usually from different categories:

    • Something you know (password)
    • Something you have (phone, token)
    • Something you are (biometrics)

🔍 Authentication vs Authorization

These are often confused:

  • Authentication: “Who are you?”
  • Authorization: “What are you allowed to do?”

📌 Why it matters

Authentication systems protect:

  • Personal accounts (email, social media)
  • Financial data (banking apps)
  • Sensitive systems (company networks)

Without authentication, anyone could access anything.
