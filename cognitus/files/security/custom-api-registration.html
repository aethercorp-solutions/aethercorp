Custom API Registration

Right now your API can log users in without actually creating or validating them—so “registration” is the missing piece before this becomes a real auth system.

Let’s wire that in properly.

✅ What you need to add

  1. A register route
  2. Password hashing (never store plain passwords)
  3. (Optional but recommended) simple in-memory DB or real DB

🔧 Step 1: Install bcrypt


npm install bcrypt

🔧 Step 2: Update your authRoutes

Here’s a clean version with register + login (with password validation):


const express = require('express');
const jwt = require('jsonwebtoken');
const bcrypt = require('bcrypt');

const router = express.Router();

// TEMP: Fake DB (replace with MongoDB later)
const users = [];

/**
 * REGISTER
 */
router.post('/register', async (req, res) => {
  try {
    const { email, password } = req.body;

    // Basic validation
    if (!email || !password) {
      return res.status(400).json({
        status: false,
        message: 'Email and password are required'
      });
    }

    // Check if user exists
    const existingUser = users.find(u => u.email === email);
    if (existingUser) {
      return res.status(400).json({
        status: false,
        message: 'User already exists'
      });
    }

    // Hash password
    const hashedPassword = await bcrypt.hash(password, 10);

    const newUser = {
      id: users.length + 1,
      email,
      password: hashedPassword
    };

    users.push(newUser);

    res.json({
      status: true,
      message: 'User registered successfully'
    });

  } catch (error) {
    res.status(500).json({
      status: false,
      message: 'Server error'
    });
  }
});


/**
 * LOGIN
 */
router.post('/login', async (req, res) => {
  try {
    const { email, password } = req.body;

    const user = users.find(u => u.email === email);

    if (!user) {
      return res.status(400).json({
        status: false,
        message: 'Invalid credentials'
      });
    }

    // Compare password
    const isMatch = await bcrypt.compare(password, user.password);

    if (!isMatch) {
      return res.status(400).json({
        status: false,
        message: 'Invalid credentials'
      });
    }

    // Create JWT
    const token = jwt.sign(
      { id: user.id, email: user.email },
      process.env.JWT_SECRET,
      { expiresIn: '7d' }
    );

    res.json({
      status: true,
      token
    });

  } catch (error) {
    res.status(500).json({
      status: false,
      message: 'Server error'
    });
  }
});

module.exports = router;

🔐 What changed (important)

  • ✔ Added /register
  • ✔ Password is hashed using bcrypt
  • ✔ Login now validates password
  • ✔ JWT only includes safe fields (no password)

⚠️ Important Reality Check

This works, but:

  • ❌ In-memory users[] resets on restart
  • ❌ Not scalable
  • ❌ No email verification

🚀 What you should do next (based on your stack)

Since you're already using:

  • AWS Lambda
  • MongoDB (Mongoose)

Upgrade this to:

  1. Replace users[] with MongoDB model
  2. Add:
    • email validation
    • password rules
    • rate limiting
  3. Store JWT secret in AWS Secrets Manager

Here’s how you can test your API using curl step by step.

✅ 1. Test server is running


curl http://localhost:3000/

Expected:


{ "message": "aethercorp api is running 🚀" }

✅ 2. Register a user


curl -X POST http://localhost:3000/auth/register \
-H "Content-Type: application/json" \
-d '{
  "email": "test@email.com",
  "password": "123456"
}'

Expected:


{
    "status": true,
    "message": "User registered successfully"
}

✅ 3. Login (get JWT token)


curl -X POST http://localhost:3000/auth/login \
-H "Content-Type: application/json" \
-d '{
  "email": "test@email.com",
  "password": "123456"
}'

Expected:


{
    "status": true,
    "token": "YOUR_JWT_TOKEN"
}

👉 Copy the token.

✅ 4. Access protected route

Replace YOUR_JWT_TOKEN:


curl http://localhost:3000/secure \
-H "Authorization: Bearer YOUR_JWT_TOKEN"

Expected:


{
    "message": "Secure data 🔐",
    "user": {
        "id": 1,
        "email": "test@email.com",
        "iat": ...,
        "exp": ...
    }
}

❌ Common errors (quick debug)

401 No token


{ "message": "No token provided" }

➡️ You forgot the Authorization header

401 Invalid token


{ "message": "Invalid or expired token" }

➡️ Token is wrong or expired

400 Invalid credentials

➡️ Email/password mismatch

⚡ Pro tip (faster testing)

Store token in a variable:


TOKEN="paste_your_token_here"

curl http://localhost:3000/secure \
-H "Authorization: Bearer $TOKEN"
