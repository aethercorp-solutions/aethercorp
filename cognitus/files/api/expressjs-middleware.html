Express.js middleware

This function is an Express.js middleware that handles Google OAuth login using an authorization code. Let’s walk through what it actually does step by step.

🔑 High-level purpose

It:

  1. Receives a Google OAuth authorization code from the request headers
  2. Exchanges that code for an access token (via GoogleService)
  3. Converts it into a usable token format
  4. Passes control to another function (signInWithGoogleToken) to complete login

The Google authorization code is a temporary, one-time code used in the OAuth 2.0 flow—specifically in the Authorization Code Grant.

Think of it as a short-lived “ticket” that proves a user successfully logged in with Google, which your backend can exchange for actual tokens.

🧠 Simple idea

It’s a two-step exchange:

  1. The user logs in → you get a temporary code
  2. Your backend swaps that code → for real tokens

🔁 Flow (clean breakdown)

1. Redirect user to Google login

Your app opens something like:


https://accounts.google.com/o/oauth2/v2/auth?
  client_id=...
  redirect_uri=...
  response_type=code
  scope=openid email profile

2. User logs in & approves

Google authenticates the user.

3. Google returns an authorization code

Redirect:


yourapp://callback?code=AUTH_CODE

This is the Authorization Code Grant part—you now have the code.

4. Backend exchanges the code

Your server (e.g., AWS Lambda) sends:


POST https://oauth2.googleapis.com/token

With:

  • code
  • client_id
  • client_secret
  • redirect_uri
  • grant_type=authorization_code

5. Google returns tokens

You get:

  • access_token → call APIs
  • id_token → user identity (JWT)
  • refresh_token → long-term access

🔐 Why this flow is important

Compared to simpler flows, Authorization Code Grant:

  • Keeps client_secret only on backend
  • Prevents tokens from being exposed to the frontend
  • Supports refresh tokens
  • Is recommended for:
    • Mobile apps (with PKCE)
    • Web apps
    • Server-side apps

🔒 Modern version: PKCE

For mobile apps like your React Native setup, you should use:

👉 Authorization Code Grant + PKCE (Proof Key for Code Exchange)

This adds:

  • code_verifier
  • code_challenge

So even if someone steals the authorization code, they can’t exchange it without the verifier.

🧠 Step-by-step breakdown

1. Check if code exists


if (req.headers.code)
  • The client (your React Native app) must send a Google authorization code in headers.
  • If missing → throws error via next().

2. Exchange code for token


let token = await GoogleService.instance.exchangeToken(req.headers.code)
  • Calls your custom service that hits Google OAuth endpoint.
  • Converts the authorization code → access token + refresh token.

If it fails:


next(new ApplicationError(err.message, err.response.status))

3. Add expiration date (⚠️ BUG here)


token.expiration_date = new Date(now + token.expires_in * (1000 * 1000)).toISOString();
  • expires_in is already in seconds
  • Correct conversion should be:

token.expiration_date = new Date(now + token.expires_in * 1000).toISOString();

👉 Your code multiplies by 1000 * 1000 (1,000,000), which makes the expiration 1000× longer than it should be.

4. Attach token to request


req.headers.token = token;
  • Stores the token in headers (not ideal — better to use req.auth or req.user).

5. Continue login flow


await signInWithGoogleToken(req, res, next);
  • This likely:
    • Verifies token with Google
    • Extracts user profile
    • Creates/fetches user in DB
    • Generates your app JWT

6. Handle invalid token


res.status(403).send({ status: false, message: 'Invalid code.' });

⚠️ Key issues / improvements

❌ 1. Wrong expiration math

Fix:


token.expiration_date = new Date(now + token.expires_in * 1000).toISOString();

❌ 2. Using headers for sensitive data

Headers are not meant for internal mutation.

Better:


req.googleToken = token;

❌ 3. Missing return after next()

You should stop execution:


return next(new ApplicationError(...));

❌ 4. OAuth compliance issue

Google expects:

  • POST request
  • application/x-www-form-urlencoded

Make sure your exchangeToken() follows proper OAuth 2.0 flow.

✅ Cleaned-up version


exports.signInWithGoogleCode = async (req, res, next) => {
    try {
        const code = req.headers.code;
        if (!code) {
            return next(new ApplicationError('Params not found.'));
        }

        const token = await GoogleService.instance.exchangeToken(code);

        if (!token) {
            return res.status(403).json({
                status: false,
                message: 'Invalid code.',
                data: []
            });
        }

        token.expiration_date = new Date(
            Date.now() + token.expires_in * 1000
        ).toISOString();

        req.googleToken = token;

        return await signInWithGoogleToken(req, res, next);

    } catch (err) {
        return next(new ApplicationError(err.message, err.response?.status || 500));
    }
};
