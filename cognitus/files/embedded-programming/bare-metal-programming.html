Bare-metal programming

Bare-metal programming means writing code that runs directly on hardware without an operating system. A classic, practical example is blinking an LED on a microcontroller—this is usually the “Hello World” of embedded systems.

🔧 Example Use Case: LED Blinking on a Microcontroller

💡 Scenario

You have a microcontroller (e.g., STM32, AVR, ESP32), and you want to:

  • Turn an LED ON
  • Wait
  • Turn it OFF
  • Repeat forever

No OS, no libraries—just direct register access.

🧠 Key Concept

Bare metal = directly controlling hardware registers

Instead of:


digitalWrite(LED_PIN, HIGH);

You do:


GPIO_REGISTER |= (1 << PIN);

🧾 Example: Simple Bare-Metal C (Generic GPIO)


#define GPIO_BASE   0x40020000
#define GPIO_MODER  (*(volatile unsigned int*)(GPIO_BASE + 0x00))
#define GPIO_ODR    (*(volatile unsigned int*)(GPIO_BASE + 0x14))

#define PIN         5

void delay() {
    for (volatile int i = 0; i < 100000; i++);
}

int main(void) {
    // Set PIN as output (01)
    GPIO_MODER &= ~(3 << (PIN * 2));   // Clear bits
    GPIO_MODER |=  (1 << (PIN * 2));   // Set as output

    while (1) {
        GPIO_ODR ^= (1 << PIN); // Toggle LED
        delay();
    }
}

⚙️ What’s Happening

1. Memory-Mapped Registers

  • Hardware registers live at fixed addresses (e.g., 0x40020000)
  • You access them like variables

2. volatile

  • Prevents compiler optimization
  • Ensures real hardware reads/writes

3. Bit Manipulation

  • |= → set bit
  • &= ~ → clear bit
  • ^= → toggle bit

🚀 Real-World Use Cases of Bare Metal

🏭 1. Industrial Control Systems

  • Control motors, sensors, PLC-like systems
  • Deterministic timing is critical

🚗 2. Automotive Systems

  • Engine control units (ECU)
  • Airbag systems (must respond instantly)

🛰️ 3. IoT / Embedded Devices

  • Smart locks, wearables
  • Low power + fast boot required

🧪 4. Bootloaders

  • First code that runs on a device
  • Initializes memory, clocks, peripherals

🎮 5. High-performance peripherals

  • Custom drivers (SPI, I2C, UART)
  • No OS overhead
