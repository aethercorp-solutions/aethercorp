practical AWS API Gateway tutorial

Here’s a clear, practical AWS API Gateway tutorial—from basics to a real working example—tailored for developers who want to build serverless APIs (REST or HTTP) 🚀

🧭 What is AWS API Gateway?

AWS API Gateway is a fully managed service that lets you:

  • Create REST APIs and HTTP APIs
  • Secure endpoints (API keys, IAM, JWT, Cognito)
  • Connect to AWS Lambda, EC2, or other backends
  • Handle throttling, caching, monitoring, and versioning

🆚 API Types (Choose First)

Type Best For Cost Features
HTTP API Simple, fast APIs Cheaper JWT auth, Lambda proxy
REST API Complex APIs 💲💲 API keys, usage plans, caching

👉 Recommendation: Start with HTTP API unless you need advanced REST features.

🛠 Step-by-Step: Create an API Gateway + Lambda (HTTP API)

1️⃣ Create a Lambda Function

  1. Open AWS Console → Lambda
  2. Click Create function
  3. Runtime: Node.js 18
  4. Function name: hello-api

export const handler = async () => {
    return {
        statusCode: 200,
        body: JSON.stringify({ message: "Hello from API Gateway!" }),
    };
};

2️⃣ Create an HTTP API

  1. Go to API Gateway
  2. Click Create API
  3. Choose HTTP API
  4. Select Add integration → Lambda
  5. Pick hello-api

3️⃣ Create a Route

  1. Method: GET
  2. Path: /hello

4️⃣ Deploy (Auto for HTTP APIs)

API Gateway automatically deploys. You’ll get a URL like:


https://abc123.execute-api.ap-southeast-1.amazonaws.com/hello

🎉 Open it in a browser → Hello from API Gateway!

🔐 Authentication Options

Auth Type Use Case
JWT (Cognito / Auth0) Mobile & web apps
IAM AWS-to-AWS
API Key Metered public APIs
Lambda Authorizer Custom logic

Common choice:

👉 Amazon Cognito + JWT

🔑 Example: Secure Route with JWT

  1. Create a Cognito User Pool
  2. In API Gateway:
    • Authorization → JWT
    • Issuer: Cognito user pool URL
    • Audience: App Client ID
  3. Attach JWT auth to /hello

📊 Monitoring & Logs

Enable logs:

  • Amazon CloudWatch
  • API Gateway → Logs
  • Lambda → Logs

You’ll see:

  • Request latency
  • Errors (4xx / 5xx)
  • Throttling

⚡ Performance & Cost Tips

✅ Prefer HTTP APIs

✅ Use Lambda proxy integration

✅ Enable throttling

✅ Avoid large payloads

✅ Use regional endpoints

💡 HTTP API is ~70% cheaper than REST API.
