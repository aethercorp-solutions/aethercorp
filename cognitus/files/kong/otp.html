OTP (One-Time Password)

OTP (One-Time Password) is typically implemented as a short-lived code generated on the server, sent to the user (SMS/email/app), and then verified within a limited time window.

Here’s a clean breakdown of how it works in real systems (including AWS Lambda / Node.js style setups like yours).

🔐 1. OTP Flow (High-Level)

Step 1: User requests OTP

  • User enters phone/email
  • Backend generates OTP
  • Backend stores OTP (temporarily)
  • OTP is sent via SMS/email

Step 2: User submits OTP

  • User enters received code
  • Backend verifies:
    • correct code
    • not expired
    • not already used

Step 3: Authentication success

  • Issue JWT / session token
  • OTP is invalidated

⚙️ 2. OTP Generation

Most systems use:

Option A: Random numeric OTP (most common)


const otp = Math.floor(100000 + Math.random() * 900000);

Example: 483920

Option B: Crypto-secure OTP (recommended)


import crypto from "crypto";

const otp = crypto.randomInt(100000, 999999);

🗄️ 3. OTP Storage (Very Important)

You must store:

  • OTP
  • Identifier (email/phone)
  • Expiry time
  • Attempt count (optional)

Example (MongoDB / DynamoDB style)


{
    identifier: "user@email.com",
    otp: "483920",
    expiresAt: Date.now() + 5 * 60 * 1000, // 5 minutes
    used: false
}

📤 4. Sending OTP

Email

  • SendGrid / SES

SMS

  • AWS SNS / Twilio

Example (AWS SNS):


await sns.publish({
    PhoneNumber: "+639xxxxxxxxx",
    Message: `Your OTP is ${otp}`
}).promise();

🧠 5. OTP Verification Logic


export const verifyOtp = async (identifier, inputOtp) => {
    const record = await OtpModel.findOne({ identifier });
    
    if (!record) return { success: false, message: "OTP not found" };
    
    if (record.used) return { success: false, message: "OTP already used" };
    
    if (Date.now() > record.expiresAt)
        return { success: false, message: "OTP expired" };
    
    if (record.otp !== inputOtp)
        return { success: false, message: "Invalid OTP" };
    
    record.used = true;
    await record.save();
    
    // Generate JWT after success
    const token = generateJWT(identifier);
    
    return { success: true, token };
};

⏱️ 6. Expiry Strategy (Critical)

Common practice:

  • OTP validity: 3–10 minutes
  • After expiry → reject automatically

You can enforce via:

  • DB timestamp check
  • Redis TTL (best option)

⚡ 7. Best Practice Architecture (AWS Lambda)

Since you're using serverless:

Recommended stack:

  • API Gateway
  • Lambda (Node.js)
  • DynamoDB or Redis (ElastiCache)
  • SNS (SMS) / SES (email)

Flow:


Client → API Gateway → Lambda (generate OTP)
                      → SNS/SES sends OTP
                      → Store in DynamoDB (TTL)

Client → API Gateway → Lambda (verify OTP)
                      → validate DynamoDB record
                      → issue JWT

🔥 8. Production Improvements

To make it secure:

✔ Rate limiting

  • max 3–5 OTP requests per minute

✔ Hash OTP before storing


const hashed = crypto.createHash("sha256").update(otp).digest("hex");

✔ Attempt limit

  • lock after 3 wrong attempts

✔ Device binding (optional)

  • tie OTP to device fingerprint

🧩 9. Common Variants

  • SMS OTP (most common)
  • Email OTP
  • Authenticator app (TOTP like Google Authenticator)
  • Magic link (OTP embedded in URL)
