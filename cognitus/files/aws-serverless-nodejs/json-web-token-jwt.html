JSON Web Token (JWT)

A JSON Web Token (JWT) is a compact, URL-safe way to securely transmit information between two parties—most commonly used for authentication and authorization in APIs.

🔐 What JWT is (simple idea)

Think of a JWT like a digitally signed ID card:

  • The server issues it after login
  • The client (mobile/web app) stores it
  • The client sends it back on every request
  • The server verifies it instead of asking for login again

🧱 Structure of a JWT

A JWT has 3 parts, separated by dots:


header.payload.signature

Example:


eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9
.
eyJ1c2VySWQiOjEyMywiZXhwIjoxNzAwMDAwMDAwfQ
.
SflKxwRJSMeKKF2QT4fwpMeJf36POk6yJV_adQssw5c

1. Header

Contains metadata:


{
    "alg": "HS256",
    "typ": "JWT"
}

2. Payload (the data)

Contains claims (information):


{
    "userId": 123,
    "email": "user@example.com",
    "exp": 1700000000
}

Types of claims:

  1. Registered: exp (expiry), iat, iss
  2. Public: shared standard fields
  3. Private: your custom data (e.g. userId)

3. Signature (security)

Created using:


HMACSHA256(
  base64UrlEncode(header) + "." + base64UrlEncode(payload),
  secret
)

This ensures:

  • Token is not tampered
  • Token is issued by trusted server

🔄 How JWT works (flow)

1. User logs in

  • Sends username/password to server

2. Server validates

  • If valid → creates JWT

3. Server returns JWT


{
    "token": "eyJhbGciOiJIUzI1NiIs..."
}

4. Client stores token

  • Local storage / Secure storage (React Native)

5. Client sends token in requests


Authorization: Bearer <token>

6. Server verifies token

  • If valid → allow access
  • If expired/invalid → reject

⚡ Why use JWT

  • Stateless (no session storage on server)
  • Fast for serverless (like AWS Lambda — relevant to your setup)
  • Works well with APIs and mobile apps
  • Scalable across microservices

⚠️ Important considerations

  • Do NOT store sensitive data in payload (it's only encoded, not encrypted)
  • Always use HTTPS
  • Use short expiration (exp)
  • Use refresh tokens for long sessions
  • Keep your secret key secure

🧠 In your context (AWS + React Native)

JWT is typically used with:

  • API Gateway + Lambda (custom authorizer or middleware)
  • Stored in React Native (SecureStore / Keychain)
  • Sent via Authorization header
