Production-style manual refresh token system

Alright—let’s build a clean, production-style manual refresh token system in an AWS serverless Node.js app. I’ll show you a practical structure, not just snippets, so you can drop it into a real project.

We’ll use:

  • AWS Lambda
  • Amazon API Gateway
  • Amazon DynamoDB

🧱 1. Architecture Overview

Endpoints:


POST /login
POST /refresh
POST /logout
GET  /protected

Flow:

  • /login → returns access + refresh token
  • /protected → requires access token
  • /refresh → rotates refresh token + returns new access token
  • /logout → invalidates refresh token

🗄️ 2. DynamoDB Table Design

Table: refresh_tokens

Field Type Notes
userId PK string
tokenId SK string (UUID)
hashedToken string store hashed version
expiresAt number TTL
createdAt number
device string optional

👉 Enable TTL on expiresAt

🔑 4. Token Utilities


// utils/tokens.js
const jwt = require("jsonwebtoken");
const crypto = require("crypto");
const { v4: uuidv4 } = require("uuid");

const ACCESS_EXPIRY = "15m";
const REFRESH_EXPIRY_DAYS = 7;

function generateAccessToken(userId) {
  return jwt.sign({ userId }, process.env.ACCESS_SECRET, {
    expiresIn: ACCESS_EXPIRY,
  });
}

function generateRefreshToken() {
  const rawToken = crypto.randomBytes(64).toString("hex");
  const tokenId = uuidv4();

  return { rawToken, tokenId };
}

function hashToken(token) {
  return crypto.createHash("sha256").update(token).digest("hex");
}

function getExpiryTimestamp(days) {
  return Math.floor(Date.now() / 1000) + days * 24 * 60 * 60;
}

module.exports = {
  generateAccessToken,
  generateRefreshToken,
  hashToken,
  getExpiryTimestamp,
};

🔁 6. Refresh Token Lambda (Rotation)


// handlers/refresh.js
const AWS = require("aws-sdk");
const db = new AWS.DynamoDB.DocumentClient();

const {
  generateAccessToken,
  generateRefreshToken,
  hashToken,
  getExpiryTimestamp,
} = require("../utils/tokens");

exports.handler = async (event) => {
  const { refreshToken } = JSON.parse(event.body);

  const [tokenId, rawToken] = refreshToken.split(".");
  const hashed = hashToken(rawToken);

  // 1. Find token
  const result = await db.query({
    TableName: "refresh_tokens",
    IndexName: "tokenId-index", // create GSI
    KeyConditionExpression: "tokenId = :tid",
    ExpressionAttributeValues: {
      ":tid": tokenId,
    },
  }).promise();

  const record = result.Items?.[0];

  if (!record || record.hashedToken !== hashed) {
    return { statusCode: 401, body: "Invalid token" };
  }

  // 2. DELETE old token (rotation)
  await db.delete({
    TableName: "refresh_tokens",
    Key: {
      userId: record.userId,
      tokenId: record.tokenId,
    },
  }).promise();

  // 3. ISSUE new tokens
  const accessToken = generateAccessToken(record.userId);
  const { rawToken: newRaw, tokenId: newId } = generateRefreshToken();

  await db.put({
    TableName: "refresh_tokens",
    Item: {
      userId: record.userId,
      tokenId: newId,
      hashedToken: hashToken(newRaw),
      expiresAt: getExpiryTimestamp(7),
      createdAt: Date.now(),
    },
  }).promise();

  return {
    statusCode: 200,
    body: JSON.stringify({
      accessToken,
      refreshToken: `${newId}.${newRaw}`,
    }),
  };
};

🛡️ 8. Protected Route (JWT Check)


// middleware/auth.js
const jwt = require("jsonwebtoken");

function verifyToken(event) {
  const token = event.headers.Authorization?.split(" ")[1];

  if (!token) throw new Error("No token");

  return jwt.verify(token, process.env.ACCESS_SECRET);
}

module.exports = { verifyToken };

🔥 9. Critical Security Upgrades (Don’t Skip)

✅ Token Rotation

Already implemented:

  • Old refresh token is deleted immediately

✅ Hash Refresh Tokens

You NEVER store raw tokens (like passwords)

✅ Use HTTP-only Cookies (Frontend)

Instead of returning refresh token in JSON:


Set-Cookie: refreshToken=xxx; HttpOnly; Secure; SameSite=Strict

✅ Add Device Tracking

Add:


device: event.headers["user-agent"]

✅ Detect Token Reuse Attack

If a deleted token is reused:

→ invalidate ALL sessions for user

⚠️ Common Mistakes

  • ❌ Storing refresh tokens in localStorage
  • ❌ Not rotating refresh tokens
  • ❌ Not hashing tokens
  • ❌ Using long-lived access tokens
  • ❌ Not setting TTL in Amazon DynamoDB

🧭 Final Flow Recap

  1. Login → issue tokens → store hashed refresh token
  2. Access API → verify JWT
  3. Expired → call /refresh
  4. Refresh → rotate token + issue new pair
  5. Logout → delete refresh token
