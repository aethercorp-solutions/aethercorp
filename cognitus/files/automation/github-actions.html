GitHub Actions

Here’s a clear, practical GitHub Actions tutorial, from zero to real-world usage. I’ll keep it structured and hands-on.

1. What is GitHub Actions?

GitHub Actions is GitHub’s built-in CI/CD system.

It lets you:

  • Automatically run tests
  • Build your app
  • Deploy to servers or cloud
  • Trigger workflows on events like:
    • push
    • pull_request
    • release
    • schedule (cron)

Everything is defined using YAML files inside your repo.

2. Basic Concepts (Very Important)

1️⃣ Workflow

  • An automated process
  • Defined in .github/workflows/*.yml

2️⃣ Event

  • What triggers the workflow

    Examples:

    
on: push
on: pull_request
on: [push, pull_request]

3️⃣ Job

  • A set of steps that run on a runner
  • Jobs can run in parallel or sequentially

4️⃣ Step

  • Individual commands or actions

5️⃣ Runner

  • The machine running the job

    Examples:

  • ubuntu-latest
  • macos-latest
  • windows-latest

3. Folder Structure

Inside your repo:


.github/
 └── workflows/
      └── ci.yml

⚠️ The folder must be named .github/workflows

4. Your First GitHub Action (Hello World)

Create .github/workflows/hello.yml


name: Hello World Workflow

on:
  push:
    branches: [ main ]

jobs:
  hello:
    runs-on: ubuntu-latest

    steps:
      - name: Print message
        run: echo "Hello from GitHub Actions!"

What happens?

  • Every time you push to main
  • GitHub runs this workflow
  • You’ll see logs in GitHub → Actions tab

5. Using Actions (Prebuilt Steps)

GitHub provides reusable actions.

Checkout your repository


- name: Checkout code
  uses: actions/checkout@v4

This is required for almost every workflow.

6. Node.js Project Example (CI)

Install dependencies + run tests


name: Node CI

on:
  push:
    branches: [ main ]
  pull_request:

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20

      - name: Install dependencies
        run: npm install

      - name: Run tests
        run: npm test

✔ Common for React, Node, Next.js, Lambda

7. Environment Variables & Secrets

GitHub Secrets

Go to:


Repo → Settings → Secrets and variables → Actions

Add:


AWS_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY

Use secrets in workflow


env:
  AWS_ACCESS_KEY_ID: ${{ secrets.AWS_ACCESS_KEY_ID }}
  AWS_SECRET_ACCESS_KEY: ${{ secrets.AWS_SECRET_ACCESS_KEY }}

⚠️ Secrets are masked in logs.

8. Deploy Example (AWS / Serverless)


name: Deploy API

on:
  push:
    branches: [ main ]

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - uses: actions/checkout@v4

      - uses: actions/setup-node@v4
        with:
          node-version: 20

      - run: npm install -g serverless
      - run: npm install

      - name: Deploy
        run: serverless deploy
        env:
          AWS_ACCESS_KEY_ID: ${{ secrets.AWS_ACCESS_KEY_ID }}
          AWS_SECRET_ACCESS_KEY: ${{ secrets.AWS_SECRET_ACCESS_KEY }}

Perfect for:

  • AWS Lambda
  • API Gateway
  • Serverless Framework

9. Multiple Jobs & Dependencies


jobs:
  test:
    runs-on: ubuntu-latest
    steps:
      - run: echo "Running tests"

  deploy:
    needs: test
    runs-on: ubuntu-latest
    steps:
      - run: echo "Deploying..."

✔ deploy runs only if test succeeds

10. Matrix Builds (Multiple Versions)


strategy:
  matrix:
    node: [18, 20]

steps:
  - uses: actions/setup-node@v4
    with:
      node-version: ${{ matrix.node }}

✔ Test on multiple Node versions automatically

11. Cron (Scheduled Jobs)


on:
  schedule:
    - cron: "0 0 * * *"

Runs daily at midnight UTC

12. Debugging Tips

  • Expand step logs
  • Add debug prints:
