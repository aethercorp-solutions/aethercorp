Automating CI/CD for a React (Vite) app using GitHub Actions, AWS S3, and CloudFront

Here’s a complete guide to automating CI/CD for a React (Vite) app using GitHub Actions, AWS S3, and CloudFront 🚀

This setup will:

  • ✅ Build your Vite React app on every push to main
  • ✅ Deploy to S3 automatically
  • ✅ Invalidate CloudFront cache
  • ✅ Serve your app globally via CDN

🏗 Architecture Overview


GitHub Push → GitHub Actions → Build Vite App → Upload to S3 → Invalidate CloudFront → Live

Services used:

  • GitHub
  • GitHub Actions
  • Amazon S3
  • Amazon CloudFront

1️⃣ Prerequisites

✅ 1. Create S3 Bucket

  • Go to AWS → S3
  • Create bucket (example: my-react-app-prod)
  • Disable block public access (if static hosting)
  • Enable Static Website Hosting
  • Set index document:

index.html

✅ 2. Create CloudFront Distribution

  • Origin → Your S3 bucket
  • Default root object: 
    
index.html
  • Enable:
    • Redirect HTTP → HTTPS
  • Copy:
    • Distribution ID
    • CloudFront URL

✅ 3. Create IAM User for GitHub

Create an IAM user with programmatic access.

Attach policy:


{
    "Version": "2012-10-17",
    "Statement": [
        {
        "Effect": "Allow",
        "Action": [
            "s3:*"
        ],
        "Resource": "arn:aws:s3:::my-react-app-prod/*"
        },
        {
        "Effect": "Allow",
        "Action": [
            "cloudfront:CreateInvalidation"
        ],
        "Resource": "*"
        }
    ]
}

Save:

  • AWS_ACCESS_KEY_ID
  • AWS_SECRET_ACCESS_KEY

2️⃣ Add GitHub Secrets

Go to:


Repo → Settings → Secrets and Variables → Actions

Add:


AWS_ACCESS_KEY_ID
AWS_SECRET_ACCESS_KEY
AWS_REGION
S3_BUCKET
CLOUDFRONT_DISTRIBUTION_ID

3️⃣ Create GitHub Actions Workflow

Create:


.github/workflows/deploy.yml

📄 deploy.yml


name: Deploy React Vite to AWS

on:
  push:
    branches:
      - main

jobs:
  deploy:
    runs-on: ubuntu-latest

    steps:
      - name: Checkout code
        uses: actions/checkout@v3

      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: 18

      - name: Install dependencies
        run: npm install

      - name: Build project
        run: npm run build

      - name: Configure AWS credentials
        uses: aws-actions/configure-aws-credentials@v2
        with:
          aws-access-key-id: ${{ secrets.AWS_ACCESS_KEY_ID }}
          aws-secret-access-key: ${{ secrets.AWS_SECRET_ACCESS_KEY }}
          aws-region: ${{ secrets.AWS_REGION }}

      - name: Deploy to S3
        run: |
          aws s3 sync dist/ s3://${{ secrets.S3_BUCKET }} --delete

      - name: Invalidate CloudFront cache
        run: |
          aws cloudfront create-invalidation \
            --distribution-id ${{ secrets.CLOUDFRONT_DISTRIBUTION_ID }} \
            --paths "/*"

4️⃣ Update Vite Config (Important)

If using React Router:

vite.config.js


export default defineConfig({
    base: "/",
});

And in S3 + CloudFront:

  • Add custom error response:
    • 404 → /index.html
    • 403 → /index.html

This prevents SPA routing issues.

5️⃣ Optional Improvements (Production-Ready Setup)

🔹 Use Cache Control Headers

Improve performance:


aws s3 sync dist/ s3://bucket-name \
  --delete \
  --cache-control "public, max-age=31536000, immutable"

For index.html:


aws s3 cp dist/index.html s3://bucket-name \
  --cache-control "no-cache"

🔹 Use Separate Dev & Prod Deployments

Create:

  • develop branch → deploy to dev S3
  • main branch → deploy to prod S3

🔹 Add Build Caching (Faster CI)


- uses: actions/cache@v3
  with:
    path: ~/.npm
    key: ${{ runner.os }}-node-${{ hashFiles('**/package-lock.json') }}

Instead of IAM user keys:

Use OIDC with GitHub → AWS role assumption (more secure, no stored secrets).
