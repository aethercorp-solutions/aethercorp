Bitwise operators

Bitwise operators are one of those things that feel low-level… until you realize they solve very real, practical problems efficiently—especially in systems, performance-critical code, and even app development.

Here are the most useful real-world use cases (with examples):

🔹 1. Flags / Permissions (VERY common)

Instead of multiple booleans, you store everything in a single number.

Example


const READ = 1 << 0;   // 0001
const WRITE = 1 << 1;  // 0010
const EXEC = 1 << 2;   // 0100

let permission = READ | WRITE; // 0011

Check permission


if (permission & READ) {
    console.log("Can read");
}

Why this matters

  • Saves memory
  • Fast checks
  • Used in:
    • File systems
    • Role-based access
    • Feature toggles

🔹 2. Toggling States (On/Off switches)

Great for UI or feature flags.


let darkMode = 0;

// Turn ON
darkMode |= 1;

// Turn OFF
darkMode &= ~1;

// Toggle
darkMode ^= 1;

🔹 3. Efficient Storage (Bit Packing)

Store multiple values in a single integer.

Example (RGB color)


let color = (red << 16) | (green << 8) | blue;

Extract values


let red   = (color >> 16) & 0xFF;
let green = (color >> 8) & 0xFF;
let blue  = color & 0xFF;

Used in:

  • Graphics
  • Game dev
  • Image processing

🔹 4. Fast Math Operations

Bitwise ops are faster than normal math in some cases.

Multiply / Divide by 2


x << 1  // x * 2
x >> 1  // x / 2

🔹 5. Checking Even / Odd


if (x & 1) {
    console.log("Odd");
} else {
    console.log("Even");
}

🔹 6. Swapping Values (without temp variable)


a ^= b;
b ^= a;
a ^= b;

(Not commonly used now, but classic interview trick)

🔹 7. Clearing / Masking Bits

Used when you only want certain bits.


let value = 0b1101;

// Clear 2nd bit
value &= ~(1 << 1);

🔹 8. Hashing / Encryption / Low-level Systems

Bitwise ops are heavily used in:

  • Cryptography
  • Hash functions
  • Compression algorithms

🔹 9. Performance-sensitive code

In:

  • Game engines
  • Embedded systems
  • Networking protocols

🔹 10. Working with Binary Protocols

Example: parsing headers in networking


let version = (packet >> 4) & 0xF;
let type = packet & 0xF;

🔥 When YOU should use it (practical advice)

Use bitwise operators when:

  • You need compact state storage (flags)
  • You want high performance
  • You're dealing with binary data / protocols
  • You're building low-level or system-heavy logic

Avoid when:

  • Readability is more important
  • Team is not familiar with bitwise logic

💡 Real-world example (your context – backend / mobile)

For your scheduling app:

You could store schedule options like:


const DAILY = 1 << 0;
const WEEKLY = 1 << 1;
const MONTHLY = 1 << 2;

let scheduleType = DAILY | WEEKLY;

Instead of:


{ daily: true, weekly: true, monthly: false }
