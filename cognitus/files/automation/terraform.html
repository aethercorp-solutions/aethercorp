Terraform

Terraform is an Infrastructure as Code (IaC) tool created by HashiCorp that lets you define, provision, and manage cloud infrastructure using code instead of manual configuration. ☁️

It is widely used to automate infrastructure across platforms like Amazon Web Services, Microsoft Azure, and Google Cloud.

Core Idea

Instead of manually creating servers, databases, networks, etc., you write configuration files describing the infrastructure. Terraform then builds and manages it automatically.

Example:


provider "aws" {
    region = "ap-southeast-1"
}

resource "aws_instance" "app_server" {
    ami           = "ami-0abcdef12345"
    instance_type = "t3.micro"
}

This code tells Terraform to create an AWS EC2 server.

How Terraform Works

Terraform follows a simple workflow:

1️⃣ Write

You write infrastructure configuration using HCL (HashiCorp Configuration Language).

2️⃣ Plan


terraform plan

Shows what Terraform will create, modify, or delete.

3️⃣ Apply


terraform apply

Terraform creates the infrastructure.

4️⃣ Destroy (optional)


terraform destroy

Removes everything defined in the config.

Key Concepts

Providers

Plugins that allow Terraform to interact with services.

Examples:

  • AWS
  • Azure
  • Google Cloud
  • Kubernetes
  • GitHub

Example:


provider "aws" {
    region = "ap-southeast-1"
}

Resources

Actual infrastructure objects.

Examples:

  • EC2
  • VPC
  • S3
  • Load Balancer

Example:


resource "aws_s3_bucket" "my_bucket" {
    bucket = "my-app-storage"
}

State File

Terraform stores infrastructure state in:


terraform.tfstate

This file tracks what resources exist so Terraform knows what to change.

In teams, it’s stored remotely (like S3 + DynamoDB locking).

Modules

Reusable Terraform configurations.

Example structure:


modules/
   vpc/
   ec2/
   database/

Modules allow infrastructure reuse across projects.

Terraform vs Other Tools

Tool Type

Ansible Configuration management

Simple Terraform Commands


terraform init     # initialize project
terraform plan     # preview changes
terraform apply    # create/update infra
terraform destroy  # delete infra
