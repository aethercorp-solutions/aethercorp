Parsing HTML

Parsing HTML means reading an HTML document and turning it into a structured, usable representation so your program can find, extract, modify, or analyze elements like tags, text, attributes, and links.

Below is a basic but detailed explanation, starting from concepts and moving to practical examples.

1. What “Parsing HTML” Really Means

HTML is just text:


<html>
    <body>
        <h1>Hello</h1>
        <p class="intro">Welcome</p>
    </body>
</html>

When you parse HTML, you convert this text into a tree structure called the DOM (Document Object Model).

DOM Tree View


Document
 └── html
     └── body
         ├── h1 → "Hello"
         └── p (class="intro") → "Welcome"

Once parsed, your code can:

  • Find elements (h1, p)
  • Read attributes (class)
  • Extract text
  • Modify or remove nodes

2. Why You Parse HTML

Common use cases:

  • 🕷 Web scraping
  • 📄 Reading HTML emails
  • 🧠 Chatbots analyzing content
  • 🔄 Converting HTML → text / JSON
  • ✏️ Editing HTML programmatically
  • 📱 Rendering or sanitizing user content

3. Key Concepts You Must Know

a. Tags


<p>Hello</p>
  • Opening tag: <p>
  • Closing tag: </p>

b. Attributes


<img src="logo.png" alt="Logo">
  • src, alt are attributes

c. Elements

Everything combined:


<p class="text">Hello</p>

d. Text Nodes


<p>Hello</p>

Hello is a text node inside

4. Parsing vs Regex (Important)

❌ Do NOT parse HTML with regex

Why?

  • HTML can be nested
  • Attributes can be in any order
  • Invalid HTML still works in browsers

✅ Always use an HTML parser

5. Basic HTML Parsing in JavaScript (Browser)

Using DOMParser


const html = `
  <div>
    <h1>Hello</h1>
    <p class="intro">Welcome</p>
  </div>
`;

const parser = new DOMParser();
const doc = parser.parseFromString(html, "text/html");

// Find elements
const title = doc.querySelector("h1").textContent;
const intro = doc.querySelector(".intro").textContent;

console.log(title); // Hello
console.log(intro); // Welcome

What Happened

  1. HTML string → DOM document
  2. querySelector works like CSS
  3. Extract text easily

6. Parsing HTML in Node.js (Server-side)

Browsers have DOM built-in. Node.js does not.

Option 1: jsdom


npm install jsdom
            

import { JSDOM } from "jsdom";

const html = `
  <div>
    <h1>Hello</h1>
    <a href="/home">Home</a>
  </div>
`;

const dom = new JSDOM(html);
const document = dom.window.document;

const link = document.querySelector("a");
console.log(link.textContent);  // Home
console.log(link.href);         // http://localhost/home

Option 2: cheerio (Fast & Lightweight)


npm install cheerio
            

import cheerio from "cheerio";

const html = `
  <ul>
    <li>Apple</li>
    <li>Banana</li>
  </ul>
`;

const $ = cheerio.load(html);

$("li").each((i, el) => {
  console.log($(el).text());
});

📌 Cheerio is ideal for scraping (no JS execution).

7. Parsing HTML in Python (Bonus)

Using BeautifulSoup


pip install beautifulsoup4
            

from bs4 import BeautifulSoup

html = """
<div>
    <h1>Hello</h1>
    <p class="intro">Welcome</p>
</div>
"""

soup = BeautifulSoup(html, "html.parser")

print(soup.h1.text)
print(soup.select_one(".intro").text)

element.textContent = "Updated"

element.remove()

9. Handling Broken or Real-World HTML

HTML in the wild is often:

  • Missing closing tags
  • Poorly nested
  • Auto-corrected by browsers

HTML parsers:

  • Fix structure automatically
  • Normalize tags
  • Ignore invalid markup

This is why parsing is safer than string manipulation.

10. Best Practices

✅ Use a real parser

✅ Use CSS selectors

✅ Sanitize untrusted HTML

❌ Never use regex

❌ Don’t assume perfect structure
