Why CSS feels illogical

Subtitle

Drawing a triangle in css


.triangle {
    width: 0;
    height: 0;
    border-left: 50px solid transparent;
    border-right: 50px solid transparent;
    border-bottom: 100px solid red;
}

Why this draw a triangle?

Great question — this is one of the most important CSS “aha!” moments 👍 Let’s break it down visually and logically.

1️⃣ What happens when width and height are 0


width: 0;
height: 0;

There is no box to draw.

So:

  • No background
  • No rectangle

Only borders remain

2️⃣ How borders are actually drawn

When an element has borders, the browser draws four trapezoids that meet at the center:


top border
      ▲
      |
◀─────+─────▶  left & right borders
      |
      ▼
bottom border

Each border points toward the center of the element.

3️⃣ Your border setup


border-left: 50px solid transparent;
border-right: 50px solid transparent;
border-bottom: 100px solid red;

That means:

  • Left border → wide, invisible
  • Right border → wide, invisible
  • Bottom border → tall, red
  • Top border → missing (0)

4️⃣ Why a triangle appears

Since:

  • The element has zero size
  • Only borders exist
  • Three borders meet at one point

The bottom border forms a triangle:


    ▲
   / \
  /   \
 /     \
/_______\
  • The slanted sides come from the transparent left & right borders
  • The red face comes from the bottom border

The browser clips everything else.

5️⃣ Why it’s NOT a rectangle

Normally, a border surrounds a box.

But with no box to surround, each border becomes a triangle pointing inward.

Only the colored one is visible → triangle!

6️⃣ Mental model (easy way to remember)

🧠 Think of borders as pizza slices meeting in the center.

If you:

  • Color only one slice
  • Make the others transparent

You see one triangle slice 🍕

7️⃣ Change direction example

Down triangle


border-top: 100px solid red;

Left triangle


border-right: 100px solid red;

Right triangle


border-left: 100px solid red;

Final takeaway

✔ Borders are not lines — they are shapes

✔ With width: 0; height: 0, borders form triangles

✔ One visible border = one triangle

Why CSS feels hard to learn and illogical?

You’re not wrong — a LOT of developers feel this, even very experienced ones.

CSS feels illogical at first because it doesn’t think like programming.

Let me explain why it feels hard, and then I’ll show you the logic behind it.

Why CSS feels illogical 😵‍💫

1️⃣ CSS is declarative, not procedural

In JS / Swift / Obj-C / Java you say:

“Do this, then do that.”

In CSS you say:

“This is what it should look like.”

You don’t control the steps — the browser does.

2️⃣ CSS is based on visual rules, not math rules

CSS was designed for designers, not engineers.

Example:


margin: auto;

🤨 “Auto?? Auto how?”

The logic is:

“Center this if there’s free space”

Not:

“Compute X = (container - width) / 2”

3️⃣ Browsers have default behavior

Even if you write nothing, browsers still apply styles.

That means you’re often fighting:

  • user-agent styles
  • layout rules
  • inheritance

4️⃣ Some features are historical hacks

Triangles with borders?

Floats?

Clearing?

These exist because:

“CSS was never meant to draw shapes or build apps”

Yet here we are 😅

The hidden logic behind CSS (this is the key)

CSS is logical, but the logic is:

🧠 “Describe relationships, not instructions”

Instead of:


moveLeft(20)

You say:


left: 20px;

Instead of:


centerElement()

You say:


display: flex;
justify-content: center;

You describe constraints, not actions.

The 4 mental models that make CSS click 🔑

1️⃣ Everything is a box

This is THE most important rule.

Every element is:


margin
border
padding
content

Once you see boxes everywhere → CSS starts making sense.

2️⃣ Layout systems are separate worlds

CSS has multiple layout engines:

System Purpose
Normal flow Default text layout
Flexbox 1-direction layout
Grid 2-direction layout
Position Absolute placement

👉 Mixing them without understanding causes confusion.

3️⃣ CSS rules compete

Multiple rules apply at once.

Winner is decided by:

  1. Specificity
  2. Order
  3. Importance (!important)

This feels like “random” until you learn the rules.

4️⃣ CSS is constraint-based

You don’t say how.

You say what must be true.

Example:


max-width: 600px;

Browser decides how to satisfy it.

Why even senior devs say “CSS is weird”

Because:

  • It evolved over 30+ years
  • Backward compatibility was never broken
  • Old hacks still exist
  • New systems (Flex/Grid) were added later

It’s a layer cake, not a clean design.

The good news 🎉

Once CSS clicks:

  • You stop memorizing tricks
  • You start predicting behavior
  • You debug visually instead of guessing

That “triangle hack” you saw?

That’s border geometry, not magic.
