XML (eXtensible Markup Language)

XML (eXtensible Markup Language) is a markup language used to store and transport data in a structured, human-readable, and machine-readable format. It's widely used in various applications where data needs to be shared between different systems, especially over the internet.

🔧 Basic Usage of XML

XML uses tags to define elements, similar to HTML, but unlike HTML, XML is descriptive, not presentation-based.

📄 Example:


<book>
  <title>Introduction to XML</title>
  <author>John Doe</author>
  <year>2025</year>
</book>

🛠️ Common Use Cases

  1. Data storage – Used to save structured data (e.g., config files).
  2. Data transport – Used in web services (e.g., SOAP APIs).
  3. Configuration files – Used in applications like Java (.xml config files).
  4. RSS/ATOM feeds – News aggregators use XML to deliver content.
  5. Office documents – Formats like .docx and .xlsx are ZIP files containing XML.
  6. Serialization – Saving objects in XML format in programming (e.g., C#, Java).

📚 Key Features of XML

  1. Hierarchical structure (tree-like).
  2. User-defined tags (customizable structure).
  3. Cross-platform and language-independent.
  4. Supports attributes and nested elements.

🔍 Example with attributes:


<book isbn="978-3-16-148410-0">
  <title>Advanced XML</title>
  <author>Jane Smith</author>
</book>

⚙️ How XML Is Used in Programming

Depending on the language you're using, XML can be parsed and manipulated with built-in libraries.

✅ Python


import xml.etree.ElementTree as ET

tree = ET.parse('data.xml')
root = tree.getroot()
print(root.tag)

✅ Java


DocumentBuilderFactory factory = DocumentBuilderFactory.newInstance();
DocumentBuilder builder = factory.newDocumentBuilder();
Document doc = builder.parse(new File("data.xml"));

✅ Pros

  • Self-descriptive structure
  • Human-readable
  • Widely supported

❌ Cons

  • Verbose (more characters than JSON)
  • Parsing can be slower than more modern formats

🔄 Alternatives

  • JSON – More compact and faster for web APIs
  • YAML – Human-friendly configuration files
  • CSV – Simple tabular data
