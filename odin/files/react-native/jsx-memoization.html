JSX memoization

JSX memoization is about avoiding unnecessary re-renders of JSX output in a React (or React Native) component. Instead of recomputing the same UI every render, you cache it and reuse it when inputs haven’t changed.

🔹 Why it matters

In React, every state or prop change triggers a re-render. That means:

  • JSX gets recalculated
  • Child components may re-render
  • Performance can degrade (especially in large trees or lists)

Memoization helps you skip work when nothing actually changed.

🔹 1. Memoizing JSX with useMemo

useMemo caches a computed value so it doesn’t recalculate on every render.

✅ Example: Expensive Calculation


import React, { useState, useMemo } from 'react';

function App() {
  const [count, setCount] = useState(0);
  const [text, setText] = useState('');

  // Expensive function
  const expensiveCalculation = (num) => {
    console.log('Calculating...');
    for (let i = 0; i < 1000000000; i++) {} // simulate heavy work
    return num * 2;
  };

  // Memoized value
  const result = useMemo(() => {
    return expensiveCalculation(count);
  }, [count]); // only runs when `count` changes

  return (
    <div>
      <h2>Result: {result}</h2>

      <button onClick={() => setCount(count + 1)}>
        Increment Count
      </button>

      <input
        value={text}
        onChange={(e) => setText(e.target.value)}
        placeholder="Type something"
      />
    </div>
  );
}

export default App;

🔍 What’s happening?

  • When you click Increment Count → calculation runs ✅
  • When you type in input → calculation does NOT run ❌
  • Without useMemo, it would run every render

🧠 Key takeaway

Use useMemo when:

  • A calculation is expensive
  • You want to avoid unnecessary recomputation

🔹 2. Memoizing Components with React.memo

Instead of memoizing JSX manually, you can memoize the entire component.


import React from 'react';
import { Text } from 'react-native';

const Item = React.memo(({ name }) => {
  console.log('Rendering Item:', name);
  return <Text>{name}</Text>;
});

Usage:

✅ What happens

  • Component only re-renders if props change
  • Prevents unnecessary updates automatically

🔹 3. When to use JSX memoization

Use it when:

  • Rendering large lists
  • Expensive computations inside JSX
  • Components re-render frequently with same props
  • Performance optimization in React Native (important for smooth UI)

🔹 When NOT to use it

Don’t overuse it:

  • Small/simple components → no benefit
  • Cheap calculations → adds unnecessary complexity
  • Premature optimization

Memoization itself has a cost.

🔹 Common mistake


const list = useMemo(() => items.map(...), []);

❌ Wrong: dependency array is empty

👉 This freezes the value forever (stale UI)

🔹 Practical mental model

  • useMemo → cache values / JSX
  • React.memo → cache components
  • useCallback → cache functions

🔹 Real-world React Native tip

If you're rendering a FlatList, prefer:

  • React.memo for item components
  • keyExtractor
  • getItemLayout (if possible)

This usually gives better performance than memoizing JSX manually.
