functional components

In React, functional components are the modern, preferred way to build UI using JSX (JavaScript XML). They’re just plain JavaScript functions that return JSX, which describes what the UI should look like.

🔹 What is a Functional Component?

A functional component is simply a function that returns JSX:


function MyComponent() {
    return <h1>Hello, world!</h1>
}

Or using arrow function syntax:


const MyComponent = () => {
    return <h1>Hello, world!</h1>
};

🔹 What is JSX?

JSX is a syntax that looks like HTML but is actually JavaScript:


const element = <h1>Hello, JSX!</h1>

Under the hood, React converts JSX into React.createElement(...).

🔹 Functional Component with Props

Props let you pass data into components:


const Greeting = ({ name }) => {
    return <h1>Hello, {name}!</h1>
};
    
// Usage
<Greeting name="Paul" />

🔹 Using Hooks in Functional Components

Functional components can manage state and side effects using hooks like useState and useEffect.

Example with state:


import React, { useState } from "react";

const Counter = () => {
  const [count, setCount] = useState(0);

  return (
    <div>
      <p>Count: {count}</p>
      <button onClick={() => setCount(count + 1)}>
        Increment
      </button>
    </div>
  );
};

🔹 JSX Rules to Remember

1. Single parent element

2. Use className instead of class


<div className="container"></div>

3. JavaScript inside {}


const name = "Paul";
<h1>Hello, {name}</h1>

🔹 Best Practices

  • Use arrow functions for cleaner syntax
  • Use React.memo for performance optimization
  • Keep components small and reusable
  • Use custom hooks for logic reuse
