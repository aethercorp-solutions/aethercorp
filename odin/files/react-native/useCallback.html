useCallback

useCallback is a React Hook that lets you memoize a function, meaning React will remember the same function instance between renders instead of creating a new one every time.

It is mainly used to optimize performance, especially when passing functions to child components.

🔹 Simple definition

useCallback = “keep the same function unless its dependencies change”

🔹 Basic syntax


const memoizedFunction = useCallback(() => {
    // your logic here
}, [dependencies]);

🔹 Why it matters

In React, every re-render creates new functions:


const handleClick = () => {
    console.log("clicked");
};

Even if the logic is the same, React sees it as a new function every render.

This can cause:

  • unnecessary re-renders in child components
  • performance issues in large apps

🔹 Example with useCallback


import React, { useState, useCallback } from "react";

function Parent() {
  const [count, setCount] = useState(0);

  const handleClick = useCallback(() => {
    console.log("Button clicked");
  }, []);

  return (
    <>
      <button onClick={() => setCount(count + 1)}>
        Increment {count}
      </button>
      <Child onClick={handleClick} />
    </>
  );
}

const Child = React.memo(({ onClick }) => {
  console.log("Child rendered");
  return <button onClick={onClick}>Child Button</button>;
});

🔹 What’s happening here

  • useCallback keeps handleClick the same between renders
  • React.memo prevents Child from re-rendering unnecessarily
  • Result: better performance

🔹 When to use useCallback

Use it when:

  • passing functions to memoized child components
  • optimizing expensive re-renders
  • working with dependency-heavy components

Don’t use it everywhere—unnecessary usage can add complexity without benefit.

🔹 Related React concept

React uses useCallback together with React.memo and useMemo for performance optimization in functional components.

🔹 One-line summary

useCallback is a React Hook that prevents a function from being recreated on every render unless its dependencies change.
