React Hook

A React Hook is a special function in React that lets you “hook into” React features—like state, lifecycle behavior, context—inside function components.

Before hooks existed, only class components could manage things like state and lifecycle methods. Hooks changed that, allowing function components to do the same things in a cleaner way.

🔹 Simple idea

A React Hook lets you:

  • store and update state
  • run side effects (like API calls)
  • share logic between components

🔹 Common React Hooks

1. useState – state management

Lets you store data inside a component.


import { useState } from "react";

function Counter() {
  const [count, setCount] = useState(0);

  return (
    <button onClick={() => setCount(count + 1)}>
      Count: {count}
    </button>
  );
}

2. useEffect – side effects

Runs code after rendering (e.g. API calls, subscriptions).


import { useEffect } from "react";

useEffect(() => {
  console.log("Component mounted or updated");
}, []);

3. useMemo – memoizing values

Optimizes performance by caching computed values.

4. useCallback – memoizing functions

Prevents functions from being recreated unnecessarily.

🔹 Why Hooks are useful

  • Cleaner and shorter code than classes
  • Easier to reuse logic
  • Better separation of concerns
  • More readable functional components

🔹 React (context)

React is the library that introduced Hooks in version 16.8 to improve functional component capabilities.

🔹 In one sentence

A React Hook is a function that allows functional components to use React features like state and lifecycle behavior.
