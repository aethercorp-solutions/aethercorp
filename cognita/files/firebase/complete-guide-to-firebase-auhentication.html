1. What is Firebase Authentication?

Firebase Authentication (Firebase Auth) is a Google-provided service that lets you:

  • Identify who the user is
  • Secure your app without building auth from scratch
  • Support multiple sign-in methods easily

It handles:

  • User sign-up
  • Login
  • Session management
  • Token creation
  • Password reset
  • Secure identity verification

üëâ Firebase Auth does NOT store your app data, only identity info.

7. Email & Password Authentication (Basic)

Enable Email Auth

Firebase Console ‚Üí Authentication ‚Üí Sign-in Method ‚Üí Email/Password ‚Üí Enable

Register User

iOS (Swift)


Auth.auth().createUser(withEmail: email, password: password) { result, error in
    if let error = error {
        print(error.localizedDescription)
        return
    }
    print("User registered:", result?.user.uid ?? "")
}

Login User


Auth.auth().signIn(withEmail: email, password: password) { result, error in
    if let error = error {
        print(error.localizedDescription)
        return
    }
    print("Logged in:", result?.user.email ?? "")
}

Get Current User


if let user = Auth.auth().currentUser {
    print(user.uid)
}

Logout


try Auth.auth().signOut()

8. Authentication State (Auto Login)

Firebase automatically remembers users.


Auth.auth().addStateDidChangeListener { auth, user in
    if let user = user {
        print("User logged in:", user.uid)
    } else {
        print("User logged out")
    }
}

10. Role-Based Authentication (Admin / User)

Why Roles?

Firebase Auth does not support roles directly.

Solution:

  • Use Custom Claims
  • Or Firestore roles

Example Role Use Case

Role Access
admin Manage users, data
user Read own data

Set Role Using Admin SDK (Server Only)


admin.auth().setCustomUserClaims(uid, { role: 'admin' });

Access Role in App


Auth.auth().currentUser?.getIDTokenResult { result, error in
    let role = result?.claims["role"] as? String
    print("Role:", role ?? "none")
}

11. Firestore Security Rules with Auth


rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {

    match /users/{userId} {
      allow read: if request.auth != null &&
                   request.auth.uid == userId;

      allow write: if request.auth.token.role == "admin";
    }

    match /adminData/{doc} {
      allow read, write: if request.auth.token.role == "admin";
    }
  }
}
