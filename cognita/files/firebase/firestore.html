üî• What is Firestore?

Cloud Firestore is a flexible, scalable NoSQL database from Firebase (by Google), used to store and sync data for web, mobile, and server development.

üß± Key Features

  • NoSQL document-based database (data is stored in documents, which are grouped into collections).
  • Realtime updates (listen to changes in data).
  • Offline support (for mobile/web apps).
  • Strong client SDK support (JS, iOS, Android, Python, etc.).
  • Security rules to control access.

üìò Firestore Data Structure

Firestore has:

  • Collections ‚Üí A group of documents.
  • Documents ‚Üí Individual records with key-value pairs (can also contain subcollections).

üîÑ 1. Firestore: Documents & Collections (Structured, Modular)

üîπ Data Model:

  • Collection ‚Üí Document ‚Üí Fields/Subcollections
  • Each document is like a record (like a JSON object), and lives inside a collection.
  • Documents can contain subcollections, allowing for nested data without nesting JSON.

üîπ Example:


users (collection)
  ‚îî‚îÄ‚îÄ user123 (document)
        ‚îú‚îÄ‚îÄ name: "Alice"

              ‚îî‚îÄ‚îÄ order456 (document)
                    ‚îú‚îÄ‚îÄ total: 59.99
                    ‚îî‚îÄ‚îÄ items: [...]

‚úÖ Advantages:

  • Easier to scale and query specific pieces of data.
  • Encourages modular, relational-like design.
  • Documents are limited in size (1MB max) ‚Äî encourages flat, efficient structure.

üå≥ 2. Realtime Database: JSON Tree (Nested, Flat or Deep)

üîπ Data Model:

  • Entire database is a single JSON tree.
  • Nodes can be deeply nested, like a full JSON object.
  • Data is stored as raw JSON with arbitrary depth.

üîπ Example:


{
    "users": {
        "user123": {
        "name": "Alice",
        "email": "alice@example.com",
        "orders": {
            "order456": {
            "total": 59.99,
            "items": [...]
            }
        }
        }
    }
}

‚ö†Ô∏è Downsides:

  • Deep nesting makes it harder to fetch or update specific pieces of data.
  • Need to flatten data manually to avoid downloading too much.
  • Querying deeply nested data is limited or inefficient.

üß† Key Differences

Feature Firestore Realtime Database
Data structure Collections & Documents JSON Tree
Data nesting Flat, relational-style Deeply nested (or flat manually)
Fetching specific data Efficient and scoped Can be inefficient
Best practice Normalize with subcollections Flatten JSON for performance
Document size limit 1MB per document No fixed document unit
