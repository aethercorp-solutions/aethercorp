Apple Sign in Demo Account

Good question — this is a very common App Store rejection point.

If you're using Google Sign-In via Firebase, you can’t really create a traditional “username/password demo account” inside Google auth itself. Google controls authentication, not you.

So you need to add an alternative login path just for reviewers.

✅ Option 1 (Recommended): Add Email/Password Demo Account in Firebase

Use Firebase Authentication’s Email/Password provider alongside Google Sign-In.

🔧 Steps

1. Enable Email/Password Auth

  • Go to Firebase Console → Authentication → Sign-in method
  • Enable Email/Password

2. Create Demo User

You can create it:

Option A: Firebase Console (quickest)

  • Authentication → Users → “Add user”
  • Example: 
    
Email: reviewer@test.com
Password: Test123456

Option B: Programmatically (Node.js Admin SDK)


import admin from "firebase-admin";

await admin.auth().createUser({
  email: "reviewer@test.com",
  password: "Test123456",
});

3. React Native Login (Demo Path)

Use Firebase client SDK:


import auth from '@react-native-firebase/auth';

const signInDemo = async () => {
  try {
    const user = await auth().signInWithEmailAndPassword(
      "reviewer@test.com",
      "Test123456"
    );
    console.log("Logged in:", user.user.uid);
  } catch (e) {
    console.error(e);
  }
};

4. Add a Hidden or Visible Button

You can add:


<Button title="Demo Login (Reviewer)" onPress={signInDemo} />

OR hide it behind:

  • long press
  • debug menu
  • “Having trouble logging in?” link

5. App Store Review Notes (VERY IMPORTANT)

In App Store Connect, write something like:


To access the app:

Option 1:
Use demo account:
Email: reviewer@test.com
Password: Test123456

(No OTP required)

Option 2:
Use Google Sign-In

All features are accessible after login.

✅ Option 2: Use Google Account as Demo (Less Reliable)

You can provide a Google account:


Email: yourtest@gmail.com
Password: xxxx

BUT ⚠️:

  • Apple reviewers may not want to log into Google
  • May trigger 2FA / OTP
  • Often leads to rejection

👉 Not recommended as the only method

✅ Option 3: OTP Bypass (Advanced / Cleanest UX)

If your app requires OTP:

In your backend (Lambda / Firebase Functions):


if (email === "reviewer@test.com") {
    return { skipOtp: true };
}

Or:


const isReviewer = email.endsWith("@test.com");

🔥 Best Practice Architecture (What top apps do)

  • Google Sign-In (normal users)
  • Apple Sign-In (required by Apple)
  • Email/Password (for reviewers + fallback)
  • OTP (optional / production only)

🚫 What NOT to Do

  • ❌ Force OTP on all login methods
  • ❌ Only provide Google Sign-In
  • ❌ Require real phone number for testing
  • ❌ Forget review notes

💡 Pro Tip (From Experience)

Apple reviewers:

  • Prefer fastest login
  • Hate friction (OTP, captcha, etc.)

👉 If they can’t login in < 30 seconds → high chance of rejection
