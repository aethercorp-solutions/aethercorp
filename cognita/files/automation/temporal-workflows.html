üöÄ What Are Temporal Workflows?

Temporal workflows are code-based definitions of business logic that need to be executed reliably and with state. They‚Äôre written in normal programming languages (like Go, Java, TypeScript, Python) but run in a fault-tolerant, durable, and distributed environment managed by the Temporal platform.

üß† Key Concepts

1. Workflow

  • A function representing a business process.
  • Can run for seconds or years.
  • Automatically resumes after crashes, restarts, or failures.
  • Maintains state across events.

2. Activities

  • External, non-deterministic tasks executed outside the workflow.
  • These could be API calls, database operations, or anything side-effecting.
  • They are retriable and idempotent.

3. Temporal Server

  • Central orchestration service that manages workflows, scheduling, retries, task queues, etc.
  • Your code runs in a worker, while the server keeps state.

4. Worker

  • Hosts your application code (workflow and activities).
  • Polls the Temporal server for tasks and executes them.

üîÅ Features

  • Durability: Workflow state is stored durably and can survive process or server failures.
  • Scalability: Horizontally scalable workers and servers.
  • Retries & Error Handling: Built-in retries, timeouts, and custom retry logic.
  • Timers & Scheduling: Native support for delays, timeouts, cron jobs.
  • Signals & Queries: Receive external inputs or expose state during execution.
