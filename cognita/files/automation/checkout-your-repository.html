Checkout your repository

You’re asking what “Checkout your repository” means, specifically in a React / Next.js GitHub Actions workflow.

What does “Checkout your repository” mean?

In GitHub Actions, each job runs on a fresh virtual machine.

👉 That machine starts EMPTY

👉 Your code is NOT there by default

So this step:


- uses: actions/checkout@v4

downloads (clones) your GitHub repository into the runner, similar to:


git clone https://github.com/your/repo.git

Without this step:

  • npm install ❌ fails
  • npm run build ❌ fails
  • next build ❌ fails

    because there is no source code

Why it’s REQUIRED for React / Next.js

A Next.js app needs:

  • package.json
  • next.config.js
  • pages/ or app/
  • node_modules (installed after checkout)

All of that comes from your repo.

So this step is mandatory for:

  • CI
  • Builds
  • Deployments

Minimal Working Next.js Workflow (Correct)


name: Next.js CI

on:
  push:
    branches: [ main ]
  pull_request:

jobs:
  build:
    runs-on: ubuntu-latest

    steps:
      # 1️⃣ Get your code
      - name: Checkout repository
        uses: actions/checkout@v4

      # 2️⃣ Setup Node.js
      - name: Setup Node
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: 'npm'

      # 3️⃣ Install dependencies
      - name: Install dependencies
        run: npm install

      # 4️⃣ Build Next.js
      - name: Build
        run: npm run build

What happens step-by-step

1️⃣ Runner starts


/home/runner/work/

2️⃣ Checkout step


/home/runner/work/your-repo/your-repo/

Your files now exist:


package.json
next.config.js
app/
pages/

3️⃣ npm install works

4️⃣ next build works

What happens if you REMOVE checkout?


steps:
  - run: npm install

❌ Error:


npm ERR! enoent ENOENT: no such file or directory, package.json

This is one of the #1 GitHub Actions beginner mistakes.

Advanced: Checkout Options (Optional)

Full git history (for versioning)


- uses: actions/checkout@v4
  with:
    fetch-depth: 0

Specific branch


- uses: actions/checkout@v4
  with:
    ref: main

Private submodules


- uses: actions/checkout@v4
  with:
    submodules: true
