ROM Boot Loader

A ROM Boot Loader (also called a Bootstrap Loader) is a small piece of code stored in Read-Only Memory (ROM) that is executed when a computer or embedded system is powered on or reset. Its main job is to initialize the hardware and load the main operating system or firmware into memory so it can start running.

üîß Functions of a ROM Boot Loader

  1. Power-On Initialization
    • Initializes CPU, memory, and essential peripherals.
    • May run basic self-tests (like POST on PCs).
  2. Load Secondary Software
    • Loads a larger bootloader (like U-Boot, GRUB) or the OS kernel from:
      • Flash memory
      • EEPROM
      • SD card
      • Hard drive
      • Network (via TFTP, PXE, etc.)
  3. Transfer Control
    • Passes control to the loaded software (e.g., jumps to the OS entry point).

üß† Why it's in ROM

  • ROM is non-volatile, so it retains data without power.
  • Ensures that the system can always boot, even if storage is corrupted.

üîÑ Boot Sequence (Simplified)

  1. Power On
  2. CPU starts executing code at a fixed address in ROM
  3. ROM bootloader runs
  4. ROM bootloader loads 2nd stage bootloader or OS from storage
  5. Control is handed over

üîç Examples

  • ARM processors: Have a built-in ROM bootloader that loads code from flash or SD.
  • x86 PCs: The BIOS or UEFI firmware in ROM acts as the bootloader.
  • Microcontrollers (e.g., STM32): Include a ROM bootloader that supports UART, USB, or CAN boot.

‚ùì How does ROM retain data without power?

ROM stands for Read-Only Memory, and it's designed to retain data even when the power is turned off. This is because it's made using non-volatile memory technology, which stores data in a way that doesn't require constant electrical power.

üî¨ Types of ROM and How They Work

Here‚Äôs how various types of ROM keep data without power:

  1. Mask ROM
    • Data is permanently written during chip manufacturing.
    • Uses physical wiring or fuses to represent 1s and 0s.
    • Cannot be changed after production.
    • Non-volatility: Data is "hard-coded" in the silicon itself.
  2. PROM (Programmable ROM)
    • One-time programmable using special equipment.
    • Uses fuses or antifuses that are blown (or connected) to represent data.
    • Non-volatility: Once programmed, the changes are physical and permanent.
  3. EPROM (Erasable Programmable ROM)
    • Can be erased with ultraviolet (UV) light.
    • Uses floating-gate transistors to store data.
    • Non-volatility: The charge stored on the floating gate stays trapped even when power is removed.
  4. EEPROM / Flash Memory
    • Electronically erasable and reprogrammable.
    • Also uses floating-gate MOSFETs.
    • Non-volatility: The trapped electrons in the floating gate maintain their state without power.

‚ö° So, How Does It Actually Retain Data?

At the hardware level:

  • These memory types use charge trapping or physical changes (like fuses or wiring) to store bits.
  • Because the data is not stored in a temporary electrical state (like in RAM), no electricity is needed to "remember" the values.
