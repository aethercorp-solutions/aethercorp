WebSocket

WebSocket is a technology that lets a browser (or app) and a server talk to each other instantly, in both directions, over one persistent connection.

Let’s break it down simply, then go a bit deeper.

1. The basic idea (very simple)

Imagine a phone call instead of sending letters.

  • HTTP (normal web) → sending letters
    • Client asks: “Any update?”
    • Server replies: “No.”
    • Client asks again… and again…
  • WebSocket → phone call
    • Client and server stay connected
    • Either side can talk any time
    • Messages arrive instantly

    • That’s WebSocket.

2. Why HTTP is not enough

With normal HTTP:

  • The client always starts the conversation
  • The server cannot push data by itself
  • For real-time apps, clients must keep asking:
    • “New message?”
    • “New data?”
    • “New update?”

This is inefficient and slow.

3. What WebSocket fixes

WebSocket gives you:

✅ One long-lived connection

✅ Two-way communication

✅ Real-time data

✅ Low latency (very fast)

Once connected:

  • Client → Server can send data anytime
  • Server → Client can send data anytime

No repeated requests.

4. How WebSocket works (step by step)

Step 1: Starts as HTTP

The browser sends a normal HTTP request:


GET /chat HTTP/1.1
Upgrade: websocket

This says:

“Hey server, can we upgrade this connection to WebSocket?”

Step 2: Server agrees

Server responds:


HTTP/1.1 101 Switching Protocols
Upgrade: websocket

Now the connection is no longer HTTP.

Step 3: Persistent connection

  • The connection stays open
  • Data flows back and forth as messages, not requests

5. How data is sent

Instead of HTTP requests/responses:

  • Data is sent as frames
  • Frames are small and efficient
  • Messages can be:
    • Text (JSON, strings)
    • Binary (audio, video, game data)

Example:


{ "type": "message", "text": "Hello!" }

6. Simple real-world example

Chat app

Without WebSocket:

  • App asks server every second:
    • “Any new messages?”
  • Wasteful and slow

With WebSocket:

  • Server instantly pushes:
    • “New message arrived!”

7. Common use cases

WebSocket is used when you need live updates:

  • 💬 Chat applications
  • 📈 Live stock prices
  • 🎮 Multiplayer games
  • 🚕 Ride tracking (Uber-style)
  • 📊 Dashboards (real-time metrics)
  • 🔔 Notifications
  • 🧠 AI streaming responses

8. WebSocket vs HTTP vs SSE (simple table)

Feature HTTP WebSocket SSE
Persistent connection ❌ ✅ ✅
Server → Client push ❌ ✅ ✅
Client → Server anytime ❌ ✅ ❌
Real-time ❌ ✅ ⚠️ limited

9. WebSocket in React Native / Web (simple code)

Client


const socket = new WebSocket("wss://example.com/socket");

socket.onopen = () => {
  socket.send("Hello server");
};

socket.onmessage = (event) => {
  console.log("Received:", event.data);
};

Server (Node.js)


import WebSocket from "ws";

const wss = new WebSocket.Server({ port: 8080 });

wss.on("connection", (ws) => {
  ws.on("message", (msg) => {
    ws.send(`Echo: ${msg}`);
  });
});

10. Important things to know

⚠️ WebSocket connections:

  • Stay open → use memory
  • Need reconnection logic
  • Can drop on network changes
  • Need authentication handling

That’s why many apps combine:

  • HTTP → login, fetch data
  • WebSocket → live updates

11. One-sentence summary

WebSocket is a persistent, two-way communication channel that lets clients and servers exchange data instantly without repeatedly making requests.
