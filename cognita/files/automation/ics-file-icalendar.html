.ics file (also known as an iCalendar file)

A .ics file (also known as an iCalendar file) is a plain text file format used to store calendar information. It's commonly used to share event details between different calendar applications and systems.

üîπ What does a .ics file contain?

An .ics file can include details such as:

  • Event title (summary)
  • Start and end time/date
  • Location
  • Description or notes
  • Recurrence rules (e.g. repeating weekly)
  • Alarms or reminders
  • Organizer and attendee information

üîπ What is it used for?

  • Sending meeting invites via email
  • Subscribing to calendars (e.g., sports schedules, holidays)
  • Importing/exporting events between calendar applications

üîπ Common apps that use .ics files:

  • Google Calendar
  • Microsoft Outlook
  • Apple Calendar (iCal)
  • Mozilla Thunderbird with Lightning
  • Any app that supports the iCalendar standard

üîπ How to open or use an .ics file?

  • Double-click it ‚Äì it often opens in your default calendar app.
  • Import it manually ‚Äì within your calendar app (e.g., "File > Import").
  • Subscribe to a .ics URL ‚Äì for live calendar updates (e.g., syncing public calendars).

You can create a .ics (iCalendar) file in several ways, depending on how you want to use it: manually, through a calendar app, or with code.

‚úÖ 1. Create a .ics file Manually (Basic Text File)

You can create a .ics file using any text editor (e.g., Notepad, VS Code):

üìå Instructions:

  1. Open a text editor.
  2. Paste the content above and customize it (dates, summary, location, etc.).
  3. Save the file with a .ics extension, e.g., meeting.ics.
  4. You can now open or import this into Google Calendar, Outlook, etc.

üîπ 3. Parse with ical.js


import ICAL from 'ical.js';

// Assuming you have the .ics content as a string
function parseICS(icsContent) {
  const jcalData = ICAL.parse(icsContent);
  const comp = new ICAL.Component(jcalData);
  const vevents = comp.getAllSubcomponents('vevent');

  const events = vevents.map(eventComp => {
    const event = new ICAL.Event(eventComp);
    return {
      summary: event.summary,
      description: event.description,
      location: event.location,
      start: event.startDate.toJSDate(),
      end: event.endDate.toJSDate(),
    };
  });

  return events;
}

üõ†Ô∏è Alternative: Parse ICS Manually (for simple use-cases)

If your .ics files are basic and you want to avoid dependencies:


function parseICSManually(icsText) {
    const events = [];
    const lines = icsText.split(/\r?\n/);
    let event = {};
    let inEvent = false;
    
    lines.forEach(line => {
        if (line.startsWith('BEGIN:VEVENT')) {
        inEvent = true;
        event = {};
        } else if (line.startsWith('END:VEVENT')) {
        inEvent = false;
        events.push(event);
        } else if (inEvent) {
        if (line.startsWith('SUMMARY:')) event.summary = line.replace('SUMMARY:', '');
        if (line.startsWith('DTSTART:')) event.start = line.replace('DTSTART:', '');
        if (line.startsWith('DTEND:')) event.end = line.replace('DTEND:', '');
        if (line.startsWith('DESCRIPTION:')) event.description = line.replace('DESCRIPTION:', '');
        if (line.startsWith('LOCATION:')) event.location = line.replace('LOCATION:', '');
        }
    });
    
    return events;
    }
