Event-Driven Architecture (EDA)

Event-Driven Architecture (EDA) is a software design pattern in which events (i.e., significant changes in state) drive the flow of program execution. Instead of following a strict control flow (like in request-response systems), components in an EDA communicate by producing and reacting to events asynchronously.

üîß Key Components of EDA

  1. Event: A record of a change or action that has occurred, e.g., "UserCreated", "OrderPlaced".
  2. Event Producer: Generates and publishes events.
  3. Event Consumer: Subscribes to and processes events.
  4. Event Channel (Broker/Bus): Transports events from producers to consumers.
    • Examples: Apache Kafka, RabbitMQ, AWS SNS/SQS.
