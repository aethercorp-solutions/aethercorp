Infrastructure as Code (IaC)

Infrastructure as Code (IaC) tools allow you to provision, manage, and configure infrastructure using code, rather than manual processes. These tools help automate infrastructure deployment, improve consistency, and enable version control.

Here's a breakdown of popular IaC tools, categorized by approach:

üîß Declarative IaC Tools

You declare the desired state and the tool figures out how to get there.

1. Terraform

  • By: HashiCorp
  • Strengths: Cloud-agnostic (AWS, Azure, GCP, etc.), modular, state management
  • Use Cases: Managing cloud infrastructure across multiple providers

2. CloudFormation

  • By: AWS
  • Strengths: Deep integration with AWS, native support
  • Use Cases: AWS infrastructure provisioning
  • Format: YAML / JSON templates

3. Azure Resource Manager (ARM) / Bicep

  • By: Microsoft
  • Strengths: Native Azure support
  • Use Cases: Azure-specific deployments
  • Note: Bicep is a more readable DSL that compiles to ARM templates

4. Google Cloud Deployment Manager

  • By: Google
  • Use Cases: Provisioning Google Cloud resources
  • Format: YAML + Jinja2 or Python templates

‚öôÔ∏è Imperative IaC Tools

You define how to reach the desired state step-by-step.

5. Pulumi

  • Languages: TypeScript, Python, Go, .NET, Java
  • Strengths: Uses general-purpose programming languages
  • Use Cases: Developers wanting full programming power for IaC
  • Supports: AWS, Azure, GCP, Kubernetes, etc.

6. Chef

  • Language: Ruby-based DSL
  • Use Cases: Configuration management
  • Approach: Imperative + configuration
  • Note: More focused on server configuration than provisioning

7. Ansible

  • By: Red Hat
  • Language: YAML (playbooks)
  • Use Cases: Configuration management, orchestration, provisioning
  • Strengths: Agentless, SSH-based
  • IaC Style: More hybrid (declarative + imperative)

8. SaltStack

  • Language: YAML + Jinja2
  • Use Cases: Configuration, orchestration, automation
  • Strengths: Fast, scalable, real-time control

üê≥ Container-Specific IaC / Orchestration Tools

9. Kubernetes (YAML manifests)

  • Use Cases: Managing containerized workloads
  • IaC Role: Manages desired state of containers, networking, volumes, etc.

10. Helm

  • Use Cases: Packaging and deploying Kubernetes applications
  • Strengths: Templating Kubernetes YAML with variables
  • Think of it as: Kubernetes' version of a package manager + templating engine

üß† Special Mention: IaC Automation / Abstraction Tools

11. Crossplane

  • Use Case: Kubernetes-native infrastructure provisioning
  • Strengths: Manage cloud infrastructure like Kubernetes resources

12. CDK (Cloud Development Kit)

  • CDK for AWS, CDKTF for Terraform
  • Languages: TypeScript, Python, Java, etc.
  • Strengths: Programmatic abstraction on top of existing IaC
