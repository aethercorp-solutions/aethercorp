React Native technical interview

Preparing for a React Native technical interview means being ready to demonstrate skills across several areas: JavaScript/TypeScript, React fundamentals, React Native APIs, native modules, performance, and even deployment. Below is a comprehensive guide to help you prepare, followed by sample interview questions (with answers where relevant).

‚úÖ Topics to Prepare

1. JavaScript & ES6+

  • let, const, destructuring, spread/rest, async/await
  • Closures, scope, hoisting
  • Promises and async patterns
  • Array methods: .map(), .filter(), .reduce()

2. React (Core Concepts)

  • Components (functional vs class)
  • Props and State
  • Lifecycle methods (useEffect, componentDidMount, etc.)
  • Hooks (useState, useEffect, useRef, useMemo, useCallback)
  • Context API

3. React Native Essentials

  • Core components: View, Text, Image, FlatList, ScrollView, etc.
  • Styles and Flexbox
  • Navigation (react-navigation)
  • Handling gestures
  • Permissions and Platform APIs (Camera, GPS, etc.)
  • Linking, Deep Linking

4. State Management

  • useState and useReducer
  • Context API
  • Redux / Zustand / MobX (if used)
  • Async storage

5. Native Modules & Bridges

  • Linking native code (Android/Java, iOS/Swift/Obj-C)
  • Using third-party native modules
  • Custom native module creation (basic idea)

6. Networking

  • fetch, axios
  • Handling API responses and errors
  • Pagination and lazy loading
  • Offline support

7. Performance Optimization

  • Re-render optimizations
  • FlatList optimizations
  • Avoiding anonymous functions in JSX
  • Memoization (React.memo, useMemo)
  • Lazy loading, code splitting

8. Testing

  • Unit testing with Jest
  • Component testing with React Native Testing Library
  • Detox for end-to-end testing

9. Build & Deployment

  • App signing
  • APK/IPA generation
  • Over-the-air updates (CodePush)
  • App Store / Play Store release process
