File-based routing in React Native

File-based routing in React Native isn't supported out-of-the-box like in Next.js or Remix, but you can achieve a similar experience using libraries or tooling that adds this functionality on top of React Navigation.

‚úÖ What is File-Based Routing?

File-based routing means your file/folder structure determines your app's routes. For example:


/screens
  /Home.js         => route: "/"
  /Profile.js      => route: "/profile"
  /settings/Index.js => route: "/settings"

üì¶ Libraries That Support File-Based Routing in React Native

  1. Expo Router (Recommended) Expo Router brings file-based routing to React Native, built on top of React Navigation, and works well for both Expo and bare React Native projects.

    üîß Setup

    1. Create a new Expo project (or use an existing one): 
      
npx create-expo-app my-app
cd my-app
    2. Install Expo Router: 
      
npx expo install expo-router
    3. Set up the routing structure:

      Create an app/ directory:

      
/app
  /index.js         => route: "/"
  /profile.js       => route: "/profile"
  /settings/index.js => route: "/settings"
    4. Update app/_layout.js for navigation: 
      
// app/_layout.js
import { Stack } from 'expo-router';

export default function Layout() {
  return <Stack />;
}
    5. Run the app: 
      
        npx expo start
    ‚ûï Pros:
    • Simple, clean, and scalable routing
    • Deep linking and web support built-in
    • Supported by Expo team
  2. React Native File Router

    For non-Expo projects or custom setups, you might use libraries like:

    • react-native-file-router (less popular)
    • Custom implementations (using Babel or Metro plugins)

    However, Expo Router is more actively maintained and better integrated.

‚úÖ When to Use File-Based Routing

  • You want clean code organization
  • You prefer convention over configuration
  • You need deep linking, universal linking, or web support

‚ùå When Not to Use

  • You prefer programmatic or manual route configuration
  • Your project already has a custom navigation setup

üîß Example: UI Code Comparison

React (Web)

    return (
        <div style={{ padding: 20 }}>
        <h1>Hello, Web!</h1>
        </div>
    );
}

React Native

function App() {
  return (
    <View style={{ padding: 20 }}>
        <Text>Hello, Mobile!</Text>
    </View>
  );
}

Notice:

  • No <div> or <h1> in React Native.
  • Uses <View> and <Text> ‚Äî these map to native components.

ü§ù Shared Concepts

Despite the differences, you can share logic (state management, hooks, business logic, API calls) between React and React Native.

Shared tools include:

  • React Hooks (useState, useEffect, etc.)
  • Context API
  • State libraries (Redux, Zustand, Recoil)
  • UI Libraries (with cross-platform support like React Native Paper, Tamagui, or NativeWind)

Great question. The CLI commands to create a React Web app vs a React Native app differ because they use different tools and scaffolding systems.

üß± 1. Creating a React Web App

‚ñ∂Ô∏è Command


npx create-react-app my-web-app

This uses Create React App (CRA), which sets up:

  • Webpack
  • Babel
  • React
  • HTML/CSS build system

‚úÖ Output

  • Runs in a web browser
  • Creates a public/index.html and injects React into it
  • Uses HTML, CSS, JSX

üì± 2. Creating a React Native App

There are two common ways to do this:

Option A: Using Expo CLI (Recommended)


npx create-expo-app my-native-app
cd my-native-app
npx expo start

‚úÖ Output

  • Cross-platform (iOS, Android, Web)
  • Includes Metro bundler
  • Very beginner-friendly
  • No need for Xcode or Android Studio to start

Option B: Using React Native CLI (Bare Workflow)


npx react-native init MyNativeApp
cd MyNativeApp
npx react-native run-android  # or run-ios (macOS only)

‚úÖ Output

  • Gives more native control (e.g., custom native modules)
  • Requires:
    • Android Studio (for Android)
    • Xcode (for iOS/macOS)
