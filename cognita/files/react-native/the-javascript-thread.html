The JavaScript Thread

It is a separate dedicated thread where all your JavaScript code runs ‚Äî including:

  • React rendering
  • State updates
  • Business logic
  • Timers (setTimeout, setInterval)
  • Bridge communication with native

It does NOT run on the main UI thread (the native thread that draws the screen).

This separation makes animations and UI interactions smoother.

üìå How it Works (Simplified)

React Native has three major threads:

1. UI Thread (Main Thread)

  • Renders views
  • Handles touches & gestures
  • Runs native UI code

2. JavaScript Thread

  • Runs your JS bundle
  • Executes React code
  • Calculates layout (via Yoga)
  • Sends commands to native

3. Shadow Thread (Layout)

  • Runs Yoga layout engine
  • Calculates view positions & sizes

The JS thread communicates with native code using a bridge:

  • JS thread sends messages to the native side (async)
  • Native sends messages back to JS (async)
  • Works like a queue-based messaging system

Example:


JS thread -> Bridge -> Native Modules (Objective-C/Swift/Java/Kotlin)
