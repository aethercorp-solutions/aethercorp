React.createRef()

React.createRef() is a method in React used to create a ref—an object that can hold a reference to a React element or a class component instance. Refs are primarily used when you need to directly access a DOM element or a component’s methods, bypassing the normal React data flow.

Here’s a breakdown of how it works and how to use it:

1. Creating a ref


import React, { Component } from 'react';

class MyComponent extends Component {
  constructor(props) {
    super(props);
    this.myInputRef = React.createRef(); // Create a ref
  }

  focusInput = () => {
    this.myInputRef.current.focus(); // Access the DOM node
  }

  render() {
    return (
      <div>
        <input type="text" ref={this.myInputRef} />
        <button onClick={this.focusInput}>Focus Input</button>
    </div>
    );
  }
}

export default MyComponent;
  • React.createRef() returns an object with a current property: { current: null }.
  • When attached to a JSX element via ref={...}, current will point to the DOM node (or component instance after mount).
  • You can then access methods like .focus(), .scrollIntoView(), etc.

2. Using refs with class components

Refs can also point to class component instances:


class Child extends Component {
    sayHello() {
        alert('Hello!');
    }
    
    render() {
        return <div>Child Component</div>;
    }
}
    
class Parent extends Component {
    constructor(props) {
        super(props);
        this.childRef = React.createRef();
    }
    
    callChildMethod = () => {
        this.childRef.current.sayHello(); // Call method on child
    }
    
    render() {
        return (
        <div>
            <Child ref={this.childRef} />
            <button onClick={this.callChildMethod}>Call Child Method</button>
        </div>
        );
    }
}
  • this.childRef.current now points to the Child instance.
  • You can call any public method defined in Child.

3. Using refs with functional components

Functional components cannot be directly given refs unless you use forwardRef:


import React, { forwardRef, useRef } from 'react';

const Child = forwardRef((props, ref) => (
  <input ref={ref} placeholder="Type here" />
));

function Parent() {
  const inputRef = useRef();

  const focusInput = () => {
    inputRef.current.focus();
  };

  return (
    <div>
        <Child ref={inputRef} />
        <button onClick={focusInput}<Focus Input</button>
    </div>
  );
}
  • Here, forwardRef allows the parent to attach a ref to the DOM element inside the functional child component.

Key Points

  • Refs bypass React’s data flow, so use them sparingly.
  • Good for focusing inputs, controlling media playback, triggering animations, or integrating with third-party libraries.
  • Class components: React.createRef() in constructor.
  • Functional components: useRef() hook.
