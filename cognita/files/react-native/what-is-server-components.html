Server Component

A Server Component is a type of UI component that runs on the server instead of in the browser. It’s most commonly used in modern React frameworks like Next.js (App Router).

Simple idea (1 sentence)

👉 Server Components render on the server, send ready-made HTML to the browser, and never run JavaScript on the client.

Why Server Components exist

Traditional React apps:

  • Run a lot of JavaScript in the browser
  • Fetch data after the page loads
  • Can be slow on first load

Server Components:

  • Fetch data on the server
  • Reduce JavaScript sent to the browser
  • Load faster and are more secure

What Server Components can do

✅ Access databases directly

✅ Call backend APIs securely

✅ Read environment variables

✅ Use server-only secrets

✅ Reduce bundle size

🚫 Cannot:

  • Use useState, useEffect
  • Handle clicks or user events
  • Access window, document
  • Run in the browser

Example (Next.js Server Component)


// app/page.tsx
export default async function Page() {
  const users = await fetch("https://api.example.com/users").then(res => res.json());

  return (
    <ul>
      {users.map(user => (
        <li key={user.id}>{user.name}</li>
      ))}
    </ul>
  );
}
  • This runs only on the server
  • The browser receives HTML, not JS logic

Client Component (for comparison)


"use client";

import { useState, useEffect } from "react";

export default function Page() {
  const [users, setUsers] = useState([]);

  useEffect(() => {
    fetch("/api/users").then(res => res.json()).then(setUsers);
  }, []);

  return <div>{users.length}</div>
}

Server vs Client Components

Feature Server Component Client Component
Runs where Server Browser
JavaScript sent ❌ None ✅ Yes
Data fetching Before render After render
Hooks ❌ No ✅ Yes
Event handling ❌ No ✅ Yes
Security ✅ High ❌ Lower

How they work together

You usually combine them:


// Server Component
import Button from "./Button";

export default function Page() {
  return <Button />;
}
            

// Client Component
"use client";

export default function Button() {
  return <button onClick={() => alert("Hi!")}>Click me</button>;
}

➡ Server Component renders the page

➡ Client Component handles interaction

When to use Server Components

✔ Fetching data

✔ Rendering lists/pages

✔ SEO-critical content

✔ Performance optimization

When to use Client Components

✔ Forms

✔ Buttons

✔ Animations

✔ Modals

✔ User interaction

One-line summary

Server Components = fast, secure, zero-JS rendering on the server.
