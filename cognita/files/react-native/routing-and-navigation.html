Routing and Navigation

Got it! Let’s go deep into routing and navigation, particularly for React Native, since that seems to be your main context. I’ll give a structured overview with explanations, examples, and best practices.

1. What is Routing and Navigation?

  • Routing: Determines which screen or component should be shown for a given URL, state, or action.
  • Navigation: The mechanism to move between screens or pass data during transitions.

In React Native, there’s no built-in routing like in web React, so libraries like React Navigation or React Native Navigation are used.

2. React Navigation Overview

React Navigation is the most popular library. It supports:

  1. Stack Navigation (like a call stack) – for push/pop transitions.
  2. Tab Navigation – for bottom or top tabs.
  3. Drawer Navigation – side menus.
  4. Nested Navigation – combining the above (e.g., tab with stacks inside).
  5. Deep Linking – linking to specific screens from external sources.

2.1 Installation


npm install @react-navigation/native
npm install react-native-screens react-native-safe-area-context
npm install @react-navigation/native-stack
npm install @react-navigation/bottom-tabs

Then wrap your app:


import { NavigationContainer } from '@react-navigation/native';

export default function App() {
  return (
    <NavigationContainer>
      {/* Your navigator goes here */}
    </NavigationContainer>
  );
}

2.2 Stack Navigation (Basic Example)


import { createNativeStackNavigator } from '@react-navigation/native-stack';
import HomeScreen from './screens/HomeScreen';
import DetailsScreen from './screens/DetailsScreen';

const Stack = createNativeStackNavigator();

export default function App() {
    return (
    <NavigationContainer>
        <Stack.Navigator initialRouteName="Home">
        <Stack.Screen name="Home" component={HomeScreen} />
        <Stack.Screen name="Details" component={DetailsScreen} />
        </Stack.Navigator>
    </NavigationContainer>
    );
}

Navigate between screens:


// Inside HomeScreen
<Button
  title="Go to Details"
  onPress={() => navigation.navigate('Details', { itemId: 42 })}
/>

// Inside DetailsScreen
const { itemId } = route.params;

2.3 Tab Navigation Example


import { createBottomTabNavigator } from '@react-navigation/bottom-tabs';
import HomeScreen from './screens/HomeScreen';
import ProfileScreen from './screens/ProfileScreen';

const Tab = createBottomTabNavigator();

export default function App() {
    return (
    <NavigationContainer>
        <Tab.Navigator>
        <Tab.Screen name="Home" component={HomeScreen} />
        <Tab.Screen name="Profile" component={ProfileScreen} />
        </Tab.Navigator>
    </NavigationContainer>
    );
}

Tabs are great for persistent navigation across multiple screens, like in Instagram or Twitter.

2.5 Passing Data Between Screens

You can pass data with navigation.navigate:


navigation.navigate('Details', {
    itemId: 42,
    otherParam: 'Hello'
});

And read it via route.params:


const { itemId, otherParam } = route.params;

2.6 Common Navigation Functions

Function Description
navigate('Screen') Go to a screen (push if in stack)
goBack() Go back to previous screen
push('Screen') Push new instance even if already on screen
replace('Screen') Replace current screen (no back)
reset({ index, routes }) Reset navigation state (common after login/logout)

2.7 Nested Navigators

You can combine navigators:


<Tab.Navigator>
    <Tab.Screen name="HomeStack" component={HomeStackNavigator} />
    <Tab.Screen name="ProfileStack" component={ProfileStackNavigator} />
</Tab.Navigator>

This is common in large apps: tab bar with independent stacks for each tab.

2.8 Deep Linking (Optional Advanced)

React Navigation can handle URLs, e.g., myapp://details/42:


const linking = {
    prefixes: ['myapp://'],
    config: {
        screens: {
        Home: 'home',
        Details: 'details/:itemId',
        },
    },
    };
    
<NavigationContainer linking={linking}>
{/* ... */}
</NavigationContainer>

✅ Best Practices

  1. Use Stack for hierarchical flows (login → dashboard → details).
  2. Use Tab for major sections.
  3. Use Drawer for app settings or secondary options.
  4. Avoid too many nested navigators without need — it can be confusing.
  5. Always type your params (TypeScript) for safety.
