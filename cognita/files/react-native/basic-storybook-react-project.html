Basic Storybook + React project structure

A basic Storybook + React project structure helps keep components, stories, and configuration organized and scalable. Here's a clean and commonly used file/folder structure for a basic setup:

Project Root


my-app/
├── .storybook/              # Storybook configuration
│   ├── main.js              # Main Storybook config (stories, addons, etc.)
│   ├── preview.js           # Global decorators, parameters
│   └── manager.js           # Optional: theming Storybook UI
│
├── src/                     # React source code
│   ├── components/          # Reusable UI components
│   │   ├── Button/          
│   │   │   ├── Button.jsx
│   │   │   ├── Button.css
│   │   │   └── Button.stories.jsx
│   │   └── ...
│   │
│   ├── App.jsx              # Main App component
│   └── index.js             # Entry point
│
├── public/                  # Static files
│   └── index.html
│
├── .gitignore
├── package.json
├── README.md
└── vite.config.js | webpack.config.js (depending on bundler)

Key Files

.storybook/main.js

Configures Storybook:


module.exports = {
    stories: ['../src/**/*.stories.@(js|jsx|ts|tsx)'],
    addons: ['@storybook/addon-links', '@storybook/addon-essentials'],
    framework: {
        name: '@storybook/react-vite', // or '@storybook/react-webpack5'
        options: {},
    },
};

.storybook/preview.js

Global decorators, parameters, themes:


export const parameters = {
    actions: { argTypesRegex: "^on[A-Z].*" },
    controls: { matchers: { color: /background|color/i, date: /Date$/ } },
};

Tips

  • Keep each component in its own folder with its .jsx, .css, and .stories.jsx files.
  • Use CSF (Component Story Format) for stories.
  • Storybook can be run using npm run storybook or yarn storybook.
