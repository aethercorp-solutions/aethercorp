Understanding Redux and Implementing it with Redux Toolkit in React Native

When you’re making a mobile app, keeping track of information is a big deal. That’s where Redux steps in! Redux is like a special helper that makes it easier to organize and handle the important details in your React Native app. In this guide, we’ll break down the key ideas behind Redux and show you how to use it in a React Native app with the user-friendly Redux Toolkit.

What is Redux?

Imagine your JavaScript app is like a big container of information. Now, think of Redux as a friendly organizer for that container, especially when you’re using popular tools like React or React Native. The main job of Redux is to keep all the important stuff in one place, making it super easy to handle and change things in a way that you can always expect.

Core Concepts in Redux:

  1. Store: The store is a single, immutable object that represents the state of your entire application. It holds the application state and provides methods to update it.
  2. Actions: Actions are plain JavaScript objects that describe changes in the application state. They are the only source of information for the store.
  3. Reducers: Reducers specify how the application’s state changes in response to actions. They are pure functions that take the previous state and an action, and return the new state.
  4. Dispatch: Dispatch is a method used to send actions to the store. It is the only way to trigger a state change.
  5. Selectors: Selectors are functions that extract specific pieces of information from the store.

Setting Up a React Native App with Redux Toolkit:

Step 1: Create a React Native App


npx react-native init ReduxExample
cd ReduxExample

This will generate the basic structure of a React Native app.

Step 2: Install Redux Toolkit and Dependencies

Install Redux Toolkit and react-redux using npm or yarn:


yarn add @reduxjs/toolkit react-redux
   or
npm install @reduxjs/toolkit react-redux

Step 3: Create a Slice

In Redux, a “slice” is a piece of the store’s state and the corresponding actions/reducers for that piece. Create a file named counterSlice.js in the src folder:


// counterSlice.js
import { createSlice } from '@reduxjs/toolkit';

export const counterSlice = createSlice({
  name: 'counter',
  initialState: {
    value: 0,
  },
  reducers: {
    increment: (state) => {
      state.value += 1;
    },
    decrement: (state) => {
      state.value -= 1;
    },
  },
});

export const { increment, decrement } = counterSlice.actions;

export const selectCount = (state) => state.counter.value;

export default counterSlice.reducer;

This file defines a Redux slice for a counter, including the initial state, actions (increment and decrement), and a selector (selectCount) for accessing the counter value.

Step 4: Create the Redux Store

In the src folder, create a file named store.js:


// store.js
import { configureStore } from '@reduxjs/toolkit';
import counterReducer from './counterSlice';

export const store = configureStore({
  reducer: {
    counter: counterReducer,
  },
});

This file creates the Redux store using configureStore from Redux Toolkit and includes the counterReducer from the counterSlice file.

Open the index.js file, and wrap your App component with the Provider from react-redux and pass the store:


// index.js
import { AppRegistry } from 'react-native';
import App from './App';
import { name as appName } from './app.json';
import { store } from './store';
import { Provider } from 'react-redux';

const ReduxExample = () => (
  <Provider store={store}>
    <App />
  </Provider>
);

AppRegistry.registerComponent(appName, () => ReduxExample);

This sets up the Redux store to be available to all components in your React Native app.

Step 6: Use Redux in your React Native Components

Now, modify the App.js file to use Redux:


// App.js
import React from 'react';
import { useSelector, useDispatch } from 'react-redux';
import { increment, decrement, selectCount } from './counterSlice';
import { View, Text, Button } from 'react-native';

const App = () => {
  const count = useSelector(selectCount);
  const dispatch = useDispatch();

  return (
    <View style={{ flex: 1, justifyContent: 'center', alignItems: 'center' }}>
        <Text>Redux Example</Text>
        <Text>Count: {count}</Text>
        <Button title="Increment" onPress={() => dispatch(increment())} />
        <Button title="Decrement" onPress={() => dispatch(decrement())} />
    </View>
  );
};

export default App;

This component uses the useSelector hook to access the counter value and the useDispatch hook to dispatch the increment and decrement actions.

Step 7: Run Your App


npx react-native run-android
# or
npx react-native run-ios

This command will start your React Native app with the configured Redux store. You can now use Redux to manage the state of your React Native application efficiently. As your app grows, continue to organize your state logic using slices and actions to maintain a clean and scalable codebase.

Conclusion:

We’ve covered the basics of Redux and how to implement Redux Toolkit in React Native. By understanding the core concepts and following the steps, you can enhance state management in your mobile apps.

As you explore React Native development, remember that Redux Toolkit simplifies the process, making it more accessible. Thanks for reading, and happy coding! If you found this helpful, give it a thumbs up!

App.js


import { useSelector, useDispatch } from "react-redux";
export default function App() {
    let counter = useSelector((state) => state.counter);
    let dispatch = useDispatch();
    let handleClick = (type) => {
        let action = {
            type
        };
        dispatch(action);
    };

    return (
        <div className="App">
            <div> Counter: {counter} </div>
            <button onClick={(e) => handleClick("add")}> add </button>
            <button onClick={(e) => handleClick("minus")}> minus </button>
        </div>
    );
}

index.js


import { StrictMode } from "react";
import ReactDOM from "react-dom";
import { Provider } from "react-redux";
import store from "./store";
import App from "./App";

ReactDOM.render(
    <StrictMode>

            <App />
        </Provider>
    </StrictMode>,

store.js


import { createStore } from "redux";

let defaultState = {
    counter: 0
};

const counterReducer = (state = defaultState, action) => {
    switch (action.type) {
        case "add":
            return {
                ...state,
                counter: state.counter + 1
            };
        case "minus":
            return {
                ...state,
                counter: state.counter - 1
            };
        default:
            return state;
    }
};

export default createStore(counterReducer);

<link rel="manifest" href="%PUBLIC_URL%/manifest.json">

The line you've posted is from a typical HTML document (usually in the section), and it's used in web applications—especially React apps—to link to a Web App Manifest file:

Here's what it does:

  • rel="manifest": Specifies that this link is to a Web App Manifest file.
  • href="%PUBLIC_URL%/manifest.json": Points to the manifest.json file, which contains metadata about the web application (name, icons, theme color, etc.).

Important Details:

  1. %PUBLIC_URL% is a placeholder used by build tools like Create React App (CRA).
    • During the build process, %PUBLIC_URL% gets replaced with the actual URL path to the public folder.
    • So at runtime, this becomes something like: 
      
<link rel="manifest" href="/manifest.json">
  2. manifest.json typically lives in the public/ directory of a React app.

Why It's Used:

  • Allows your web app to be installed on devices (like a native app).
  • Provides metadata such as:
    • App name
    • App icon
    • Start URL
    • Background and theme colors
    • Display mode (fullscreen, standalone, etc.)
