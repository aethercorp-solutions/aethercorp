üöß React Native 0.5x (2017-era) ‚Äì iOS Build System

‚ùå No CocoaPods support

React Native 0.5 (and early 0.x versions) did NOT use CocoaPods for iOS dependency management.

What that means:

  • All native modules required manual linking.
  • You had to edit:
    • Libraries/ folder (drag the Xcode project of the native module)
    • Link Binary with Libraries
    • Header Search Paths
    • Sometimes add .m and .h files manually.

Example: Importing RNFirebase or react-native-maps

You had to:

  1. Download the native .xcworkspace or .xcodeproj
  2. Drag it into your iOS project
  3. Manually link binaries
  4. Add frameworks manually
  5. Add Obj-C bridging code manually (when needed)

Build System Used:

  • Legacy Xcode build system
  • No automatic dependency resolution
  • No Podfile
  • No ‚Äúpod install‚Äù

üöÄ React Native 0.60+ ‚Äì iOS Build System

‚úÖ Full CocoaPods support

Starting RN 0.60, iOS uses CocoaPods by default.

What that means:

  • Autolinking (RN CLI auto-detects native modules)
  • All dependencies installed via: 
    
cd ios
pod install
  • Libraries provide .podspec files
  • Linking via Xcode is no longer needed

Build System Used:

  • Modern Xcode build system
  • Uses .xcworkspace instead of .xcodeproj
  • Autolinking via react-native.config.js
