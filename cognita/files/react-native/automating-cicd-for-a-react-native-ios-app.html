Automating CI/CD for a React Native iOS app to Apple’s App Store

Automating CI/CD for a React Native iOS app to Apple’s App Store means automatically building, signing, and uploading your app to App Store Connect whenever you push code to Git.

This is common for teams and especially useful if you're maintaining apps like your TryCastanet React Native project because it removes manual Xcode builds and reduces release errors. 🚀

Below is a complete production-level CI/CD architecture.

1. Overall CI/CD Architecture

Typical pipeline:


Developer pushes code → GitHub/GitLab
        ↓
CI Server (GitHub Actions / Bitrise / CircleCI / Codemagic)
        ↓
Install dependencies
        ↓
Run tests / lint
        ↓
Build iOS app
        ↓
Code signing
        ↓
Archive .ipa
        ↓
Upload to App Store Connect
        ↓
TestFlight / App Store Release

Popular CI tools:

For most React Native apps today, GitHub Actions + Fastlane is the standard stack.

2. Required Apple Setup

Before CI/CD works, configure these:

1️⃣ Apple Developer Account

Create certificates and provisioning profiles in

Apple Developer

You need:

  • Distribution Certificate
  • App Store Provisioning Profile
  • App Identifier (Bundle ID)

Example:


com.castanet.app

2️⃣ App Store Connect App

Create the app in

App Store Connect

Configure:

  • Bundle ID
  • Version
  • Build number
  • TestFlight

3️⃣ API Key for Uploading

Generate API key in App Store Connect


Users and Access
→ Keys
→ App Store Connect API
→ Generate API Key

You will get:


Issuer ID
Key ID
.p8 key file

Generate:


Users and Access
→ Keys
→ App Store Connect API Key

Save:


Issuer ID
Key ID
AuthKey_XXXX.p8

CI systems use this to upload builds automatically.

3. Install Fastlane (Core Automation Tool)

Fastlane automates iOS builds and App Store uploads.

Install:


brew install fastlane

or


sudo gem install fastlane

Inside your React Native project:


cd ios
fastlane init

Choose:


2. Automate beta distribution to TestFlight

4. Fastlane Folder Structure

Fastlane creates:


ios/
 └ fastlane/
      Appfile
      Fastfile

5. Configure Appfile


app_identifier "com.castanet.app"
apple_id "your@email.com"

team_id "YOUR_TEAM_ID"

6. Create Fastlane Build Script

Example Fastfile:


default_platform(:ios)

platform :ios do

  desc "Build and upload to TestFlight"
  lane :beta do

    increment_build_number(
      xcodeproj: "Castanet.xcodeproj"
    )

    build_app(
      scheme: "Castanet",
      export_method: "app-store"
    )

    upload_to_testflight(
      skip_waiting_for_build_processing: true
    )

  end
end

This automatically:

1️⃣ increments build number

2️⃣ builds app

3️⃣ uploads to TestFlight

7. Code Signing Automation

Use Fastlane match to manage certificates.


fastlane match appstore

It stores certificates in a private Git repo.

Benefits:

  • team-friendly
  • CI friendly
  • secure

8. GitHub Actions CI/CD Pipeline

Example .github/workflows/ios.yml


name: iOS Build

on:
  push:
    branches:
      - main

jobs:
  build:

    runs-on: macos-latest

    steps:
      - name: Checkout repo
        uses: actions/checkout@v3

      - name: Install Node
        uses: actions/setup-node@v3
        with:
          node-version: 18

      - name: Install dependencies
        run: |
          npm install
          cd ios
          pod install

      - name: Install Fastlane
        run: sudo gem install fastlane

      - name: Build and Upload
        run: |
          cd ios
          fastlane beta

When you push to main, CI automatically:


builds app
uploads to TestFlight

5. Handling Code Signing Automatically

iOS signing is the hardest part.

Best solution:


fastlane match

Match stores certificates in a private repo.

Initialize:


fastlane match init

Example Matchfile:


git_url("git@github.com:company/certificates.git")

storage_mode("git")

type("appstore")

Then run:


fastlane match appstore

Now CI can automatically download signing certificates.

10. Production Release Automation

Add a release lane.

Example:


lane :release do
  build_app(
    scheme: "Castanet",
    export_method: "app-store"
  )

  upload_to_app_store(
    submit_for_review: true
  )
end

Now CI can fully automate publishing.
