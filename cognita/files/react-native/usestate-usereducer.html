useState and useReducer

In React Native (just like in React for the web), useState and useReducer are two important React Hooks used for managing state in function components.

🔹 useState

✅ When to use:

  • For simple state (primitives, small objects).
  • When your state updates are straightforward.

📌 Syntax:


const [state, setState] = useState(initialValue);

📦 Example:


import React, { useState } from 'react';
import { View, Text, Button } from 'react-native';

const Counter = () => {
  const [count, setCount] = useState(0);

  return (
    <View>
        <Text>Count: {count}</Text>
        <Button title="Increase" onPress={() => setCount(count + 1)} />
    </View>
  );
};

export default Counter;

🔸 useReducer

✅ When to use:

  • For complex state logic (e.g., multiple related values).
  • When the next state depends on the previous state.
  • For better scalability and testability.

📌 Syntax:


const [state, dispatch] = useReducer(reducerFunction, initialState);

📦 Example:


import React, { useReducer } from 'react';
import { View, Text, Button } from 'react-native';

const initialState = { count: 0 };

function reducer(state, action) {
  switch (action.type) {
    case 'increment':
      return { count: state.count + 1 };
    case 'decrement':
      return { count: state.count - 1 };
    default:
      return state;
  }
}

const Counter = () => {
  const [state, dispatch] = useReducer(reducer, initialState);

  return (
    <View>
        <Text>Count: {state.count}</Text>
        <Button title="Increase" onPress={() => dispatch({ type: 'increment' })} />
        <Button title="Decrease" onPress={() => dispatch({ type: 'decrement' })} />
    </View>
  );
};

export default Counter;

⚖️ useState vs useReducer

Feature useState useReducer
Complexity Simple state Complex state or multiple sub-values
Update logic Inline in component Centralized in reducer function
Suitable for 1–2 state variables Multiple related states
Debuggability Less (scattered logic) More (pure functions, easier to test)
Performance Faster for simple cases Better when managing deeply nested state
