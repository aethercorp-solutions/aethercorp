Webpack Loader

A Webpack loader is a tool that transforms the files you import or require in your JavaScript code into modules that Webpack can understand and include in your final bundle.

ðŸ”§ What does a loader do?

Loaders let you preprocess files as you import or load them. This is especially useful for:

  • Transpiling modern JavaScript (e.g., with Babel)
  • Converting TypeScript to JavaScript
  • Loading CSS or SASS
  • Optimizing images
  • Handling Vue, React, or Angular components

ðŸ§± Basic Usage in Webpack Config

Loaders are defined in the module.rules array inside the webpack.config.js file:


module.exports = {
    module: {
        rules: [
        {
            test: /\.js$/,              // Which files to apply the loader to
            exclude: /node_modules/,    // Exclude certain paths
            use: {
            loader: 'babel-loader',   // Which loader to use
            options: {
                presets: ['@babel/preset-env']
            }
            }
        }
        ]
    }
};

ðŸ”„ Chain of Loaders

You can use multiple loaders on the same file, applying them from right to left (last to first in the array):


use: ['style-loader', 'css-loader']

This example does:

  1. css-loader processes the CSS file (resolves @import and url())
  2. style-loader injects CSS into the DOM
