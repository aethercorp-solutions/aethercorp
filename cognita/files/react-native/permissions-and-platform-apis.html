üì± Platform-Specific APIs & Permissions

üî∑ Android

  • Permissions declared in AndroidManifest.xml.
  • From Android 6.0 (API 23) onward: Runtime permissions are required.
  • Use ContextCompat.checkSelfPermission() and requestPermissions().

üîß Example (Kotlin):


if (ContextCompat.checkSelfPermission(this, Manifest.permission.CAMERA)
    != PackageManager.PERMISSION_GRANTED) {
    ActivityCompat.requestPermissions(this, arrayOf(Manifest.permission.CAMERA), CAMERA_REQUEST_CODE)
}

üçé iOS

  • Permissions declared in Info.plist.
  • Prompt automatically shown when API is used for the first time.
  • Use CLLocationManager (for GPS), AVCaptureDevice (for Camera/Mic).

üîß Example (Swift):

if cameraStatus == .notDetermined {
    AVCaptureDevice.requestAccess(for: .video) { granted in
        // Handle permission
    }

‚öôÔ∏è Platform APIs Overview

1. Camera API

  • Use Cases: Barcode scanning, face detection, taking pictures.
  • Web: MediaDevices.getUserMedia()
  • Android: CameraX, Camera2 API
  • iOS: AVCaptureSession

2. GPS / Location API

  • Use Cases: Maps, navigation, geofencing.
  • Web: navigator.geolocation
  • Android: FusedLocationProviderClient
  • iOS: CoreLocation

3. Microphone API

  • Use Cases: Voice input, audio recording.
  • Web: getUserMedia({ audio: true })
  • Android: MediaRecorder, AudioRecord
  • iOS: AVAudioRecorder

4. Bluetooth API

  • Use Cases: Wearables, IoT devices.
  • Web: Web Bluetooth API (limited support)
  • Android: BluetoothAdapter, BluetoothGatt
  • iOS: CoreBluetooth

üîê Best Practices

  1. Request only what you need ‚Äì Avoid asking for unnecessary permissions.
  2. Provide context ‚Äì Explain why you need the permission before prompting.
  3. Handle denial gracefully ‚Äì Let users continue with limited functionality if possible.
  4. Respect platform guidelines ‚Äì Especially for Apple App Store and Google Play.
