Jest is a zero-config testing framework

Jest is a delightful, zero-config testing framework maintained by Meta (Facebook), widely used for testing JavaScript applications ‚Äî especially React apps, though it works well with Node.js and other libraries too.

üîß Key Features of Jest

  • Zero Config ‚Äì Works out of the box for most JS projects.
  • Snapshot Testing ‚Äì For UI consistency.
  • Mocking ‚Äì Built-in support for function and module mocks.
  • Isolated Tests ‚Äì Runs tests in parallel to improve performance.
  • Code Coverage ‚Äì Built-in coverage reports.

üì¶ Installing Jest


npm install --save-dev jest

Or with Yarn:


yarn add --dev jest

Then, add a test script in your package.json:


"scripts": {
    "test": "jest"
}

üß™ Writing a Simple Test

sum.js


function sum(a, b) {
    return a + b;
}
module.exports = sum;

sum.test.js


const sum = require('./sum');

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});

Then run:


npm test

üß∞ Useful Jest Functions

  • test() / it() ‚Äì Define a test case.
  • expect() ‚Äì Assertion library (e.g., expect(value).toBe(expected)).
  • beforeEach(), afterEach() ‚Äì Setup/teardown per test.
  • beforeAll(), afterAll() ‚Äì Setup/teardown per test file.
  • .mock() / jest.fn() ‚Äì Mocking functions/modules.

‚úÖ Example: Testing an Async Function


const fetchData = () => Promise.resolve('peanut butter');

test('the data is peanut butter', async () => {
  const data = await fetchData();
  expect(data).toBe('peanut butter');
});

‚úÖ Step-by-Step: Setting Up Jest

1. üèóÔ∏è Initialize Your Project

If you haven't already:


npm init -y

2. üì¶ Install Jest


npm install --save-dev jest

If you're using TypeScript, also install:


npm install --save-dev ts-jest @types/jest typescript

3. ‚öôÔ∏è Configure Jest

A. JavaScript Projects

In package.json, add:


"scripts": {
    "test": "jest"
}

You can run tests using:


npm test

B. TypeScript Projects

Initialize ts-jest:


npx ts-jest config:init

This creates a jest.config.js file with TS support.

Update your package.json scripts:


"scripts": {
    "test": "jest"
}

Make sure your tsconfig.json is configured for Jest (e.g., esnext modules, "module": "commonjs").

4. üìÅ Create Test Files

By convention, Jest looks for files like:

  • *.test.js / *.test.ts
  • *.spec.js / *.spec.ts
  • Or any test files in a __tests__ folder

Example:

math.js


function add(a, b) {
    return a + b;
}
module.exports = add;

math.test.js


const add = require('./math');

test('adds 2 + 3 to equal 5', () => {
  expect(add(2, 3)).toBe(5);
});

5. üöÄ Run Tests


npm test

Jest will automatically find and run your tests.

üß™ Optional: Configure Jest Further

Create a manual jest.config.js:


// jest.config.js
module.exports = {
  testEnvironment: 'node', // or 'jsdom' for browser-like env (React)
  verbose: true,
  collectCoverage: true,
  coverageDirectory: 'coverage',
};

üß∞ Bonus: Useful Flags


# Run specific test file
npx jest math.test.js

# Watch mode (great for TDD)
npx jest --watch

# Show coverage report
npx jest --coverage

3. Example Test

Create a file:

src/App.test.js


import { render, screen } from '@testing-library/react';
import App from './App';

test('renders learn react link', () => {
  render(<App />);
  const linkElement = screen.getByText(/learn react/i);
  expect(linkElement).toBeInTheDocument();
});

This is a basic unit test for a React component (App) using React Testing Library and Jest. Here's a breakdown of what it's doing:

üîç What this test does:

  1. Render the App component: 
    
render(<App />);
    • This simulates rendering your component in a virtual DOM.
  2. Find an element with the text "learn react" (case-insensitive): 
    
const linkElement = screen.getByText(/learn react/i);
    • This uses a regular expression (/learn react/i) to search for text on the screen.
    • i means it's case-insensitive.
  3. Check that the element exists in the document: 
    
expect(linkElement).toBeInTheDocument();
    • This is an assertion that ensures the text is present.

‚úÖ What this test is validating:

It confirms that when App is rendered, it contains a visible element (like a link or heading) with the text "learn react". This is typically from a default create-react-app scaffold, where the App.js file includes a link like:


<a href="https://reactjs.org">Learn React</a>

üß™ To run the test:

You can run this test with:


npm test

or


yarn test

It should pass if the text ‚Äúlearn react‚Äù exists somewhere in the App component.

üõ†Ô∏è Tip:

If you later change or remove the "learn react" text from your App, this test will fail ‚Äî so you should update the test or the component accordingly.

üß™ Example test setup for your React Native App component:

If you're just getting started and want a basic smoke test (e.g., "Does the app render without crashing?"), here's how you can write a test:

1. ‚úÖ Install the correct testing library: 


npm install --save-dev @testing-library/react-native

or


yarn add --dev @testing-library/react-native

2. ‚úÖ Write a test: App.test.js

import App from '../App'; // Adjust path as needed

describe('App', () => {
  it('renders without crashing', () => {
    const { getByTestId } = render();
    // You can check for a specific screen or element here if known
    expect(getByTestId('root-navigation')).toBeTruthy();
  });
});

3. ‚úÖ Add a testID to something like RootNavigation:

Update your App.js (or RootNavigation component) like this:

This helps the test know what to look for.

‚úÖ How to mock native modules (e.g., Notifee, Firebase, etc.)

In your test setup file or inside the test:


jest.mock('@notifee/react-native', () => ({
    requestPermission: jest.fn(() => Promise.resolve({ authorizationStatus: 1 })),
}));
    
jest.mock('react-native-push-notification-popup', () => 'NotificationPopup');

jest.mock('@react-native-firebase/app', () => ({}));
jest.mock('./src/services/firebase.service', () => ({

    createNotificationListeners: jest.fn(() => jest.fn()),
    firstTimeAskPermission: jest.fn(),
},
}));

  • Replace @testing-library/react with @testing-library/react-native.
