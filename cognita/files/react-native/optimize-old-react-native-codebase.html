Optimize older React Native codebase

If you're working with an older React Native codebase (especially pre–0.65 or without the new architecture), optimization focuses mainly on reducing unnecessary re-renders, improving list performance, and minimizing bridge usage.

Here’s a practical guide tailored for legacy React Native apps:

1️⃣ Optimize Re-renders

✅ Use PureComponent or React.memo

For class components:


class MyComponent extends React.PureComponent { }

For functional components:


export default React.memo(MyComponent);

This prevents unnecessary re-renders when props haven’t changed.

✅ Implement shouldComponentUpdate

For heavy components:


shouldComponentUpdate(nextProps) {
    return nextProps.value !== this.props.value;
}

Avoid deep comparisons — they can be slower than re-rendering.

2️⃣ Optimize FlatList (Very Important)

In older RN versions, list rendering is usually the biggest performance bottleneck.

❌ Avoid:

  • ScrollView for large lists
  • Inline arrow functions inside renderItem
  • Missing keyExtractor

✅ Use:


<FlatList
  data={data}
  renderItem={renderItem}
  keyExtractor={(item) => item.id}
  initialNumToRender={10}
  maxToRenderPerBatch={5}
  windowSize={5}
  removeClippedSubviews={true}
/>

Extra Tips:

  • Use getItemLayout if items have fixed height
  • Memoize renderItem
  • Avoid dynamic inline styles

3️⃣ Avoid Heavy Inline Objects

❌ Bad:


<View style={{ margin: 10 }} />

✅ Good:


const styles = StyleSheet.create({
    container: { margin: 10 }
});

4️⃣ Reduce JS ↔ Native Bridge Calls

Old architecture relies heavily on the bridge, which is slow.

Avoid:

  • Frequent setState calls in loops
  • Excessive logging (console.log)
  • Heavy animations in JS thread

Use:

  • InteractionManager
  • Native-driven animations:

Animated.timing(value, {
    toValue: 1,
    useNativeDriver: true
}).start();

5️⃣ Optimize Images

Use:

  • Proper image sizes
  • Caching libraries (like react-native-fast-image)
  • Avoid huge PNGs

For heavy apps:

  • Preload images
  • Use WebP if supported

6️⃣ Disable Dev Mode in Production

Development mode is MUCH slower.

Always test performance in:


react-native run-android --variant=release

or build a release IPA/APK.

7️⃣ Memory Leak Prevention

Common old RN problems:

  • Unremoved listeners
  • Uncancelled timers
  • EventEmitter leaks

Always clean up:


componentWillUnmount() {
    this.listener?.remove();
}

8️⃣ Use Hermes (If Possible)

If you're on RN 0.64+, enable Hermes — huge JS performance gain on Android.

9️⃣ Avoid Large Redux State Updates

If using Redux:

  • Normalize state
  • Avoid deep nested objects
  • Use selectors (reselect)
  • Connect smaller components instead of entire screens

🔟 Profile Before Optimizing

Use:

  • Flipper (for newer legacy versions)
  • Chrome DevTools profiler
  • why-did-you-render

Don’t optimize blindly.

What is class MyComponent extends React.PureComponent { }

Here’s what that means:

🔹 What is React.PureComponent?

React.PureComponent is a base class provided by React that automatically implements a performance optimization.

It performs a shallow comparison of:

  • this.props
  • this.state

If neither has changed, the component does NOT re-render.

🔹 Normal Component vs PureComponent

Regular component:


class MyComponent extends React.Component {
}
  • Re-renders whenever parent re-renders (by default).

Pure component:


class MyComponent extends React.PureComponent {
}
  • Only re-renders if props or state actually change (based on shallow comparison).

🔹 What is “shallow comparison”?

It checks:

  • Primitive values (number, string, boolean) → compares by value.
  • Objects & arrays → compares by reference (not deep equality).

Example:


this.setState({ count: 1 }) // if count was already 1

A PureComponent will NOT re-render because the value hasn’t changed.

But:


this.setState({ items: [] })

Even if logically empty before, this creates a new array reference, so it WILL re-render.

🔹 When to Use PureComponent

✅ Good for:

  • Performance optimization
  • Components that render the same output for the same props/state
  • Immutable data patterns

❌ Avoid if:

  • You mutate objects instead of creating new ones
  • You rely on deep object comparisons

🔹 Modern React Equivalent

In modern React (with function components), the equivalent optimization is:


export default React.memo(MyComponent);

React.memo does the same shallow comparison for functional components.

🔹 Summary

class MyComponent extends React.PureComponent { } means:

“Create a React class component that automatically skips unnecessary re-renders if props and state haven’t changed.”
