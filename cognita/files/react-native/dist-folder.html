dist folder (short for "distribution")

In the context of a website, the dist folder (short for "distribution") is a directory that contains the final, production-ready version of your website or web application. This is the version that gets deployed to a web server.

Key Points:

  • ✅ Minified and optimized files: The contents are usually minified (smaller in size), often including:
    • HTML
    • CSS (possibly from SASS/SCSS or LESS)
    • JavaScript (often bundled from many modules)
    • Images or other assets (compressed or optimized)
  • ✅ Generated by build tools like:
    • Webpack
    • Vite
    • Rollup
    • Parcel
    • or frameworks like Angular, React, Vue (e.g., using npm run build)
  • ✅ Static files only — no source code, no development tools, just what's needed to run the site in the browser.

Example

Let’s say you have a React project with source code in /src.

When you run:


npm run build

You might get a /dist (or sometimes /build) folder that contains:


dist/
├── index.html
├── main.abc123.js
├── style.abc123.css
├── assets/

These are the files you upload to your server or host on platforms like Netlify, Vercel, or Firebase Hosting.

Summary

The dist folder is the final product of your website or app — clean, efficient, and ready to go live. It's not meant to be edited, only served.
