3. Navigator Options


const option = {
    tabBarComponent: null,
    style: {
        backgroundColor: colours.background.default,
    },
    swipeEnabled: true,
}

tabBarComponent: null

  • Hides the tab bar completely
  • This turns the navigator into a gesture-only container

üëâ No tabs visible

üëâ No labels

üëâ No indicators

This is the key trick.

style.backgroundColor

  • Applies background color to the navigator container
  • Prevents white flashes during transitions
  • Matches your app‚Äôs theme

swipeEnabled: true

  • Enables left/right swipe gestures
  • This is why MaterialTopTabNavigator is used instead of Stack

Perfect for:

  • Onboarding
  • Walkthroughs
  • Intro slides

4. Creating the Navigator


const SplashNavigator = createMaterialTopTabNavigator(screens, option)

This produces a navigator component that:

  • Holds one (or more) onboarding screens
  • Supports swipe gestures
  • Has no visible UI chrome

5. Export


export default SplashNavigator

This is usually used inside a SwitchNavigator or Root Navigator, for example:


createSwitchNavigator({
    Splash: SplashNavigator,
    App: MainNavigator,
})
