Third-Party native modules in React Native

When you're using third-party native modules in React Native, you're incorporating code‚Äîusually written in Java (Android) or Objective-C/Swift (iOS)‚Äîinto your React Native app. These modules allow you to access native device features or performance-optimized code that isn't possible (or is too slow) in JavaScript alone.

üîß What Are Native Modules in React Native?

Native modules are Objective-C, Swift (iOS), or Java/Kotlin (Android) code that you can call from JavaScript. They're typically used when:

  • You need access to platform-specific APIs (e.g., Bluetooth, sensors, background services).
  • You require performance that JavaScript alone can‚Äôt offer.
  • There's an existing SDK or library written in native code that you want to use.

‚úÖ Ways to Use Third-Party Native Modules

1. Using Existing Community Modules

React Native has a huge community that provides native modules via npm. For example:

Feature Library

Then link the native code:

With React Native 0.60+, auto-linking is supported, so manual linking is usually not required.

Usage:

const deviceId = await DeviceInfo.getDeviceId();

2. Writing Your Own Native Module

If no library exists or you need specific functionality, you can create your own:

iOS (Objective-C / Swift):

  • Create a new .m (Objective-C) or .swift file.
  • Expose methods using RCT_EXPORT_METHOD.

Android (Java / Kotlin):

  • Create a class extending ReactContextBaseJavaModule.
  • Annotate methods with @ReactMethod.

React Native bridges the native code and exposes it to JavaScript.

Example: Basic Android Native Module

MyModule.java:


package com.yourapp;

import com.facebook.react.bridge.ReactApplicationContext;
import com.facebook.react.bridge.ReactContextBaseJavaModule;
import com.facebook.react.bridge.ReactMethod;

public class MyModule extends ReactContextBaseJavaModule {
  public MyModule(ReactApplicationContext reactContext) {
    super(reactContext);
  }

  @Override
  public String getName() {
    return "MyModule";
  }

  @ReactMethod
  public void showToast(String message) {
    Toast.makeText(getReactApplicationContext(), message, Toast.LENGTH_SHORT).show();
  }
}

Then access in JS:


import { NativeModules } from 'react-native';

NativeModules.MyModule.showToast('Hello from native!');

üîó What Does ‚ÄúLink the Native Code‚Äù Mean in React Native?

When you install a third-party native module, you're typically pulling in a package that includes native platform code (Objective-C/Swift for iOS, Java/Kotlin for Android) in addition to JavaScript.

‚ÄúLinking‚Äù is the process of:

Connecting the native code of that module to your main React Native app, so your app knows how to use it on iOS and Android.

üß± Why Linking Is Necessary

React Native apps are made of:

  • JavaScript code (runs with React Native)
  • Native code (platform-specific, needs to be compiled with the app)

When you install a native module, the JavaScript part is available right away‚Äîbut the native code needs to be added to your iOS and Android projects. This is what linking does.

üîó How Linking Works

There are two main ways to link:

‚úÖ 1. Auto-Linking (React Native 0.60+)

As of React Native 0.60 and above, linking is done automatically.

What you need to do:


npm install <native-module-name>
npx pod-install # for iOS

‚úÖ That's it. React Native handles the rest.

Example:


npm install react-native-device-info
npx pod-install

üõ† 2. Manual Linking (Pre-0.60 or Edge Cases)

Before RN 0.60 or in special cases, you had to manually edit native files:

üß© Android:

  • Add the module to android/settings.gradle
  • Add to android/app/build.gradle
  • Register the module in MainApplication.java

üçè iOS:

  • Drag the .xcodeproj or .xcframework file into your Xcode project.
  • Add the library to "Linked Frameworks and Libraries."
  • Sometimes adjust Info.plist.

üß† Summary

Term Meaning
Linking Connecting the native parts of a module to your React Native project (so native code runs on the device).
Auto-Linking Happening automatically in RN 0.60+, you just install and run pod install.
Manual Linking You edit native files yourself (only needed in rare cases today).

‚ùó Tip

If a module has native code, and you forget to link (or run pod install), your app will crash or throw an error like:

or
