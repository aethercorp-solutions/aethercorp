In Objective-C, nonatomic is a property attribute that controls how the getter/setter are generated.

nonatomic means:

üîç Full Explanation

When you declare a property like:


@property (nonatomic, strong) NSString *name;

You're telling the compiler:

  • Do NOT add atomic locks around getter/setter
  • This makes access faster
  • But not safe when multiple threads read/write at the same time

If you DO NOT specify nonatomic, the default is atomic:


@property (atomic, strong) NSString *name;   // default

atomic adds locking to ensure:

  • A full, valid value is always returned
  • But it's slower, and still does NOT guarantee thread safety at the app level

üß™ Example

atomic (default)


@property (atomic, strong) NSString *title;

Getter is like:


- (NSString *)title {
    @synchronized(self) {
        return _title;
    }
}

nonatomic (common)


@property (nonatomic, strong) NSString *title;

Getter is basically:


- (NSString *)title {
    return _title;
}

üöÄ Why 99% of iOS Developers Use nonatomic

Because:

  • UIKit is not thread-safe
  • Properties are usually accessed on main thread only
  • nonatomic is faster and avoids unnecessary locking

Apple even recommends using nonatomic for almost all UI-related properties.

atomic does not guarantee full thread safety‚Äîit only ensures a fully formed object is returned.
