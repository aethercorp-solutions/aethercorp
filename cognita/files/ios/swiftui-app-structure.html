SwiftUI app structure

Ah, this is a subtle but important part of SwiftUI app structure. Let’s break it down carefully.

1️⃣ Scene in SwiftUI

  • A Scene represents a unit of UI that can appear on screen.
  • Think of it as a container for your app’s user interface that the system manages.
  • You can have multiple scenes in your app, e.g., multiple windows on iPad or macOS.
  • Every SwiftUI app starts with a Scene in the @main App struct.

Example of a generic Scene:


@main
struct MyApp: App {
    var body: some Scene {
        WindowGroup {
            ContentView()
        }
    }
}
  • Scene is protocol-based, so you can create different types:
    • WindowGroup → for normal app windows (iOS, iPadOS, macOS)
    • DocumentGroup → for document-based apps
    • Settings → for app settings on macOS
    • MenuBarExtra → macOS menu bar apps

2️⃣ WindowGroup in SwiftUI

  • WindowGroup is a specific type of Scene.
  • It creates one or more windows that can each independently display the content you provide.
  • On iPhone, it usually results in one window, but on iPad and macOS, it allows multiple windows.
  • This is how SwiftUI apps support multi-window capabilities automatically.

Example of WindowGroup:


WindowGroup {
    ContentView()
}
  • Inside the closure, you put the root view for that window.
  • Each new window can have its own state independently.

3️⃣ Key Differences

Feature Scene WindowGroup
Concept Abstract container for app UI Concrete scene type for windows
Protocol/Type Scene protocol Struct conforming to Scene
Multi-window support Can support multiple scenes Provides multi-window automatically
Examples WindowGroup, DocumentGroup, Settings Only WindowGroup (for standard windows)

✅ TL;DR:

  • Scene = abstract concept of a UI container in SwiftUI.
  • WindowGroup = concrete implementation of a Scene that manages app windows.

Think:

Scene is the “idea” of a screen; WindowGroup is one “realization” of that idea.
