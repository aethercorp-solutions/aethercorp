1. Breakpoints

Breakpoints pause your app at specific lines so you can inspect variables.

Types:

  • Line Breakpoint ‚Äî click the gutter.
  • Symbolic Breakpoint ‚Äî pause on a method name (e.g., -[UIView setFrame:]).
  • Exception Breakpoint ‚Äî pauses on all Objective-C/Swift exceptions (must-have).
  • Conditional Breakpoint ‚Äî run until a condition is met.
  • Log Breakpoint ‚Äî prints a message instead of stopping execution.

Enable an exception breakpoint:

Breakpoint navigator ‚Üí + bottom left ‚Üí Exception breakpoint.

2. LLDB (Debugger Console)

Lets you inspect and modify at runtime.

Common commands:


po variable          // print object
p variable           // print primitive
bt                   // backtrace (call stack)
frame variable       // list local vars
expr var = newValue  // change variable value at runtime

Examples:


po self.view
po myArray[0]
p (int)[myString intValue]

3. View Debugger (UI Debugging)

Visualizes your UI hierarchy in 3D.

Helps with:

  • Autolayout issues
  • Invisible views intercepting touches
  • Views clipped or offscreen
  • Debugging constraints

Click Debug View Hierarchy (3D cube icon).

4. Memory Graph Debugger

Identifies retain cycles & leaks.

Use when:

  • Something never deallocs
  • Memory increases over time
  • Views/controllers "stick" in memory
