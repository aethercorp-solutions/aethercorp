Authentication in Swift

When building authentication in Swift, especially for iOS apps, you'll typically deal with:

  • User login/logout (email/password, social, biometrics)
  • Secure storage (e.g., Keychain)
  • Networking (sending credentials to server)
  • Token handling (e.g., JWT, OAuth tokens)
  • Protocols for abstraction and testing

‚úÖ Example: Email/Password Authentication with Protocols

  1. Define a Protocol 
    
protocol AuthService {
    func login(email: String, password: String, completion: @escaping (Result) -> Void)
    func logout()
    var currentUser: User? { get }
}
  2. Define Supporting Types 
    
struct User {
    let id: String
    let email: String
}

enum AuthError: Error {
    case invalidCredentials
    case networkError
    case unknown
}
  3. Implement the Protocol 
    
class FirebaseAuthService: AuthService {
    private var user: User?

    func login(email: String, password: String, completion: @escaping (Result) -> Void) {
        // Simulated network request
        DispatchQueue.global().asyncAfter(deadline: .now() + 1) {
            if email == "test@example.com" && password == "password123" {
                let user = User(id: "abc123", email: email)
                self.user = user
                completion(.success(user))
            } else {
                completion(.failure(.invalidCredentials))
            }
        }
    }

    func logout() {
        user = nil
    }

    var currentUser: User? {
        return user
    }
}
  4. Use the AuthService via Dependency Injection 
    
class LoginViewModel {
    private let authService: AuthService

    init(authService: AuthService) {
        self.authService = authService
    }

    func login(email: String, password: String) {
        authService.login(email: email, password: password) { result in
            switch result {
            case .success(let user):
                print("Logged in as \(user.email)")
            case .failure(let error):
                print("Login failed: \(error)")
            }
        }
    }
}

    Usage:

    
let authService = FirebaseAuthService()
let viewModel = LoginViewModel(authService: authService)
viewModel.login(email: "test@example.com", password: "password123")

  6. üß™ Why Use Protocols for Authentication?
    • Easier to mock during testing
    • Enables multiple implementations (e.g., FirebaseAuthService, MockAuthService)
    • Encourages clean architecture (decouples view models from specific services)
