AVCaptureMetadataOutputObjectsDelegate callback

That method is the AVCaptureMetadataOutputObjectsDelegate callback. It is called every time the capture session detects metadata objects (e.g. barcodes, QR codes, faces) in the video stream.

What this method is


- (void)captureOutput:(AVCaptureOutput *)captureOutput
didOutputMetadataObjects:(NSArray *)metadataObjects
     fromConnection:(AVCaptureConnection *)connection

This is invoked by AVCaptureMetadataOutput, not by the camera directly.

When it is called

It is called when ALL of the following are true:

  1. AVCaptureSession is running
  2. AVCaptureMetadataOutput is added to the session
  3. metadataObjectTypes contains supported types (e.g. QR, EAN, Code128)
  4. The camera sees something matching those types

üëâ It may be called many times per second while the barcode stays visible.

What each parameter means

captureOutput

Usually:


AVCaptureMetadataOutput *

You can check:


if ([captureOutput isKindOfClass:[AVCaptureMetadataOutput class]]) { ... }

metadataObjects

An array of detected metadata objects.

Most common class:


AVMetadataMachineReadableCodeObject

If no barcode is detected, this array can be empty.

connection

The AVCaptureConnection between the input (camera) and output.

Usually not needed unless you manage orientation or video mirroring.

Typical barcode handling pattern


- (void)captureOutput:(AVCaptureOutput *)captureOutput didOutputMetadataObjects:(NSArray *)metadataObjects fromConnection:(AVCaptureConnection *)connection
{
    if (metadataObjects.count == 0) {
        return;
    }

    AVMetadataMachineReadableCodeObject *code = (AVMetadataMachineReadableCodeObject *)metadataObjects.firstObject;

    if (![code isKindOfClass:[AVMetadataMachineReadableCodeObject class]]) {
        return;
    }

    NSString *value = code.stringValue;
    if (!value) return;

    NSLog(@"Barcode detected: %@", value);

    // IMPORTANT: prevent repeated scans
    [self.captureSession stopRunning];
}

Mapping barcode to preview coordinates (very common)

If you use AVCaptureVideoPreviewLayer:


AVMetadataObject *transformed = [self.previewLayer transformedMetadataObjectForMetadataObject:code];

This converts camera-space coordinates ‚Üí screen-space coordinates.

Why this method fires repeatedly

By design:

  • The camera keeps detecting the same barcode every frame
  • You must manually block duplicates

Common solutions:

  • Stop the session
  • Use a boolean flag
  • Debounce with a timestamp
  • Track scanned values

Example:


if (self.isProcessingScan) return;
self.isProcessingScan = YES;

Required setup (often missed)


AVCaptureMetadataOutput *metadataOutput = [[AVCaptureMetadataOutput alloc] init];
[self.captureSession addOutput:metadataOutput];

[metadataOutput setMetadataObjectsDelegate:self
                                     queue:dispatch_get_main_queue()];

metadataOutput.metadataObjectTypes = @[
    AVMetadataObjectTypeQRCode,
    AVMetadataObjectTypeEAN13Code,
    AVMetadataObjectTypeCode128Code
];

Threading behavior

  • Runs on the queue you provided
  • If you passed dispatch_get_main_queue(), UI updates are safe
  • Heavy work should be dispatched to a background queue
