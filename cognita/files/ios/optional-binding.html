Optional Binding

In Swift, if let is a common construct used for optional binding. It safely unwraps an optional value — that is, it checks if the optional contains a value, and if it does, it assigns that value to a new constant or variable, which you can then use inside the if block.

Basic Syntax


if let unwrapped = optionalValue {
    // Use 'unwrapped' here — it's guaranteed to be non-nil
} else {
    // The optional was nil
}

Example


var name: String? = "Alice"

if let actualName = name {
    print("Hello, \(actualName)!")
} else {
    print("No name provided.")
}

Explanation:

  • name is an optional String?.
  • if let actualName = name tries to unwrap name.
  • If name contains a non-nil value ("Alice"), it's unwrapped and assigned to actualName, and the if block runs.
  • If name is nil, the else block runs.

Optional Binding with Multiple Values

You can unwrap multiple optionals in a single if let:


let firstName: String? = "John"
let lastName: String? = "Doe"

if let f = firstName, let l = lastName {
    print("Full name: \(f) \(l)")
}

Optional Binding with guard let

Sometimes, especially in functions, guard let is preferred for early exit:


func greet(name: String?) {
    guard let unwrappedName = name else {
        print("No name")
        return
    }
    print("Hello, \(unwrappedName)")
}
