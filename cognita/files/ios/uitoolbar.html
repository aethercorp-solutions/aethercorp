UIToolbar

An iOS UIToolbar is a standard UIKit component used to display a row of action buttons (and other controls) that let users perform tasks related to the current screen.

Think of it as a horizontal action bar that usually appears at the top or bottom of the screen.

What UIToolbar is used for

A UIToolbar typically contains:

  • Buttons (UIBarButtonItem)
  • Flexible / fixed spaces
  • System items (Done, Cancel, Add, Edit, etc.)
  • Custom views (like UIButton, UISegmentedControl, or UITextField)

Common use cases:

  • Editing actions (Save, Delete, Share)
  • Navigation-related actions
  • Keyboard accessory toolbars (e.g., “Done” button above the keyboard)

Where it appears

  • Bottom of the screen (most common)
  • Top of the screen (less common)
  • Above the keyboard as an inputAccessoryView

Key characteristics

  • Holds UIBarButtonItem, not UIButton directly
  • Automatically adapts to:
    • Light / Dark Mode
    • Device size (iPhone / iPad)
    • Rotation
  • Supports system icons and text styles
  • Can be translucent or opaque

Basic example (Objective-C)


UIToolbar *toolbar = [[UIToolbar alloc] init];
toolbar.frame = CGRectMake(0, 0, self.view.frame.size.width, 44);

UIBarButtonItem *done =
    [[UIBarButtonItem alloc] initWithBarButtonSystemItem:UIBarButtonSystemItemDone
                                                   target:self
                                                   action:@selector(doneTapped)];

UIBarButtonItem *flex =
    [[UIBarButtonItem alloc] initWithBarButtonSystemItem:UIBarButtonSystemItemFlexibleSpace
                                                   target:nil
                                                   action:nil];

toolbar.items = @[flex, done];

[self.view addSubview:toolbar];

Toolbar vs Navigation Bar

Feature UIToolbar UINavigationBar
Purpose Actions Navigation
Location Top or bottom Top only
Items UIBarButtonItem Back button, title
Stack-based ❌ ✅

Common properties


toolbar.hidden = YES;           // hide/show
toolbar.translucent = YES;      // blur effect
toolbar.barTintColor = UIColor.blackColor;
toolbar.tintColor = UIColor.whiteColor;

Important notes

  • Hiding a button’s custom view does NOT hide the UIBarButtonItem itself
  • To hide toolbar content, you usually:
    • Hide the toolbar
    • Remove or replace its items
  • On modern iOS (iOS 15+), appearance is controlled via:
    • UIToolbarAppearance

When to use UIToolbar

Use it when:

  • You need contextual actions
  • Navigation is NOT the main purpose
  • Actions should be easily accessible

Avoid it when:

  • You only need navigation (use UINavigationBar)
  • Actions are better placed inline in the UI
