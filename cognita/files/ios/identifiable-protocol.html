Identifiable Protocol

In Swift, an identifiable protocol typically refers to the Identifiable protocol from the Swift standard library, which is commonly used to uniquely identify instances of a type—especially in contexts like SwiftUI lists, ForEach loops, or data binding.

✅ Definition of Identifiable

Here's how the protocol is defined:


public protocol Identifiable {
    associatedtype ID: Hashable
    var id: ID { get }
}

✅ How to Use Identifiable

You conform a struct or class to Identifiable by providing an id property that uniquely identifies each instance.

Example:


struct User: Identifiable {
    var id: UUID
    var name: String
}

Or if you want a simpler ID, like an integer:


struct Product: Identifiable {
    var id: Int
    var name: String
}

You can even make id a computed property:


struct Book: Identifiable {
    var title: String
    var author: String
    var id: String {
        "\(title)-\(author)"
    }
}

✅ Why Use Identifiable?

  • SwiftUI uses it extensively in views like List and ForEach.
  • It helps SwiftUI track and update views efficiently.
  • It provides clarity and consistency in how unique instances are handled in collections.

Example in SwiftUI:


struct ContentView: View {
    let users: [User] = [
        User(id: UUID(), name: "Alice"),
        User(id: UUID(), name: "Bob")
    ]

    var body: some View {
        List(users) { user in
            Text(user.name)
        }
    }
}

Here, SwiftUI uses the id from the Identifiable conformance to track each row.
