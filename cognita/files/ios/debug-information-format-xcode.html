What is Debug Information Format in Xcode?

The Debug Information Format setting in Xcode controls how and where debug symbols are generated when you build your app. These symbols are critical for:

  • Debugging in Xcode (stepping through code, inspecting variables)
  • Symbolicating crash logs
  • Profiling with Instruments

üîß Location in Xcode

  • Go to Build Settings
  • Search for: Debug Information Format
  • You'll see this under both Project and Target settings

üß© Available Debug Information Formats

Format Technical Name What It Does Use Case
None none No debug info generated. Smallest binary size, but no debugging.
DWARF dwarf Embeds debug symbols directly in the binary. OK for local debugging, but not ideal for symbolication.
DWARF with dSYM File dwarf-with-dsym Generates a separate .dSYM bundle containing full symbol information. ‚úÖ Best for production builds and symbolication.

üß† Why This Setting Matters

  1. Symbolication of crash logs
    • Without proper symbols (like .dSYM), crash logs will show raw memory addresses.
    • With them, you see readable function names and line numbers.
  2. Debugging in Xcode
    • Enables breakpoints, stack traces, and variable inspection.
  3. Performance
    • Including symbols in the binary (DWARF) can bloat app size.
    • Separating them (DWARF with dSYM File) keeps the app smaller and cleaner for release.

üîó What is the relationship between Debug Information Format, .dSYM files, and Firebase Crashlytics?

Firebase Crashlytics helps you track and analyze crashes in your iOS app ‚Äî but to make those crash reports human-readable, it needs debug symbols.

üß† Crashlytics Needs Your dSYM Files

When a crash occurs on a user‚Äôs device, the crash log will contain memory addresses instead of function names (e.g., 0x103a6f2b8).

To symbolicate (translate) those addresses into readable code (e.g., MyViewController.swift:42), Crashlytics needs the .dSYM files that match the app version.

üîÅ How It Works

  1. You build your app with Xcode
    • If Debug Information Format is set to DWARF with dSYM File, Xcode creates a .dSYM file.
  2. You archive and upload the app (e.g., to TestFlight or App Store)
  3. You upload the .dSYM files to Crashlytics
    • Crashlytics matches the .dSYM with the build UUID.
    • When a crash happens, Crashlytics symbolicates the stack trace using those symbols.

  2. Enable Automatic Upload (Recommended)
