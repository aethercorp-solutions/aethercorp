end-to-end explanation of Apple’s code-signing tools

Below is a clear, end-to-end explanation of Apple’s code-signing tools, how they work together, why each exists, and a practical example (from “no app” → “signed app ready to install”). I’ll keep it technical but structured so you can reason about it like a system.

🍎 Apple Code Signing — Complete Guide (Tools + Examples)

Why Apple Requires Code Signing (Big Picture)

Apple code signing ensures:

  1. Identity – Who built this app?
  2. Integrity – Has the app been modified?
  3. Authorization – Is this app allowed to run on this device?
  4. Entitlements – What is this app allowed to do?

Every Apple platform app must be signed to run on a real device or be distributed.

Core Code-Signing Components (Mental Model)

Think of code signing as a chain of trust:


Apple Root CA
   ↓
Apple Developer Certificate
   ↓
App Signature
   ↓
Device Trust (Provisioning Profile)

🔐 1. Certificates (Identity)

Tool: Keychain Access

(macOS GUI)

Certificates prove who you are.

Types of Apple Certificates

Certificate Type Used For
Apple Development Running on devices
Apple Distribution App Store / TestFlight
Developer ID Application macOS apps outside App Store

📍 Stored in Keychain

Example: View Certificates


security find-identity -v -p codesigning

Output:


1) XXXXXXXXXX "Apple Development: Paul Dionisio (ABCDE12345)"
2) YYYYYYYYYY "Apple Distribution: Paul Dionisio (ABCDE12345)"

📄 2. App ID (Bundle Identity)

Tool: Apple Developer Portal

An App ID is:


TeamID + Bundle ID

Example:


ABCDE12345.com.trycastanet.app

This ID links:

  • App
  • Entitlements
  • Provisioning profile

📦 3. Provisioning Profiles (Permission Slip)

Tool:

  • Xcode
  • Apple Developer Portal
  • profiles (legacy)
  • Manual install: ~/Library/MobileDevice/Provisioning Profiles

A provisioning profile says:

“This app, signed by this certificate, may run on these devices, with these entitlements.”

Types

Profile Type Purpose
Development Local testing
Ad Hoc Limited external testing
App Store App Store distribution
Enterprise Internal company apps

Inspect a Provisioning Profile


security cms -D -i MyProfile.mobileprovision

✍️ 4. Codesign Tool (The Signer)

Tool: codesign

(CLI — core signing engine)

This tool:

  • Hashes the app
  • Encrypts hash with your private key
  • Embeds entitlements

Example: Sign an App Manually


codesign \
  --sign "Apple Development: Paul Dionisio (ABCDE12345)" \
  --entitlements entitlements.plist \
  --deep \
  MyApp.app

Verify Signature


codesign -dv --verbose=4 MyApp.app

📜 5. Entitlements (Capabilities)

Tool:

  • Xcode
  • .entitlements file

Entitlements are permissions like:

  • Push notifications
  • iCloud
  • App Groups
  • Sign in with Apple

Example:


<key>com.apple.developer.push-notifications</key>
<string>development</string>

Entitlements must match the provisioning profile.

🛠 6. Xcode (Automation Layer)

Xcode handles:

  • Certificate creation
  • Profile generation
  • Signing
  • Entitlements
  • Device registration

When you enable:


Automatically manage signing ✅

Xcode silently does:

  • codesign
  • profile matching
  • entitlement merging

📦 7. xcodebuild (CI/CD Tool)

Used in:

  • GitHub Actions
  • Fastlane
  • Jenkins

Example: Build + Sign from CLI


xcodebuild \
  -workspace MyApp.xcworkspace \
  -scheme MyApp \
  -configuration Release \
  -archivePath build/MyApp.xcarchive \
  archive

Then export:


xcodebuild \
  -exportArchive \
  -archivePath build/MyApp.xcarchive \
  -exportOptionsPlist ExportOptions.plist \
  -exportPath build/

📤 8. altool / notarytool (Distribution)

Tool:

  • notarytool (modern)
  • altool (legacy)

Used for:

  • App Store uploads
  • macOS notarization

Example (macOS notarization):


xcrun notarytool submit MyApp.zip \
  --apple-id you@email.com \
  --team-id ABCDE12345 \
  --password app-specific-password

🔍 9. spctl (Gatekeeper Check)

Tool: spctl

Checks if macOS trusts the app.


spctl -a -vv MyApp.app

🧪 Full Practical Flow Example

Goal: Run iOS App on iPhone

1️⃣ Create Apple Development Certificate

2️⃣ Create App ID

3️⃣ Register iPhone UDID

4️⃣ Create Development Provisioning Profile

5️⃣ Build app

6️⃣ codesign embeds certificate + entitlements

7️⃣ iOS verifies:

  • signature
  • profile
  • device
    • 8️⃣ App launches ✅

🧠 Common Errors Explained

Error Meaning
No matching provisioning profiles Profile missing or wrong
Code object is not signed codesign failed
Entitlements mismatch App > Profile conflict
A valid provisioning profile for this executable was not found Wrong cert/profile
