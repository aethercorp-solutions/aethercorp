High-Order Functions

In Swift, map, flatMap, compactMap, and reduce are high-order functions commonly used on collections (like arrays). They each serve different purposes. Here's a breakdown of what they do and how they differ:

map

Purpose: Transforms each element of a collection.

  • Returns a new array containing the results of applying a transformation to each element.
  • Keeps the same number of elements as the original array.

Example:


let numbers = [1, 2, 3]
let squared = numbers.map { $0 * $0 }
// Result: [1, 4, 9]

compactMap

Purpose: Transforms each element and removes nil values.

  • Applies a transformation that returns an optional.
  • Removes any nil results, returning a non-optional array.

Example:


let strings = ["1", "two", "3"]
let numbers = strings.compactMap { Int($0) }
// Result: [1, 3] — "two" is skipped

flatMap (for collections)

Purpose: Flattens a collection of collections into a single array.

  • Useful when each transformation returns a collection and you want a flat result (no nested arrays).

Example 1 – transforming with flattening:


let data = [[1, 2, 3], [4, 5], [6]]
let flat = data.flatMap { $0 }
// Result: [1, 2, 3, 4, 5, 6]

Example 2 – optional values (older Swift versions):

In earlier Swift versions, flatMap also removed nils (similar to compactMap now).

reduce

Purpose: Combines all elements of a collection into a single value.

  • Takes an initial value and a closure that combines each element into a running result.

let numbers = [1, 2, 3, 4]
let sum = numbers.reduce(0) { $0 + $1 }
// Result: 10

Another example – string concatenation:


let words = ["Swift", "is", "fun"]
let sentence = words.reduce("") { $0 + " " + $1 }
// Result: " Swift is fun"
