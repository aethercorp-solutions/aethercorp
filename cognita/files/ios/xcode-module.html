Xcode Module

In Xcode, a module is essentially a self-contained unit of code that can be imported and used in other parts of your project or other projects. Understanding how modules work in an Xcode project helps you better organize, reuse, and encapsulate code.

In Swift and Objective-C, a module represents a namespace and a binary unit of code distribution. You can think of it as:

ðŸ”¹ Types of Modules in Xcode Projects

There are several kinds of modules:

  1. App Target (main module)

    The main app target is also a module. You can import it in other targets like test targets:

    
@testable import MyAppModule
  2. Framework Target

    A common way to create reusable modules. You can import the framework into other apps or targets:

    
import MyFramework
  3. Swift Package

    Swift Packages are modular by design. Each package defines its own module(s), and you can import them easily:

    
import MyPackageModule
  4. Objective-C Modules / Bridging

    Objective-C modules (like UIKit) are also imported with import UIKit in Swift, thanks to the Clang module system.

ðŸ”¹ How Modules Work in an Xcode Project

  1. Targets as Modules

    Each target in Xcode can be treated as a module:

    • Targets define what to build.
    • Each target has its own product name, which becomes the module name.
    • Example: If your target is named MyLibrary, then: 
      
import MyLibrary
  2. Module Interface

    A module defines its public interface:

    • Only public or open declarations are accessible from outside the module.
    • You must set the correct access control modifiers (e.g., internal, public, open) to control visibility across modules.

ðŸ”¹ How to Create a Module in Xcode

ðŸ”§ Creating a Framework (as a module):

  1. Open your project.
  2. Add a new target: File > New > Target.
  3. Choose Framework or Static Library (depending on use case).
  4. Give it a name (e.g., MyFramework).
  5. Move shared code into that target.
  6. Make sure the code is marked public.
  7. Import the module in your main app target: 
    
import MyFramework

ðŸ”¹ Special Notes

  • Bridging Between Swift and Objective-C uses a bridging header or module map.
  • Module Maps are used in Clang-based languages (C/C++) for creating modules manually.
  • Swift Module Interface Files (.swiftinterface) are used in Swift for ABI stability and module distribution.
