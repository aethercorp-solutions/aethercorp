üîπ What is UISceneDelegate?

UISceneDelegate is a protocol introduced in iOS 13 to support multi-window (multi-scene) functionality, especially useful in iPadOS and modern iOS applications. It works alongside AppDelegate to manage UI lifecycle per scene (window).

  • Each scene (i.e., instance of your app's UI) has a SceneDelegate.
  • Handles scene lifecycle events like:
    • Scene creation
    • Activation/deactivation
    • Entering background/foreground
  • Separates UI responsibilities from application-level tasks (AppDelegate).

üîß How to Use UISceneDelegate

Used only if your app supports scenes ‚Äî which is the default for UIKit apps starting with iOS 13.

üß± Typical SceneDelegate.swift Structure


class SceneDelegate: UIResponder, UIWindowSceneDelegate {

    var window: UIWindow?

    // Called when the scene connects to the app session
    func scene(_ scene: UIScene,
                willConnectTo session: UISceneSession,
                options connectionOptions: UIScene.ConnectionOptions) {
        
        guard let windowScene = scene as? UIWindowScene else { return }

        // Create the UIWindow with the windowScene
        let window = UIWindow(windowScene: windowScene)
        window.rootViewController = MyInitialViewController()  // or a UINavigationController, etc.
        self.window = window
        window.makeKeyAndVisible()
    }

    // Called when the scene becomes active
    func sceneDidBecomeActive(_ scene: UIScene) {
        // Resume tasks, animations
    }

    // Called when the scene resigns active state (e.g., phone call)
    func sceneWillResignActive(_ scene: UIScene) {
        // Pause tasks, disable timers
    }

    // Scene is entering the background
    func sceneDidEnterBackground(_ scene: UIScene) {
        // Save data, release shared resources
    }

    // Scene is coming back to foreground
    func sceneWillEnterForeground(_ scene: UIScene) {
        // Undo changes made when entering background
    }

    // Scene was disconnected (can be used for cleanup)
    func sceneDidDisconnect(_ scene: UIScene) {
        // Called when scene is released by the system
    }
}

‚öôÔ∏è Required Setup in Info.plist

To use SceneDelegate, your Info.plist must include a scene configuration:


<key>UIApplicationSceneManifest</key>
<dict>
    <key>UIApplicationSupportsMultipleScenes</key>
    <true/>
    <key>UISceneConfigurations</key>
    <dict>
        <key>UIWindowSceneSessionRoleApplication</key>
        <array>
            <dict>
                <key>UISceneConfigurationName</key>
                <string>Default Configuration</string>
                <key>UISceneDelegateClassName</key>
                <string>$(PRODUCT_MODULE_NAME).SceneDelegate</string>
            </dict>
        </array>
    </dict>
</dict>

üîÅ Interaction with AppDelegate

Your AppDelegate still exists and handles global application events (e.g., push notifications, app launch).

But UI-specific setup (like creating windows) moves to SceneDelegate.

Example in AppDelegate.swift:


func application(
    _ application: UIApplication,
    configurationForConnecting connectingSceneSession: UISceneSession,
    options: UIScene.ConnectionOptions
) -> UISceneConfiguration {
    return UISceneConfiguration(name: "Default Configuration", sessionRole: connectingSceneSession.role)
}

üéØ When Should You Use UISceneDelegate?

Use UISceneDelegate if:

  • You're supporting iOS 13 or later.
  • Your app may support multiple windows (iPad multitasking).
  • You want better separation of concerns between UI and app logic.
