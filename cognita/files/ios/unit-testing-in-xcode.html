‚úçÔ∏è How to Write a Simple Unit Test in Swift

Assume you have a function in your app:


// Calculator.swift
class Calculator {
    func add(_ a: Int, _ b: Int) -> Int {
        return a + b
    }
}

Create a test case for it:


// CalculatorTests.swift
import XCTest
@testable import YourProjectName

class CalculatorTests: XCTestCase {

    func testAddition() {
        let calc = Calculator()
        let result = calc.add(2, 3)
        XCTAssertEqual(result, 5, "Expected 2 + 3 to equal 5")
    }
}

üß™ Useful XCTest Assertions

Assertion Description
XCTAssertEqual(a, b) Passes if a == b
XCTAssertTrue(condition) Passes if condition is true
XCTAssertNil(value) Passes if value is nil
XCTAssertThrowsError() Passes if an error is thrown
XCTFail() Always fails the test

üìå Best Practices

  • Use descriptive test method names like testLoginFailsWithWrongPassword()
  • Keep tests small and focused on a single behavior
  • Avoid relying on UI or network in unit tests (use mocks/stubs instead)
  • Run tests frequently, especially after changes
