Reference an Image File

In a React web app, how you reference an image file depends on where the image is located in your project.

Here’s a quick guide for all common cases 👇

🗂 1. Image in public/ folder

Use a relative URL path (not an import).

✅ Example:


<img src="/images/logo.png" alt="Logo" />

📁 Folder structure:


public/
 └── images/
     └── logo.png
src/
 └── App.js

💡 The path /images/logo.png maps directly to public/images/logo.png.

📦 2. Image in src/ folder

You must import it first — React will bundle and optimize it.

✅ Example:


import logo from './assets/logo.png';

function App() {
  return <img src={logo} alt="Logo" />;
}

📁 Folder structure:


src/
 ├── assets/
 │    └── logo.png
 └── App.js

💡 Webpack (or Vite, CRA, etc.) handles the import and generates a hashed URL automatically.

🌐 3. External URL

Just use the full link:

✅ Example:


<img src="https://example.com/logo.png" alt="Logo" />

⚠️ Common mistakes to avoid

  • ❌ Don’t use relative paths like "../public/images/logo.png" — public files aren’t accessible that way.
  • ✅ Always use /images/... for public or import for src.
