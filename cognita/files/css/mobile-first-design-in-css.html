Mobile-first design in CSS

Mobile-first design in CSS means you design your website starting with styles for the smallest (mobile) screens, and then use media queries to progressively enhance the layout for tablets, laptops, and desktops.

This is a best practice because it ensures your site is accessible and fast for mobile users first, who often have limited bandwidth and smaller screens.

ðŸ”§ Example: Mobile-First CSS


/* Base styles: Mobile first */
body {
  font-family: sans-serif;
  font-size: 16px;
  margin: 0;
  padding: 1rem;
  background-color: #f4f4f4;
}

.container {
  display: flex;
  flex-direction: column;
  gap: 1rem;
}

.card {
  background: white;
  padding: 1rem;
  border-radius: 8px;
  box-shadow: 0 2px 5px rgba(0,0,0,0.1);
}

/* Tablet styles (min-width: 600px) */
@media (min-width: 600px) {
  .container {
    flex-direction: row;
    flex-wrap: wrap;
  }

  .card {
    flex: 1 1 calc(50% - 1rem);
  }
}

/* Desktop styles (min-width: 1024px) */
@media (min-width: 1024px) {
  .card {
    flex: 1 1 calc(33.33% - 1rem);
  }
}

1rem is a CSS unit of measurement used in web development to set sizes (like fonts, spacing, margins, etc.). It stands for "root em".

What does 1rem mean?

  • 1rem equals the font size of the root element ( tag).
  • By default, most browsers set the root font size to 16px, so:
    • 1rem = 16px (unless the default is changed)

Example:


html {
    font-size: 16px; /* Default */
}
    
p {
    font-size: 1rem; /* 16px */
}

If you changed the root font size like this:


html {
    font-size: 18px;
}

Then:


p {
    font-size: 1rem; /* 18px */
}

Why use rem instead of px?

  • Scalability: It's easier to make your design responsive and accessible by adjusting just the root font size.
  • Consistency: All elements scale based on one central value.
  • Accessibility: Users can change base font size in browser settings, and your site will respond correctly.

Here's a comparison between rem, em, px, and % with clear examples and use cases.

rem Root em Root element's font size (html) Scalable, consistent sizing
% Percentage Parent element's size Relative layouts

Goal Use
Responsive typography rem

ðŸ”¹ Summary

  • Use rem for consistent, root-based scaling.
  • Use em for scaling based on parent.
  • Use px when you need precise control.
  • Use % mainly for layouts (like width/height).
