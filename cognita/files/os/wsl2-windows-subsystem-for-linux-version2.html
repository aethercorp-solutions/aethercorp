WSL 2 (Windows Subsystem for Linux version 2)

WSL 2 (Windows Subsystem for Linux version 2) is a compatibility layer developed by Microsoft that allows you to run a full Linux distribution on Windows, without the need for a virtual machine or dual-boot setup. It's particularly popular among developers who want to use Linux tools while working in a Windows environment.

üîç Key Features of WSL 2:

Feature Description
Full Linux kernel WSL 2 uses a real Linux kernel built by Microsoft, improving compatibility with Linux applications.
Faster performance File system operations are much faster than in WSL 1, especially when accessing Linux files.
Full system call compatibility Supports more applications (e.g., Docker) because it handles system calls like a traditional Linux kernel.
Lightweight Much lighter than using a full virtual machine.
Integration with Windows You can access Windows files from Linux and vice versa (/mnt/c for example).

üöÄ How to Install WSL 2:

  1. Enable WSL and Virtual Machine Platform: 
    
dism.exe /online /enable-feature /featurename:Microsoft-Windows-Subsystem-Linux /all /norestart
dism.exe /online /enable-feature /featurename:VirtualMachinePlatform /all /norestart
  2. Restart your computer.
  3. Install WSL 2 as the default version: 
    
wsl --set-default-version 2
  4. Install a Linux distribution:
    • Open Microsoft Store.
    • Search for a distro (e.g., Ubuntu, Debian, Kali Linux).
    • Click install.
  5. Set the installed distro to use WSL 2 (if needed): 
    
wsl --set-version <distro-name> 2

    Example:

    
wsl --set-version Ubuntu 2

Docker Desktop uses WSL 2 for its backend. You can install Docker Desktop on Windows and enable WSL 2 integration, allowing you to run Linux containers natively on Windows.
