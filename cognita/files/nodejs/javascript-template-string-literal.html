JavaScript Template String Literal

A JavaScript template string literal (also known as a template literal) is a way to define strings that allows embedded expressions, multi-line strings, and improved readability.

ðŸ”¹ Syntax

Template literals use backticks ` instead of single ' or double " quotes.


const name = "Alice";
const greeting = `Hello, ${name}!`;
console.log(greeting); // Output: Hello, Alice!

ðŸ”¹ Key Features & Usage

  1. String Interpolation (Expression Embedding)

    You can embed variables or expressions inside a string using ${...}.

    
const a = 5;
const b = 10;
console.log(`The sum of ${a} and ${b} is ${a + b}.`);
// Output: The sum of 5 and 10 is 15.
  2. Multi-line Strings

    Template literals preserve newlines and whitespace.

    
const multiLine = `This is
a string that spans
multiple lines.`;
console.log(multiLine);
  3. Function Calls Inside Expressions

    You can run functions inside ${...}.

    
function getUser() {
    return "Bob";
}
console.log(`Welcome, ${getUser()}!`);
// Output: Welcome, Bob!
  4. Tagged Template Literals

    You can use a function (tag) to process the template string.

    
function emphasize(strings, ...values) {
    return strings.reduce((result, str, i) => result + str + (values[i]?.toUpperCase() || ""), "");
}
    
const name = "jane";
console.log(emphasize`Hello, ${name}!`);
// Output: Hello, JANE!
  5. HTML Templates (e.g., for front-end frameworks)

    You can build HTML fragments easily:

    
const title = "Welcome";
const html = `
  <div>
    <h1>${title}</h1>
</div>
`;
