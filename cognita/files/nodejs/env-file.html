.env file

A .env file is a plain text file used to store environment variables for an application, typically during development.

These variables can include things like:

  • API keys
  • Database URLs
  • Secret tokens
  • Configuration options (e.g., DEBUG=true, PORT=3000)

Purpose of .env

It helps separate configuration from your application code, making it easier to manage different environments (development, testing, production).

Example .env file


DATABASE_URL=postgres://user:pass@localhost:5432/mydb
API_KEY=abc123secret
DEBUG=true
PORT=8000

How itâ€™s used

In most programming environments, especially in web development (Node.js, Python, etc.), a library is used to load these variables from the .env file into the system's environment so that your app can use them.

For example:

  • Node.js: Uses the dotenv package 
    
require('dotenv').config();
console.log(process.env.API_KEY);
  • Python: Uses packages like python-dotenv 
    
from dotenv import load_dotenv
import os

load_dotenv()
api_key = os.getenv("API_KEY")

Best practices

  • Do not commit .env files to version control (e.g., Git). Use .gitignore.
  • Use different .env files for different environments (e.g., .env.production, .env.local).
  • Keep sensitive credentials secure.
