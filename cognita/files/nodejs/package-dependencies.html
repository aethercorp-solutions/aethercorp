devDependencies

In a Node.js (JavaScript) project using npm or yarn, devDependencies are packages that are only needed during development, not in production.

Where You See Them

In package.json:


{
    "devDependencies": {
        "eslint": "^8.50.0",
        "webpack": "^5.89.0",
        "jest": "^29.7.0"
    }
}

Purpose

devDependencies typically include tools like:

  • Build tools (e.g. Webpack, Vite, Babel)
  • Linters (e.g. ESLint, Prettier)
  • Testing frameworks (e.g. Jest, Mocha)
  • Type checkers (e.g. TypeScript)
  • Code formatters (e.g. Prettier)

Installing a Package as a Dev Dependency

Use one of the following commands:


npm install --save-dev <package-name>
# or
yarn add --dev <package-name>

ESLint

ESLint is a powerful tool used to analyze, catch, and fix problems in JavaScript or TypeScript code, especially related to code quality, potential bugs, and coding style.

üß† What It Does

ESLint lints your code ‚Äî that is, it checks your code against a set of rules and reports problems (and sometimes fixes them).

It helps with:

  • Catching common bugs
  • Enforcing consistent code style
  • Preventing bad practices
  • Integrating with Prettier to auto-format code

üì¶ Installing ESLint

To install as a devDependency:


npm install --save-dev eslint
# or
yarn add --dev eslint

‚öôÔ∏è Setup

To set up ESLint in your project:


npx eslint --init

This interactive CLI will ask:

  • What type of project? (JavaScript, TypeScript, React, etc.)
  • Which style guide? (Airbnb, Standard, etc.)
  • Use TypeScript? Babel?
  • Format code with Prettier?
  • Where to store config? (.eslintrc file, JSON, YAML, or JS)

This generates a config file like .eslintrc.json or .eslintrc.js.

üß™ Example ESLint Config

.eslintrc.json:


{
    "env": {
        "browser": true,
        "es2021": true,
        "node": true
    },
    "extends": [
        "eslint:recommended",
        "plugin:prettier/recommended"
    ],
    "parserOptions": {
        "ecmaVersion": "latest",
        "sourceType": "module"
    },
    "rules": {
        "no-unused-vars": "warn",
        "no-console": "off"
    }
}

üõ†Ô∏è Usage

To lint your files:


npx eslint .              # Check all files
npx eslint . --fix        # Automatically fix problems

ü§ù ESLint + Prettier

To make ESLint and Prettier work together:

  1. Install required packages: 
    
npm install --save-dev eslint-config-prettier eslint-plugin-prettier prettier
  2. Extend config: 
    
"extends": ["plugin:prettier/recommended"]

This disables ESLint rules that conflict with Prettier and lets ESLint run Prettier as a plugin.
