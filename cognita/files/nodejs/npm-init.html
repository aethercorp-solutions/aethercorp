npm init -y

Running npm init -y is a quick way to create a package.json file in your current directory with all the default settings.

Here’s what it does:

  • Initializes a new Node.js project.
  • Creates a package.json file with default values.
  • Skips the interactive questionnaire and answers “yes” to all prompts automatically.

Here’s a quick rundown of what the default package.json looks like after running npm init -y:


{
    "name": "your-folder-name",
    "version": "1.0.0",
    "description": "",
    "main": "index.js",
    "scripts": {
        "test": "echo \"Error: no test specified\" && exit 1"
    },
    "keywords": [],
    "author": "",
    "license": "ISC"
}

What these fields mean:

  • name: Your project’s name (usually the folder name).
  • version: Current version of your project.
  • description: A brief description (you can fill this in).
  • main: The entry point for your app (default is index.js).
  • scripts: Commands you can run using npm run <script> (like tests).
  • keywords: An array of keywords for your project (optional).
  • author: Your name or author info (optional).
  • license: The license type for your project (default is ISC).

"test": "jest"

"test": "jest" in the scripts section of your package.json means that when you run npm test (or yarn test), it will execute the Jest testing framework.

What is Jest?

Jest is a popular JavaScript testing framework developed by Facebook. It’s widely used for testing JavaScript code, especially React apps, but it works well for any JavaScript project.

Jest helps you write and run tests for your code, making sure your functions and components behave as expected.

How it works:

  • When you run npm test and your package.json has "test": "jest", it runs the Jest command.
  • Jest looks for test files (usually files ending with .test.js or .spec.js).
  • It runs the tests and shows you the results in the console.
  • It helps catch bugs early by making sure your code behaves correctly.

Example

Suppose you have a function like this in a file called sum.js:


function sum(a, b) {
    return a + b;
}
module.exports = sum;

And a test file called sum.test.js:


const sum = require('./sum');

test('adds 1 + 2 to equal 3', () => {
  expect(sum(1, 2)).toBe(3);
});

If you have "test": "jest" in your package.json, running npm test will run the test and tell you if it passes or fails.
