Modern Node.js development

In modern Node.js development, Express.js is the most common and recommended approach to create web server applications over using the built-in http module directly.

Here‚Äôs a breakdown comparing the built-in http module vs Express in terms of modern practices:

üß± 1. Built-in http module

Example:


const http = require('http');

const server = http.createServer((req, res) => {
  if (req.url === '/' && req.method === 'GET') {
    res.writeHead(200, { 'Content-Type': 'text/plain' });
    res.end('Hello World');
  } else {
    res.writeHead(404);
    res.end('Not Found');
  }
});

server.listen(3000, () => {
  console.log('Server running on http://localhost:3000');
});

‚úÖ Pros:

  • No dependencies.
  • Fine-grained control over requests/responses.
  • Good for learning core Node.js.

‚ùå Cons:

  • Verbose for routing.
  • No built-in middleware support (e.g., parsing JSON, URL-encoded bodies).
  • Harder to scale for larger apps.

‚ö° 2. Express.js (modern, widely adopted)

Example:


const express = require('express');
const app = express();

app.use(express.json()); // for parsing application/json

app.get('/', (req, res) => {
  res.send('Hello World');
});

app.listen(3000, () => {
  console.log('Server running on http://localhost:3000');
});

‚úÖ Pros:

  • Simple and readable syntax.
  • Built-in routing and middleware support.
  • Huge ecosystem and community support.
  • Scales better for real-world apps.
  • Easy integration with other tools (e.g., logging, error handling, validation).

‚ùå Cons:

  • Slightly more overhead due to abstraction.
  • Adds dependency (but widely trusted and maintained).

‚úÖ Recommended (Modern) Approach in 2025

Use Express.js (or newer frameworks like Fastify if performance matters more) for most applications unless:

  • You are building something very lightweight or educational.
  • You want full control over the HTTP lifecycle.
