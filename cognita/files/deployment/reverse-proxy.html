Reverse Proxy

A reverse proxy is a type of server that sits between the client (browser) and one or more backend servers, and it handles incoming requests on behalf of those backend servers.

üîÅ In simple terms:

It receives requests from users and forwards them to the appropriate server, then sends the server‚Äôs response back to the user.

üîç Why is it called a reverse proxy?

  • A forward proxy (like a VPN) hides the client from the server.
  • A reverse proxy hides the server from the client.

So, in a reverse proxy setup:

  • The client doesn't directly connect to the backend server.
  • Instead, it connects to the reverse proxy, which handles the request.

üì¶ Common uses of a reverse proxy:

Use Case Description
Load balancing Distributes traffic to multiple backend servers to prevent overloading any single one.
SSL termination Handles HTTPS encryption/decryption so backend servers can stay on HTTP (simpler).
Caching Caches responses to serve repeated requests faster.
Security Hides internal server structure, filters requests, blocks attacks.
Compression / Optimization Compresses data before sending to the client.
Routing / URL rewriting Directs traffic to different services based on URLs or headers.
