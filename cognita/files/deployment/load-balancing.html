Load Balancing

Load balancing is the process of distributing incoming network or application traffic across multiple servers or resources. The goal is to optimize resource use, maximize throughput, minimize response time, and avoid overload on any single server.

Why is Load Balancing Important?

  • Improves availability and reliability by distributing workload across multiple servers.
  • Enhances scalability as new servers can be added without downtime.
  • Prevents server overload, reducing the risk of crashes or slowdowns.
  • Ensures better user experience by reducing latency and response time.

Types of Load Balancing

  1. Hardware Load Balancers: Physical devices designed for load balancing.
  2. Software Load Balancers: Programs running on general-purpose servers (e.g., Nginx, HAProxy).
  3. DNS Load Balancing: Using DNS to distribute requests by resolving to different IPs.
  4. Cloud Load Balancing: Managed services provided by cloud providers (AWS ELB, Azure LB, Google Cloud LB).

Common Load Balancing Algorithms

  • Round Robin: Distributes requests sequentially.
  • Least Connections: Sends requests to the server with the fewest active connections.
  • IP Hash: Routes based on client IP address.
  • Weighted Round Robin: Assigns requests based on server capacity (weight).
  • Random: Randomly distributes requests.

Load Balancer Deployment Models

  • Layer 4 (Transport Layer): Balances traffic based on TCP/UDP information.
  • Layer 7 (Application Layer): Balances traffic based on application data (HTTP headers, cookies, etc.).
