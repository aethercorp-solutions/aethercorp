A simple configuration to serve a static site:

Great ‚Äî using Nginx with Node.js is a common and powerful setup. Nginx acts as a reverse proxy in front of your Node.js app, handling incoming web requests, SSL termination, and load balancing, while Node.js handles the application logic.

‚úÖ Why Use Nginx with Node.js?

  • Handles static files more efficiently than Node.js
  • Improves performance by offloading HTTP work
  • Adds security layers (e.g., only expose Nginx to the internet)
  • Easier SSL setup with Let's Encrypt
  • Enables load balancing for multiple Node processes

üîß Typical Setup

Let‚Äôs say your Node.js app is running on:

  • localhost:3000

You want Nginx to:

  • Listen on port 80
  • Forward requests to Node.js
  • Optionally serve static files

üóÇÔ∏è 1. Node.js App Example

Start with a basic Express app:

const app = express();

app.get('/', (req, res) => {
    res.send('Hello from Node.js!');
});

app.listen(3000, () => {
    console.log('Server running on http://localhost:3000');
});

Run it:


node app.js

üõ†Ô∏è 2. Nginx Config (Reverse Proxy)

Open your Nginx config file (often in /etc/nginx/sites-available/default or /etc/nginx/nginx.conf):


server {
    listen 80;
    server_name yourdomain.com;

    location / {
        proxy_pass http://localhost:3000;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }
}

Then restart Nginx:


sudo nginx -t      # Test config
sudo systemctl restart nginx

Now your Node.js app is accessible via http://yourdomain.com

To make your local server (e.g., running Nginx + Node.js on port 3000) available publicly, you need to:

‚úÖ Step-by-Step Guide

1. üõ†Ô∏è Run the Server Locally

Make sure your app is running and accessible locally.

Example (Node.js):

Test:

2. üì° Find Your Local IP Address

On your computer, find your private IP:

  • Windows: ipconfig in Command Prompt
  • Linux/macOS: ip a or ifconfig

Example result:


192.168.1.5

This is your machine‚Äôs IP on the local network.

3. üö™ Set Up Port Forwarding on Your Router

You need to forward a port from your public IP to your local IP.

How:

  • Login to your router's admin interface (usually 192.168.1.1 or similar)
  • Look for a section like Port Forwarding or NAT
  • Add a rule:
Setting Value
External Port 80 (or another public port)
Internal IP 192.168.1.5 (your machine)
Internal Port 3000 (Node.js app port)
Protocol TCP

‚û°Ô∏è This means anyone visiting your public IP on port 80 will be forwarded to your local server on port 3000.

4. üåê Find Your Public IP Address

Visit:

  • https://whatismyipaddress.com
  • Or run: 
    
curl ifconfig.me

Example: 203.0.113.25

Now, try from another device or network:


http://203.0.113.25

‚úÖ If port forwarding and firewall are working, it should load your local server.

5. üî• Allow Through Firewall (If Needed)

On your computer, make sure the firewall allows incoming traffic on the port you're using.

On Linux (UFW):


sudo ufw allow 3000

On Windows:

  • Go to Windows Defender Firewall settings
  • Allow your app or port through public/private networks

6. üîÅ Handle Dynamic IP (Optional but Important)

Your public IP can change (e.g., your ISP reassigns it).

To deal with this:

  • Use a Dynamic DNS (DDNS) service like:
    • No-IP
    • DuckDNS
  • Install their updater tool so your domain always points to your current IP

7. üîê Secure with HTTPS (Optional but Recommended)

If you're exposing your local server to the internet:

  • Use Let's Encrypt with DNS challenge (since port 80 might be blocked)
  • Or use Cloudflare Tunnel (more below)
