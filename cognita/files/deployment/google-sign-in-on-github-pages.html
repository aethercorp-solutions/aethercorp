Google Sign-In on a static site hosted on GitHub Pages

If you want to implement Google Sign-In on a static site hosted on GitHub Pages, here’s a straightforward approach:

Quick Overview

  • GitHub Pages is a static hosting service — no backend server.
  • Google Sign-In uses OAuth 2.0, which requires some setup.
  • You can use Google Identity Services JavaScript SDK for client-side sign-in.
  • After sign-in, you get a JWT token or user profile info in the browser.
  • No backend needed if you just want user authentication on client side.

Step-by-step to add Google Sign-In on GitHub Pages

  1. Create a Google Cloud Project & OAuth Client ID
    1. Go to Google Cloud Console.
    2. Create a new project or use an existing one.
    3. Navigate to APIs & Services > Credentials.
    4. Click Create Credentials > OAuth client ID.
    5. Select Web application.
    6. Add your GitHub Pages URL under Authorized JavaScript origins, e.g.: 
      
https://yourusername.github.io
    7. Copy the Client ID.

  2. Add Google Identity Services JS SDK to your HTML

    In your GitHub Pages repo, add this to your HTML page:

    
<script src="https://accounts.google.com/gsi/client" async defer></script>

  3. Add a Sign-In button and handle the callback

    Example HTML + JavaScript:

    
<!DOCTYPE html>
<html lang="en">
<head>
  <meta charset="UTF-8" />
  <title>Google Sign-In on GitHub Pages</title>
  <script src="https://accounts.google.com/gsi/client" async defer></script>
</head>
<body>
  <div id="g_id_onload"
       data-client_id="YOUR_CLIENT_ID.apps.googleusercontent.com"
       data-callback="handleCredentialResponse">
  </div>

  <div class="g_id_signin" data-type="standard"></div>

  <script>
    function handleCredentialResponse(response) {
      // response.credential is the ID token (JWT)
      console.log("Encoded JWT ID token: " + response.credential);

      // Optionally decode JWT here or send to backend
      // For demo, just display user info with JWT decode
      // You can use jwt-decode library or parse it yourself
      alert('Signed in! Token: ' + response.credential);
    }
  </script>
</body>
</html>
  4. Deploy to GitHub Pages
    • Push your changes.
    • Visit your GitHub Pages URL.
    • You should see a Google Sign-In button.
    • Clicking it triggers Google Sign-In and calls your handleCredentialResponse callback.

Notes

  • This approach uses the new Google Identity Services — which is the recommended way now.
  • If you want to verify the token, you normally send it to a backend for validation, but since GitHub Pages is static, you can only decode it client-side or use third-party backend.
  • Make sure your OAuth Client ID has the GitHub Pages URL as an authorized origin.
  • If you want to restrict sign-in only for your domain, configure that in Google Cloud Console.
