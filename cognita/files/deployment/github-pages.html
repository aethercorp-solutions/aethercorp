GitHub Pages

GitHub Pages is a free hosting service offered by GitHub that lets you deploy static websites directly from a GitHub repository.

Core Concepts

  • Static Site: GitHub Pages only supports static content—HTML, CSS, JavaScript, and static assets (images, fonts, etc.). No backend/server-side processing (e.g., PHP, Node.js).
  • Repository: Your website's files must be stored in a GitHub repository.
  • Branch: You can publish your site from:
    • main or master branch
    • /docs folder in the main branch
    • A gh-pages branch (often used for deployment)

Domain & URLs

  • Default URL: 
    
https://<username>.github.io/<repo-name>/
  • Custom Domain: You can configure a custom domain (e.g., mywebsite.com) via the repository settings and a CNAME file in your repo.

Configuration & Deployment

  • Settings > Pages:

    This is where you configure the source (branch/folder) for deployment and custom domain if applicable.

  • CNAME File:

    A file that contains your custom domain name, required if you're using a custom domain.

  • .nojekyll File:

    Optional file that tells GitHub Pages to skip Jekyll processing (useful for folders that start with _).

Supported Tools & Workflows

  • Jekyll: GitHub Pages natively supports Jekyll, a popular static site generator. You can use _config.yml and other Jekyll-specific files for templating and blog functionality.
  • CI/CD with GitHub Actions: Automate build and deployment steps (especially useful for React, Vue, Next.js, or other build-step frameworks).

  • Static Site Generators: If you're using something like React or Vue, you’ll need to:

    • Build your site (npm run build)
    • Deploy the build/ folder to gh-pages using a tool like gh-pages

Advantages of GitHub Pages

  • Free hosting
  • Easy integration with Git workflows
  • Version control built-in
  • HTTPS by default
  • Supports custom domains

Limitations

  • Static only (no server-side code)
  • 1 GB size limit per repository
  • 100 GB/month bandwidth limit
  • Not suitable for dynamic web apps requiring a backend
