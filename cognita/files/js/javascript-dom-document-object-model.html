JavaScript DOM (Document Object Model)

The JavaScript DOM (Document Object Model) is a programming interface provided by web browsers that allows JavaScript to interact with and manipulate HTML and XML documents.

üîç What is the DOM?

  • The DOM represents the structure of a web page as a tree of objects (nodes).
  • Every element, attribute, and piece of text in an HTML document becomes a node in this tree.
  • JavaScript can use the DOM to access, change, add, or delete elements and content on a web page.

Example Breakdown

Given this HTML:


<!DOCTYPE html>
<html>
  <body>
    <h1 id="title">Hello World</h1>
    <button onclick="changeText()">Click Me</button>
  </body>
</html>

You could use JavaScript + DOM like this:


function changeText() {
    document.getElementById("title").innerText = "You clicked the button!";
}
  • üîπ document ‚Äì The global object representing the web page
  • üîπ getElementById("title") ‚Äì Finds the element with ID title
  • üîπ .innerText ‚Äì Accesses or changes the text inside that element

‚ú® What Can You Do With the DOM?

  • Read or change content: element.innerText, element.innerHTML
  • Change styles: element.style.color = "red"
  • Handle events: element.addEventListener("click", callback)
  • Create or remove elements: document.createElement(), element.remove()
  • Navigate the tree: parentNode, childNodes, nextSibling

üí° Summary

The JavaScript DOM is how your scripts can dynamically interact with and change a web page after it's loaded, making web apps interactive and dynamic.

It represents the structure of a web page as a tree of nodes, where each node is an object representing a part of the document (like an element, attribute, or text).

üîπ Core Concepts

1. Accessing Elements

Use DOM methods to select HTML elements:


// By ID
const heading = document.getElementById('main-title');

// By class
const items = document.getElementsByClassName('item');

// By tag name
const paragraphs = document.getElementsByTagName('p');

// Modern way (returns first match)
const firstItem = document.querySelector('.item');

// Returns all matches as a NodeList
const allItems = document.querySelectorAll('.item');
            

const firstItem = document.querySelector('.item');

Explanation:

  • document.querySelector() is a method that returns the first element within the document that matches the specified CSS selector.
  • '.item' is the CSS selector used here. It selects elements with the class name item.
  • const firstItem stores a reference to that first matching element.

What it does:

This code selects the first HTML element with the class item and assigns it to the firstItem variable. If no such element exists, firstItem will be null.

Example HTML:


<div class="item">Item 1</div>
<div class="item">Item 2</div>

In this case, firstItem will refer to the first

with class item ‚Äî i.e., "Item 1".

2. Changing Content or Attributes

You can change text, HTML, or element attributes.


// Text content
heading.textContent = "New Title";

// Inner HTML
heading.innerHTML = "<span>New <em>Title</em></span>";

// Set attribute
heading.setAttribute("class", "highlighted");

// Get attribute
let className = heading.getAttribute("class");

3. Styling Elements

Directly manipulate styles.


heading.style.color = "red";
heading.style.fontSize = "24px";

4. Adding/Removing Elements

Dynamically update the page.


// Create element
const newDiv = document.createElement('div');
newDiv.textContent = "Hello, world!";
document.body.appendChild(newDiv); // Add to body

// Remove element
document.body.removeChild(newDiv);

5. Event Handling

Respond to user interactions.


const button = document.querySelector('#click-me');

button.addEventListener('click', function () {
  alert("Button clicked!");
});

üß† Summary

The DOM lets you:

  • Read and change the page content.
  • React to user actions.
  • Dynamically create or delete content.

Great question! Understanding how the DOM relates to React Native components is key to grasping how web development differs from mobile development using React.

React (Web) uses the DOM (e.g., <div>, <p>, <button>) to build UI in a browser. React Native uses native components (e.g., <View>, <Text>, <Button>) to build UI for mobile apps ‚Äî no DOM involved.

üîç Detailed Comparison

Concept React (Web) React Native
Underlying UI System HTML + CSS + DOM Native UI components (iOS/Android)
Component Example <div>, <h1>, <input>, <button></button> <View>, <Text>, <TextInput>, <Button></Button>
Rendered to Browser DOM Native platform's UI system
Style System CSS / CSS-in-JS StyleSheet.create() / inline styles
Event Handling DOM Events (onClick, onChange) React Native events (onPress, onChangeText)
DOM Manipulation? Yes (via JavaScript) ‚ùå No DOM ‚Äî you use props/state only

‚úÖ Similarities

  • Both use JSX (JavaScript XML syntax).
  • Both use components and props/state.
  • Both use React's rendering and reconciliation logic.

üö´ Key Differences

  • No DOM in React Native:

    React Native doesn't run in a browser, so there’s no HTML or CSS. Instead, it maps components like <View> or <Text> to native platform components (e.g., UIView on iOS or View on Android).

  • No document.querySelector or document.getElementById:

    These are DOM methods, which don’t exist in React Native.

üß© Analogy: DOM vs Native Components

Imagine React is a translator.

  • In React Web, it translates your JSX into DOM nodes.
  • In React Native, it translates your JSX into native UI elements on the mobile OS.

üëá Example

React (Web):


<function App() {
  return (
    <div>
      <h1>Hello, World!</h1>
      <button onClick={() => alert('Clicked!')}>Click Me</button>
    </div>
  );
}

React Native:


import { View, Text, Button } from 'react-native';

function App() {
  return (
    <View>
      <Text>Hello, World!</Text>
      <Button title="Click Me" onPress={() => alert('Clicked!')} />
    </View>
  );
}

Summary

  • DOM is not used in React Native.
  • React Native components are rendered via native APIs, not HTML.
  • The concepts of components, props, state, and events are shared, but the rendering targets are completely different.
