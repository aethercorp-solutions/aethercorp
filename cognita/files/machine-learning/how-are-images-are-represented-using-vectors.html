How images are represented using vectors

Great question—this is one of those topics that sounds abstract at first but becomes very intuitive once the pieces click. Let’s walk through it from the ground up and then zoom out to how modern AI actually uses image vectors.

1. The core idea: images → numbers → vectors

A vector is just an ordered list of numbers.

An image, at its heart, is also just numbers:

  • Numbers that describe brightness
  • Or color intensities
  • Arranged in a grid (rows × columns)

So representing images as vectors means:

taking all those pixel values and arranging them into a numerical vector that a computer (or ML model) can work with.

2. Pixels: the building blocks

Grayscale images

In a grayscale image:

  • Each pixel has one value
  • Usually between 0 and 255
    • 0 = black
    • 255 = white

Example:

A tiny 3×3 grayscale image


[  
    0,   50,  100
    150, 200, 255
    30,  80,  130 
]

This is already numerical data.

Color images (RGB)

Most images are RGB:

  • Each pixel has three values:
    • Red
    • Green
    • Blue

Each value is usually in [0, 255]

Example pixel:


(120, 200, 75)

So an image of size:


Height × Width × 3

Example:

  • 64 × 64 RGB image
  • Total numbers = 64 × 64 × 3 = 12,288

3. Flattening an image into a vector

Machine learning models often expect 1D vectors, not 2D or 3D grids.

So we flatten the image.

Example: 2×2 RGB image


Pixel grid:
[
 [(255,0,0), (0,255,0)],
 [(0,0,255), (255,255,255)]
]

Flattened into a vector:


[255,0,0,  0,255,0,  0,0,255,  255,255,255]

That’s now a vector in 12-dimensional space.

Every image of the same size becomes a point in the same high-dimensional vector space.

4. Normalization: making vectors behave nicely

Raw pixel values (0–255) are often scaled to make learning easier.

Common approaches:

  • Divide by 255 → values in [0,1]
  • Standardize to mean 0, variance 1

Example:


255 → 1.0
128 → 0.502

This doesn’t change the image—just how it’s represented numerically.

5. Images as points in high-dimensional space

Once flattened:

  • A 28×28 grayscale image → vector in 784D
  • A 224×224 RGB image → vector in 150,528D

It’s describing a way to think about images mathematically, not something you’d ever visualize directly.

Here’s the intuition step by step.

1. Images are just numbers

A digital image is a grid of pixel values.

  • Grayscale image: each pixel = 1 number (intensity)
  • RGB image: each pixel = 3 numbers (red, green, blue)

2. “Flattening” an image

Flattening means taking that 2D (or 3D for RGB) grid and laying all the numbers out in one long list.

  • A 28 × 28 grayscale image

    → 28 × 28 = 784 pixels

    → one list of 784 numbers

    → a vector in 784-dimensional space

  • A 224 × 224 RGB image

    → 224 × 224 × 3 = 150,528 values

    → one list of 150,528 numbers

    → a vector in 150,528-dimensional space

3. “High-dimensional space” idea

In math:

  • 2 numbers → a point on a 2D plane
  • 3 numbers → a point in 3D space
  • 784 numbers → a point in 784D space
  • 150,528 numbers → a point in 150,528D space

So each image is a single point in a space with one dimension per pixel value.

You can’t visualize this space, but mathematically it works the same way.

4. Why this matters (the real point)

This idea lets machine learning models:

  • Compare images using distance (e.g., “how similar are these two points?”)
  • Apply linear algebra (dot products, projections, transformations)
  • Learn patterns by reshaping or compressing these giant spaces (e.g., neural networks)

For example:

  • Two similar images → points close together
  • Very different images → points far apart

One important caveat

Flattening throws away spatial structure (which pixel was next to which), which is why:

  • Classical ML used flattened vectors
  • CNNs keep the 2D structure instead

6. Why raw pixel vectors are not enough

Flattened pixel vectors have problems:

  • Lose spatial structure (nearby pixels matter!)
  • Very high dimensional
  • Sensitive to small changes (lighting, noise, rotation)

Two images of the same cat can be far apart as raw vectors.

So modern systems use learned vector representations.

7. Feature vectors from neural networks (the important part)

Instead of using raw pixels, we pass the image through a neural network (usually a CNN or Vision Transformer).

What happens:

  1. Early layers detect:
    • Edges
    • Corners
    • Textures
  2. Middle layers detect:
    • Shapes
    • Parts (eyes, wheels)
  3. Final layers detect:
    • High-level concepts (faces, dogs, cars)

The output of a hidden layer is a feature vector (also called an embedding).

Example:


Image → Neural Network → 512-dimensional vector

This vector:

  • Encodes meaning, not just pixels
  • Is robust to small changes
  • Clusters semantically similar images together

8. Image embeddings: how AI “understands” images

An image embedding vector might look like:


[0.12, -0.03, 0.88, ..., -0.41]

Properties:

  • Fixed length (e.g. 256, 512, 1024)
  • Dense (not sparse like raw pixels)
  • Learned from massive datasets

Used for:

  • Image search (“find images like this”)
  • Face recognition
  • Object detection
  • Multimodal models (image ↔ text)

9. Vector similarity in practice

To compare two images:

  1. Convert each image to a vector
  2. Compute similarity

Common metric:

  • Cosine similarity 
    
similarity = (v₁ · v₂) / (||v₁|| ||v₂||)
  • Close to 1 → very similar
  • Close to 0 → unrelated

This is the backbone of:

  • Google Images
  • Pinterest visual search
  • CLIP-style image–text matching

10. Big picture summary

Representation What it captures Vector size
Raw pixels Exact colors Huge
Flattened image Pixel values only Very large
CNN features Shapes & patterns Medium
Embeddings Semantic meaning Compact

In short:

Images are represented as vectors by turning pixel values—or learned visual features—into numerical lists that live in high-dimensional space, where geometry encodes visual similarity.
