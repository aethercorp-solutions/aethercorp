Complete step-by-step guide to configure Amazon CloudFront

Below is a complete step-by-step guide to configure Amazon CloudFront with a typical setup (S3 static website + HTTPS + custom domain).

I’ll cover:

  1. ✅ Basic setup (S3 + CloudFront)
  2. 🔐 Enable HTTPS
  3. 🌍 Add custom domain
  4. ⚡ Optimize caching
  5. 🚀 Production best practices

✅ Scenario Example

We’ll configure:

  • Origin → Amazon S3
  • CDN → CloudFront
  • DNS → Amazon Route 53
  • SSL → AWS Certificate Manager

🧱 STEP 1: Create S3 Bucket (Origin)

  1. Go to AWS Console
  2. Open S3
  3. Click Create bucket
  4. Enter:
    • Bucket name: my-static-website
    • Region: Choose region
  5. Disable "Block all public access" (if using public content)
  6. Upload:
    • index.html
    • CSS / JS / images

🔐 (Recommended) Use S3 with Origin Access Control (Private Bucket)

Instead of making S3 public:

  1. Keep S3 bucket private
  2. Let CloudFront access it securely using:
    • Origin Access Control (OAC)

This is the recommended production approach.

🌍 STEP 2: Create CloudFront Distribution

  1. Open CloudFront
  2. Click Create Distribution

1️⃣ Origin Settings

  • Origin domain → Select S3 bucket
  • Origin access → Choose:
    • Origin Access Control (recommended)

Click Create OAC if needed.

2️⃣ Default Cache Behavior

Set:

  • Viewer protocol policy → Redirect HTTP to HTTPS
  • Allowed HTTP methods → GET, HEAD (or more for API)
  • Cache policy → CachingOptimized

3️⃣ Settings

  • Price class → Choose region coverage
  • Default root object → index.html

Click Create Distribution

⏳ Deployment takes 5–15 minutes.

You’ll get a domain like:


d123abc.cloudfront.net

🔐 STEP 3: Enable HTTPS (SSL Certificate)

Go to:

→ AWS Certificate Manager (ACM)

Important:

Certificate MUST be created in us-east-1 (N. Virginia) for CloudFront.

  1. Request public certificate
  2. Enter domain:
    • example.com
    • www.example.com
  3. Choose DNS validation
  4. Validate via Route 53 (auto)

🌎 STEP 4: Add Custom Domain to CloudFront

Go back to CloudFront:

  1. Open distribution
  2. Edit
  3. Add:
    • Alternate domain name (CNAME): example.com
  4. Choose:
    • Custom SSL certificate → Select ACM certificate

Save changes.

🌐 STEP 5: Configure DNS (Route 53)

  1. Open Route 53
  2. Hosted Zones
  3. Select domain
  4. Create record:

Type: A

Alias: YES

Alias target → Select CloudFront distribution

Now:


example.com → CloudFront → S3

⚡ STEP 6: Test It

Open browser:


https://example.com

You should see your website via CloudFront.

🧠 STEP 7: Configure Caching Properly

In CloudFront → Behaviors:

You can control:

  • TTL (Time to Live)
  • Cache based on:
    • Headers
    • Cookies
    • Query strings

🔄 STEP 8: Invalidate Cache (When Updating Files)

If you update index.html:

  1. Go to CloudFront
  2. Invalidations
  3. Create invalidation
  4. Add: 
    
/*

This forces CloudFront to refresh cache.

🛡 STEP 9: (Optional) Add Security Layer

You can integrate:

  • AWS WAF → block attacks
  • AWS Shield → DDoS protection

🚀 STEP 10: Production Best Practices

✅ Use Origin Access Control (keep S3 private)

✅ Enable compression (Brotli, Gzip)

✅ Redirect HTTP → HTTPS

✅ Set proper cache headers

✅ Use versioning for static files:


style.v2.css

✅ Enable logging (store logs in S3)
