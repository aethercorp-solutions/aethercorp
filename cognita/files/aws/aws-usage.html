ðŸ”§ Step-by-Step Implementation

  1. Create an AWS Account
    • Go to: https://aws.amazon.com/
    • Sign up and set up billing alerts (important to avoid surprise charges).
  2. Create an S3 Bucket
    • Go to S3 from the AWS Console.
    • Click "Create bucket"
      • Give it a unique name (e.g., my-portfolio-site).
      • Choose a region (close to your users).
      • Uncheck "Block all public access" under permissions (youâ€™ll make it public).
  3. Upload Website Files
    • Prepare simple HTML/CSS files (e.g., index.html, styles.css).
    • Upload them into the S3 bucket.
  4. Configure Static Website Hosting
    • Go to Properties of the S3 bucket.
    • Scroll to Static website hosting and enable it.
      • Set index.html as the Index Document.
    • Save and note the endpoint URL AWS provides (e.g., http://your-bucket-name.s3-website-us-east-1.amazonaws.com)
  5. Make Files Public
    • Go to the "Permissions" tab of the bucket.
    • Click "Bucket Policy" and paste the following (update bucket name):
    
{
    "Version": "2012-10-17",
    "Statement": [
        {
        "Sid": "PublicReadGetObject",
        "Effect": "Allow",
        "Principal": "*",
        "Action": "s3:GetObject",
        "Resource": "arn:aws:s3:::your-bucket-name/*"
        }
    ]
}
  6. Access Your Website
    • Visit the S3 website endpoint.
    • You should see your static site live!

(Optional) Use a Custom Domain

If you have a domain (e.g., from Namecheap or GoDaddy), you can:

  • Point it to your S3 static site using Route 53 or domain registrar settings.
  • Use CloudFront to serve over HTTPS.
