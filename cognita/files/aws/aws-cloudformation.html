AWS CloudFormation

AWS CloudFormation is a service that enables you to model, provision, and manage AWS resources using infrastructure as code (IaC). A CloudFormation stack is a collection of AWS resources that are created, updated, or deleted together as a single unit based on a CloudFormation template.

ðŸ”¹ Key Concepts

  1. CloudFormation Template
    • A JSON or YAML file that describes the AWS resources you want to provision.
    • Contains:
      • Resources: The actual AWS services to deploy (EC2, S3, RDS, etc.).
      • Parameters (optional): Input values you can pass at runtime.
      • Outputs (optional): Values you want returned (like resource IDs or URLs).
      • Mappings, Conditions, Metadata, etc.
  2. Stack
    • A stack is the instantiation of a CloudFormation template.
    • All resources defined in the template are provisioned and managed together.
    • You can create, update, or delete the stack to manage the lifecycle of the resources.
  3. StackSet
    • Used to deploy a CloudFormation stack across multiple AWS accounts and/or regions.

ðŸ”¹ Example Template (YAML)


AWSTemplateFormatVersion: '2010-09-09'
Description: Create an S3 bucket

Resources:
  MyS3Bucket:
    Type: AWS::S3::Bucket
    Properties:
      BucketName: my-unique-bucket-name-123456

ðŸ”¹ Creating a Stack (CLI)


aws cloudformation create-stack \
  --stack-name MyTestStack \
  --template-body file://template.yaml

ðŸ”¹ Use Cases

  • Automate infrastructure provisioning (servers, databases, networks)
  • Ensure consistent environments across dev/staging/prod
  • Version control your infrastructure in Git
  • Roll back infrastructure changes safely

ðŸ”¹ Benefits

  • IaC: Repeatable, testable deployments
  • Automation: Reduces manual errors
  • Drift Detection: Identify differences between actual resources and template
  • Change Sets: Preview changes before applying
