AWS Lambda is a serverless compute service that lets you run code without managing servers.

You just upload your function code (in Node.js, Python, Go, etc.), and AWS runs it on demand ‚Äî when triggered by an event such as an API request, file upload, or scheduled task.

Key points:

  • You pay only for the execution time (when your code runs).
  • Lambda automatically scales based on incoming requests.
  • Each Lambda function runs in a stateless container that AWS provisions automatically.

Example use cases:

  • Process HTTP requests (e.g., from API Gateway)
  • Run background tasks (e.g., image processing)
  • Respond to events from AWS S3, DynamoDB, or CloudWatch

üåê Amazon API Gateway

API Gateway is a fully managed service for creating, publishing, and managing APIs.

It acts as a front door that:

  • Receives HTTP requests (e.g., from browsers or mobile apps)
  • Validates and transforms them
  • Forwards them to backend services ‚Äî like AWS Lambda

Key features:

  • Supports REST, WebSocket, and HTTP APIs
  • Handles authentication, rate limiting, and CORS
  • Can generate API keys and usage plans
  • Natively integrates with AWS Lambda
