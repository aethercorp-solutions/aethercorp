üîß AWS Services Used (Explained Clearly)

AWS Amplify itself is a framework‚Äîunder the hood it automatically creates and manages multiple AWS services:

AWS Service Purpose Why It's Used in the Notes App
Amazon Cognito Authentication (signup, signin, MFA, social login) Securely manage users. Avoid building custom auth.
AppSync (GraphQL API) Serverless API layer that auto-connects to DynamoDB Fast, real-time API without managing servers.
DynamoDB NoSQL database Stores notes with instant scalability.
S3 (Simple Storage Service) File storage (images, uploads) Store note attachments or user files.
IAM (Identity and Access Management) Permissions and roles Secure access between services. Amplify auto-configures roles.
Amplify Hosting / Amplify Console Static web hosting & CI/CD Deploy React or Next.js app globally.
Amplify CLI & Libraries Developer tools Generates backend resources and client code.

You rarely touch these services directly‚ÄîAmplify configures them for you.

üìù Example: Create a Note


import { API, graphqlOperation } from 'aws-amplify';
import { createNote } from './graphql/mutations';

async function addNote() {
  await API.graphql(graphqlOperation(createNote, {
    input: {
      name: "My first note",
      description: "Hello Amplify!"
    }
  }));
}

üì∑ Example: Upload File to S3


import { Storage } from 'aws-amplify';

async function upload(file) {
  await Storage.put(file.name, file, {
    contentType: file.type
  });
}
