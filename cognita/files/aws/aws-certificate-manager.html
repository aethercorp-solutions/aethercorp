AWS Certificate Manager (ACM)

AWS Certificate Manager (ACM) is an AWS service that lets you provision, manage, and deploy SSL/TLS certificates for use with AWS services and your internal connected resources. These certificates are used to secure network communications and establish the identity of your websites over HTTPS.

The command you provided is correct for requesting an AWS ACM (AWS Certificate Manager) certificate using DNS validation. Here's a detailed breakdown and what to expect next:

  • aws acm request-certificate: Requests a new SSL/TLS certificate.
  • --domain-name "api.trycastanet.com": The exact domain you want the certificate for.
  • --validation-method DNS: Uses DNS validation (you will need to add a DNS record to prove domain ownership).
  • --region us-east-1: ACM certificates for use with CloudFront must be in us-east-1. For API Gateway regional endpoints, you can use your endpoint‚Äôs region.

What happens after running this

  1. AWS will issue a Certificate ARN in the response.
  2. ACM will provide DNS validation records (CNAME) you must add to your domain‚Äôs DNS.
  3. Once the DNS record propagates, ACM validates the domain and issues the certificate.
  4. After issuance, you can attach it to:
    • API Gateway custom domains
    • CloudFront distributions
    • Elastic Load Balancers
