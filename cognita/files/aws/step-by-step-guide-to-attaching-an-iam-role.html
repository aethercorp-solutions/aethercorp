Step-by-step guide to attaching an IAM role for access to Amazon S3 in Amazon Web Services

Here’s a detailed, step-by-step guide to attaching an IAM role for access to Amazon S3 in Amazon Web Services.

I’ll cover the most common scenario:

  • Create an IAM role
  • Attach an S3 access policy
  • Attach the role to an EC2 instance

    (You can adapt the same idea for Lambda, ECS, etc.)

✅ Step 1: Understand the Components

Before starting, know the 3 key pieces:

  1. IAM Role – A set of permissions that AWS services can assume.
  2. IAM Policy – Defines what actions are allowed (e.g., read/write S3).
  3. Trusted Entity – The AWS service that can assume the role (e.g., EC2).

IAM is managed through AWS Identity and Access Management.

✅ Step 2: Create an IAM Role

1️⃣ Go to IAM Console

  • Log in to AWS Console
  • Search for IAM
  • Click Roles
  • Click Create role

2️⃣ Choose Trusted Entity

Select the AWS service that will use the role.

For example:

  • Choose AWS service
  • Select EC2 (if attaching to an EC2 instance)
  • Click Next

(If using Lambda, choose Lambda instead.)

✅ Step 3: Attach S3 Permissions Policy

Now attach permissions.

Option A: Use AWS Managed Policy (Simple Way)

Search and select:

  • AmazonS3ReadOnlyAccess (read-only)
  • AmazonS3FullAccess (full access)

These are AWS managed policies for Amazon S3.

Click Next

Option B: Create a Custom Policy (Recommended for Production)

  1. Click Create policy
  2. Choose JSON
  3. Example: Allow access to a specific bucket: 
    
{
    "Version": "2012-10-17",
    "Statement": [
        {
        "Effect": "Allow",
        "Action": [
            "s3:GetObject",
            "s3:PutObject"
        ],
        "Resource": [
            "arn:aws:s3:::my-bucket-name/*"
        ]
        }
    ]
}
  4. Click Next
  5. Name the policy (e.g., MyAppS3AccessPolicy)
  6. Create policy
  7. Go back and attach it to your role

✅ Step 4: Name and Create the Role

  • Enter a role name (e.g., EC2-S3-Access-Role)
  • Review settings
  • Click Create role

Your IAM role is now created.

✅ Step 5: Attach the Role to an EC2 Instance

If you're using Amazon EC2:

Option A: Attach During Launch

  • When launching EC2
  • Under Advanced details
  • Choose the IAM role

Option B: Attach to Existing Instance

  1. Go to EC2 Dashboard
  2. Select your instance
  3. Click Actions
  4. Choose: 
    
Security → Modify IAM role
  5. Select your role
  6. Click Update IAM role

Done ✅

✅ Step 6: Verify It Works

SSH into your EC2 instance and test:


aws s3 ls

If configured correctly:

  • It will list buckets
  • No access key configuration is needed

IAM roles automatically provide temporary credentials.

🔐 Important Best Practices

✔ Follow least privilege principle

✔ Avoid AmazonS3FullAccess in production

✔ Restrict access to specific bucket ARN

✔ Use IAM roles instead of access keys

✔ Enable CloudTrail for auditing

🧠 What Happens Behind the Scenes?

When EC2 starts:

  1. It assumes the IAM role
  2. AWS generates temporary credentials
  3. Credentials are available via Instance Metadata Service
  4. AWS SDK automatically uses them

No hardcoded keys needed.
