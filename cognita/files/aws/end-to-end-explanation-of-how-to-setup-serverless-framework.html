end-to-end explanation of how to set up and use the Serverless Framework

Below is a deep, practical, end-to-end explanation of how to set up and use the Serverless Framework, focused on AWS + Node.js, and aligned with how you’ve been building APIs with Lambda, API Gateway, MongoDB, JWT/Auth, etc.

What is the Serverless Framework (in plain terms)

The Serverless Framework is a deployment + configuration tool that:

  • Converts a simple serverless.yml file
  • Into AWS CloudFormation
  • Which creates & manages:
    • Lambda functions
    • API Gateway routes
    • IAM roles & permissions
    • Environment variables
    • Logs, stages, regions

👉 You write application code, not infrastructure boilerplate.

Core Concepts (important to understand first)

Concept Meaning
Service One project (API, worker, backend)
Function One Lambda function
Event What triggers the function (HTTP, SQS, cron, etc.)
Provider Cloud provider (AWS, Azure, GCP)
Stage Environment (dev, staging, prod)
Region AWS region (ap-southeast-1, us-east-1)
serverless.yml The heart of everything

1️⃣ Install Requirements

Install Node.js (LTS recommended)


node -v
npm -v

Install Serverless Framework globally

Verify:


serverless --version

2️⃣ Configure AWS Credentials (CRITICAL)

Serverless does NOT deploy without AWS credentials.

Option A: Using AWS CLI (recommended)


aws configure

You will enter:

  • Access Key
  • Secret Key
  • Region (e.g. ap-southeast-1)
  • Output format (json)

Option B: Named Profiles (best for multiple environments)


aws configure --profile dev
aws configure --profile prod

3️⃣ Create a New Serverless Project


serverless create --template aws-nodejs --path my-api
cd my-api

Project structure:


my-api/
├── handler.js
├── serverless.yml
├── package.json

4️⃣ Understand serverless.yml (MOST IMPORTANT FILE)

Example:


service: my-api

frameworkVersion: '3'

provider:
  name: aws
  runtime: nodejs18.x
  region: ap-southeast-1
  stage: dev
  memorySize: 512
  timeout: 10

functions:
  hello:
    handler: handler.hello
    events:
      - http:
          path: hello
          method: get

Breakdown

service

Name of your backend (used in AWS resource naming)

provider

Controls AWS settings:


provider:
  runtime: nodejs18.x
  stage: dev
  region: ap-southeast-1

This creates:

  • Lambda runtime
  • API Gateway stage
  • CloudFormation stack

5️⃣ Lambda Function Code

handler.js


export const hello = async (event) => {
    return {
        statusCode: 200,
        body: JSON.stringify({
        message: "Hello from Serverless!"
        }),
    };
};

6️⃣ Deploy to AWS


serverless deploy

Behind the scenes:

  1. Packages your code
  2. Uploads to S3
  3. Creates CloudFormation stack
  4. Creates Lambda + API Gateway
  5. Outputs endpoint URL

Output example:


endpoint: GET https://xxxx.execute-api.ap-southeast-1.amazonaws.com/dev/hello

7️⃣ Calling the API


curl https://xxxx.execute-api.ap-southeast-1.amazonaws.com/dev/hello

8️⃣ Multiple Routes (REST API Pattern)


functions:
  getUsers:
    handler: users.get
    events:
      - http:
          path: users
          method: get

  createUser:
    handler: users.create
    events:
      - http:
          path: users
          method: post

users.js


export const get = async () => ({
    statusCode: 200,
    body: JSON.stringify({ users: [] }),
    });
    
    export const create = async (event) => {
    const body = JSON.parse(event.body);
    return {
        statusCode: 201,
        body: JSON.stringify(body),
    };
};

9️⃣ Environment Variables (MongoDB, JWT, etc.)


provider:
  environment:
    MONGO_URI: ${env:MONGO_URI}
    JWT_SECRET: ${env:JWT_SECRET}

Set locally:


export MONGO_URI="mongodb+srv://..."
export JWT_SECRET="supersecret"

Or use .env with plugin:


npm install serverless-dotenv-plugin
            

plugins:
  - serverless-dotenv-plugin

🔟 IAM Permissions (Very Important)

By default Lambda has NO AWS permissions.


provider:
  iam:
    role:
      statements:
        - Effect: Allow
          Action:
            - s3:GetObject
            - dynamodb:PutItem
          Resource: "*"

1️⃣1️⃣ Stages (dev / prod)


serverless deploy --stage prod

In YAML:


provider:
  stage: ${opt:stage, 'dev'}

Environment variables per stage:


custom:
  mongoUri:
    dev: mongodb://dev
    prod: mongodb://prod

provider:
  environment:
    MONGO_URI: ${self:custom.mongoUri.${self:provider.stage}}

1️⃣2️⃣ MongoDB + Lambda (Best Practice)

⚠️ Never open a new DB connection per request


let cachedDb = null;

export const handler = async () => {
  if (!cachedDb) {
    cachedDb = await connectToMongo();
  }
};

1️⃣3️⃣ Packaging Optimization (VERY IMPORTANT)


package:
  individually: true
  patterns:
    - '!node_modules/aws-sdk/**'

1️⃣4️⃣ Plugins You’ll Actually Use


npm install --save-dev
  serverless-offline
  serverless-dotenv-plugin
  serverless-plugin-optimize
            

plugins:
  - serverless-offline
  - serverless-dotenv-plugin

Run locally:


serverless offline

1️⃣5️⃣ Logs & Debugging


serverless logs -f hello
serverless logs -f hello --tail

1️⃣6️⃣ Remove Everything (Clean Up AWS)


serverless remove

Deletes:

  • Lambda
  • API Gateway
  • IAM roles
  • CloudFormation stack
