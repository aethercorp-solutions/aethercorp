practical ways to locally test an AWS Node.js app

Here are the practical ways to locally test an AWS Node.js app, depending on what you’re building (Lambda, API Gateway, Serverless, etc.).

1️⃣ If you’re testing a plain Node.js app

Nothing AWS-specific yet—just run it locally:


node index.js

or with hot reload:


npm install -g nodemon
nodemon index.js

Good for business logic before wrapping it in Lambda.

2️⃣ Testing AWS Lambda locally (most common)

✅ Option A: AWS SAM (recommended)

Best if you’re using Lambda + API Gateway.

Install


brew install aws-sam-cli

Example Lambda handler


exports.handler = async (event) => {
    return {
        statusCode: 200,
        body: JSON.stringify({ message: "Hello from Lambda" })
    };
};

Local invoke


sam local invoke \
  --event event.json

Local API Gateway


sam local start-api

Then open:


http://localhost:3000

💡 SAM uses Docker to mimic AWS runtime.

✅ Option B: Serverless Framework

Perfect if you’re already using serverless.yml.

Install


npm install -g serverless
npm install serverless-offline

serverless.yml


plugins:
  - serverless-offline

Run locally


serverless offline

API runs at:


http://localhost:3000

👉 This is very popular for Node.js + MongoDB + API Gateway setups.

3️⃣ Testing Lambda without any framework

You can simulate Lambda calls manually:


const { handler } = require("./index");

handler({ key: "value" }, {}, console.log);

Great for unit tests.

4️⃣ Unit Testing (Jest / Mocha)

Jest example


                npm install --save-dev jest
            

const { handler } = require("../index");

test("Lambda returns 200", async () => {
  const res = await handler({});
  expect(res.statusCode).toBe(200);
});

Run:


npm test

5️⃣ Mock AWS services locally (VERY useful)

DynamoDB


docker run -p 8000:8000 amazon/dynamodb-local

S3 / SNS / SQS

Use LocalStack:


brew install localstack
localstack start

Then point SDK to:


http://localhost:4566

6️⃣ Environment variables (like AWS)


export AWS_REGION=us-east-1
export STAGE=dev

Or use .env:


npm install dotenv
            

require("dotenv").config();

🔥 Recommended combo (real-world)

For serious development:

  • Serverless Framework + serverless-offline
  • Jest for unit tests
  • LocalStack for AWS services
  • Docker installed
