AWS Lambda is a serverless compute service provided by Amazon Web Services

AWS Lambda is a serverless compute service provided by Amazon Web Services that lets you run code without provisioning or managing servers. It automatically handles the infrastructure for you ‚Äî including scaling, fault tolerance, and maintenance.

üîß How AWS Lambda Works (Step-by-Step):

  1. Write Your Code (Function)

    You write your code in a supported language like:

    • Python
    • Node.js
    • Java
    • Go
    • C#
    • Ruby
    • Custom Runtime (via container images or Lambda layers)
    Lambda functions have a specific structure ‚Äî they must define a handler, which AWS Lambda uses as the entry point.

  2. Deploy the Function

    You upload your code:

    • Directly in the AWS Console
    • Using the AWS CLI or SDKs
    • As a container image (up to 10 GB)
    • Via infrastructure-as-code tools (like Terraform or CloudFormation)

  3. Set a Trigger (Event Source)

    Lambda is event-driven. It runs in response to events, such as:

    • API Gateway (HTTP requests)
    • S3 (file uploads)
    • DynamoDB (data changes)
    • CloudWatch (cron jobs)
    • SNS/SQS (messages)
    • Step Functions, and more

  4. Execution

    When an event is triggered:

    • AWS Lambda instantiates an execution environment (a sandboxed container).
    • The handler function is invoked with:
      • The event data
      • A context object with runtime info
    • Your code runs and returns a result (or error).
  5. Automatic scaling
    • AWS Lambda automatically scales horizontally based on the number of events.
    • Each function invocation runs in a separate, isolated environment.
  6. Pricing
    • You pay only for compute time used ‚Äî metered in milliseconds.
    • Free tier: 1 million requests & 400,000 GB-seconds/month.

    1. You connect Lambda to API Gateway.
    2. A user sends an HTTP request.
    3. API Gateway triggers your Lambda.
    4. Lambda runs your code (e.g., fetches from a DB, processes data).
    5. Lambda returns a response to API Gateway ‚Üí back to the user.

Concept Description
