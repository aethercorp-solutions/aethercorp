AWS CLI profiles

Ah! You want to understand how to use AWS CLI profiles (dev, prod) with the Serverless Framework. Let me explain step by step, carefully.

1️⃣ What --profile does

When you run:


aws configure --profile dev

AWS CLI saves a set of credentials under the name dev.

It creates (or updates) entries in:


~/.aws/credentials
~/.aws/config

Example ~/.aws/credentials:


[dev]
aws_access_key_id = AKIAxxxxxxxx
aws_secret_access_key = xxxxxxxxxxxxx

[prod]
aws_access_key_id = AKIAyyyyyyyy
aws_secret_access_key = yyyyyyyyyyyyy

Example ~/.aws/config:


[profile dev]
region = ap-southeast-1
output = json

[profile prod]
region = us-east-1
output = json

Each profile can have different keys, regions, or output formats.

2️⃣ Using profiles with the Serverless Framework

Option A: Directly in deploy command


serverless deploy --stage dev --aws-profile dev
serverless deploy --stage prod --aws-profile prod

Explanation:

  • --stage dev → sets the stage in serverless.yml
  • --aws-profile dev → tells Serverless which AWS credentials to use

✅ This is the most common approach.

Option B: Default profile in serverless.yml

You can also set it in your config:


provider:
  name: aws
  runtime: nodejs18.x
  stage: ${opt:stage, 'dev'}
  profile: ${opt:aws-profile, 'dev'}   # <-- default AWS profile
  region: ap-southeast-1

Then deploy like:


serverless deploy --stage prod --aws-profile prod

Or, if you don’t specify, it defaults to dev because of profile: dev.

Option C: Environment variable (optional)


export AWS_PROFILE=dev
serverless deploy
  • Serverless uses AWS_PROFILE if --aws-profile is not provided.

3️⃣ Switching between dev and prod

  • Step 1: Create profiles once: 
    
aws configure --profile dev
aws configure --profile prod
  • Step 2: Deploy to dev: 
    
serverless deploy --stage dev --aws-profile dev
  • serverless deploy --stage prod --aws-profile prod

Each stage uses its own credentials, region, and resources.

4️⃣ Verify which profile Serverless is using


serverless info --verbose

It shows:

  • Stage
  • Region
  • Profile
  • API endpoints
  • Lambda functions

🔹 Key Tips

  1. Never use your personal root account; create IAM users per profile with only needed permissions.
  2. Profiles let you manage dev/staging/prod independently.
  3. You can mix profile + stage + environment variables for fully separate deployments.
