1. What is an S3 Bucket?

An S3 bucket is part of Amazon S3 (Simple Storage Service), which is a storage service provided by Amazon Web Services (AWS). Think of it as a cloud folder where you can store files like HTML, CSS, JavaScript, images, videos, or any other data.

Key characteristics:

  • Scalable: Can store unlimited files.
  • Accessible: You can make files private or public.
  • Durable: AWS promises 99.999999999% durability of stored objects.
  • Static website hosting: S3 can host static websites directly (HTML, CSS, JS).

A “bucket” is just a container in S3. Each bucket has a unique name across AWS and can have its own settings (like permissions, region, versioning).

2. Why Use S3 for Deploying React-Vite Apps?

React apps built with Vite are static websites after build (they become HTML, JS, CSS files). Since S3 can host static websites, it’s perfect for:

  • Hosting the production-ready files.
  • Serving them via HTTP/HTTPS.
  • Scaling automatically with traffic.
  • Integrating with CloudFront for CDN support.

3. Steps to Deploy a React-Vite App on S3

Let’s go step by step:

Step 1: Build the React-Vite App

  1. Go to your project directory: 
    
cd my-vite-app
  2. Build the production-ready files: 
    
npm run build

    This creates a dist folder with index.html, assets, and JS bundles.

Step 2: Create an S3 Bucket

  1. Log in to AWS Management Console → Go to S3.
  2. Click Create bucket.
    • Give it a unique name (e.g., my-react-vite-app).
    • Choose a region close to your users.
    • Uncheck “Block all public access” (because a website should be public).
  3. Create the bucket.

Step 3: Configure the Bucket for Static Website Hosting

  1. Open your bucket → Go to Properties.
  2. Enable Static website hosting.
  3. Set:
    • Index document: index.html
    • Error document: index.html (important for React routes)

    React uses client-side routing, so all routes should fallback to index.html.

Step 4: Upload Your Files

  1. Go to Objects → Upload → Upload all files from the dist folder.
  2. Make the files publicly readable:
    • Select the files → Actions → Make public

      OR

    • Configure Bucket Policy (JSON example): 
      
{
    "Version": "2012-10-17",
    "Statement": [
        {
        "Sid": "PublicReadGetObject",
        "Effect": "Allow",
        "Principal": "*",
        "Action": "s3:GetObject",
        "Resource": "arn:aws:s3:::my-react-vite-app/*"
        }
    ]
}

Step 5: Access the Website

  • After upload, go to Properties → Static website hosting → Endpoint.
  • This URL serves your React-Vite app publicly.

Optional Step 6: Use a Custom Domain with CloudFront

  • If you want your app at www.example.com, use CloudFront (AWS CDN) in front of S3.
  • Benefits:
    • HTTPS
    • Caching
    • Global performance boost

4. Notes for React-Vite Apps on S3

  • Routing: Use index.html as a fallback for SPA (Single Page Application) routes.
  • Environment Variables: Use VITE_ prefix and build with npm run build.
  • Cache Busting: Vite automatically hashes files in the build for caching.

✅ Summary:

  1. S3 bucket = cloud folder for files.
  2. React-Vite produces static files in dist/.
  3. Upload dist/ to S3 → enable static hosting.
  4. Set public access + index.html fallback → your app is live.

Perfect! Let’s do a full AWS CLI deployment for your React-Vite app. I’ll give you a step-by-step command setup so you can deploy your app entirely from the terminal.

1️⃣ Prerequisites

Make sure you have:

  • AWS CLI installed and configured: 
    
aws configure

    Enter your Access Key ID, Secret Access Key, default region, and output format (json is fine).

  • Node.js & npm installed for building Vite apps.
  • Your React-Vite app ready (dist/ folder after build).

2️⃣ Build Your React-Vite App

From your project folder:


npm run build
  • This creates the dist/ folder with your production files.

3️⃣ Create an S3 Bucket via CLI

Replace my-react-vite-app with a globally unique bucket name:


aws s3 mb s3://my-react-vite-app --region us-east-1
  • mb = make bucket.
  • --region = choose your preferred AWS region.

4️⃣ Configure Bucket for Public Access

You need a bucket policy so anyone can access your files:


aws s3api put-bucket-policy --bucket my-react-vite-app --policy '{
    "Version": "2012-10-17",
    "Statement": [
        {
        "Sid": "PublicReadGetObject",
        "Effect": "Allow",
        "Principal": "*",
        "Action": "s3:GetObject",
        "Resource": "arn:aws:s3:::my-react-vite-app/*"
        }
    ]
}'

This makes all files publicly readable.

5️⃣ Enable Static Website Hosting


aws s3 website s3://my-react-vite-app/ --index-document index.html --error-document index.html
  • The error document is also index.html so React routing works.

6️⃣ Upload Your React-Vite Build


aws s3 sync dist/ s3://my-react-vite-app/ --delete
  • sync = uploads files and deletes old files not in dist/.
  • --delete = keeps S3 in sync with your local build.

7️⃣ Access Your Website

Once uploaded:


http://my-react-vite-app.s3-website-us-east-1.amazonaws.com
  • Replace us-east-1 with your bucket region if different.
  • Your React-Vite app should be live!

8️⃣ Optional: Use CloudFront for HTTPS


aws cloudfront create-distribution --origin-domain-name my-react-vite-app.s3.amazonaws.com
  • CloudFront gives you HTTPS, caching, and global delivery.

✅ Summary of CLI Commands


npm run build
aws s3 mb s3://my-react-vite-app --region us-east-1
aws s3api put-bucket-policy --bucket my-react-vite-app --policy ''
aws s3 website s3://my-react-vite-app/ --index-document index.html --error-document index.html
aws s3 sync dist/ s3://my-react-vite-app/ --delete
