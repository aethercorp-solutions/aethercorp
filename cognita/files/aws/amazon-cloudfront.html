Amazon CloudFront

Amazon CloudFront is a Content Delivery Network (CDN) service that securely delivers content (websites, APIs, videos, images, software downloads) to users with low latency and high transfer speeds by using a global network of edge locations.

🌍 1. Core Idea: How a CDN Works

Instead of serving content from one central server (origin), CloudFront:

  1. Caches copies of your content
  2. Stores them in many Edge Locations worldwide
  3. Delivers content from the closest edge location to the user

This reduces:

  • Latency
  • Load on origin servers
  • Network congestion
  • Infrastructure costs

🏗 2. CloudFront Architecture Components

1️⃣ Origin

The original source of your content.

Common origins:

  • Amazon S3 (static files like images, JS, CSS)
  • Amazon EC2 (web applications)
  • Elastic Load Balancing
  • On-premise servers

2️⃣ Edge Locations

  • Global data centers
  • Cache content close to users
  • First place user requests go

3️⃣ Regional Edge Caches

  • Intermediate caching layer
  • Reduces load on origin even more
  • Improves cache hit ratio

4️⃣ Distribution

A CloudFront Distribution is the configuration that connects:

  • Origin
  • Cache behaviors
  • Security rules
  • Domain name
  • SSL certificate

There are two types:

  • Web distribution (HTTP/HTTPS)
  • RTMP (legacy streaming)

🔄 3. Step-by-Step: How CloudFront Works

Example: User Requests an Image

Step 1: User Request

A user types:


https://example.com/logo.png

DNS routes the request to the nearest CloudFront edge location.

Step 2: Cache Check

Edge location checks:

  • Is logo.png already cached?

✅ If YES (Cache Hit)

→ File is returned immediately

→ Fast response

❌ If NO (Cache Miss)

→ CloudFront forwards request to origin

Step 3: Fetch from Origin

CloudFront:

  • Retrieves file from origin (e.g., S3)
  • Caches it at edge location
  • Sends it to user

Step 4: Future Requests

Next users in that region:

→ Get file directly from cache

→ No origin request needed

⚙️ 4. Caching Behavior (Important)

CloudFront caches based on:

  • URL path
  • Query strings
  • Headers
  • Cookies

You can configure:

  • TTL (Time To Live) – How long content stays cached
  • Minimum TTL
  • Maximum TTL
  • Default TTL

Example:

  • Set TTL = 24 hours
  • CloudFront keeps file for 24h before checking origin again

🔐 5. Security Features

CloudFront integrates with:

🛡 AWS Shield

  • Protects against DDoS attacks

🔥 AWS WAF

  • Blocks malicious traffic
  • Rate limiting
  • IP filtering

🔒 HTTPS / SSL

  • Free SSL via AWS Certificate Manager
  • Enforces HTTPS

🔑 Signed URLs & Cookies

  • Restrict access to premium content
  • Time-limited access

🚀 6. Performance Optimization

CloudFront improves performance using:

  • HTTP/2 and HTTP/3 support
  • Persistent connections
  • Brotli & Gzip compression
  • TCP optimization
  • Edge computing (Lambda@Edge, CloudFront Functions)

🧠 7. Advanced Features

Lambda@Edge

Run code at edge locations:

  • Modify headers
  • Authentication
  • Redirects
  • A/B testing

Origin Failover

  • Primary + Secondary origin
  • Automatic failover if origin is down

Field-Level Encryption

  • Encrypt specific sensitive fields

📦 8. Static vs Dynamic Content

Content Type How CloudFront Handles It
Static files Cached at edge
Dynamic content Forwarded but optimized
API calls Cached selectively
Live streaming Supported

🌎 9. Real-World Example Architecture

User → CloudFront → S3 (static content)

User → CloudFront → ALB → EC2 (web app)

User → CloudFront → API Gateway (API backend)

💰 10. Pricing Model

You pay for:

  • Data transfer out (per GB)
  • Requests (per 10,000)
  • Invalidation requests
  • Edge compute (if used)

Pricing varies by region.

🏁 Summary

CloudFront works by:

  1. Receiving user requests
  2. Checking nearest edge cache
  3. Fetching from origin if needed
  4. Caching and delivering content
  5. Improving speed, scalability, and security
