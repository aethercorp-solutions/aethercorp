Create HTML Template Using Bash Script

Here's a Bash script that echoes the provided HTML content into an index.html file:

Instructions:


#!/bin/bash

cat << 'EOF' > index.html
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>

</body>
</html>
EOF
  1. Save the code above into a file, e.g., create_index.sh
  2. Make the script executable: 
    
chmod +x create_index.sh
  3. Run the script: 
    
./create_index.sh

This will create an index.html file with the HTML content.

Here's a version of the Bash script that allows you to specify both the path and filename for the HTML file (e.g., ./create_index.sh ./pages/index.html), and it will create the directories if they donâ€™t already exist:


#!/bin/bash

# Check if path/filename is provided
if [ -z "$1" ]; then
    echo "Usage: $0 path/filename.html"
    exit 1
fi

FILEPATH="$1"
DIRNAME=$(dirname "$FILEPATH")

# Create directory if it doesn't exist
mkdir -p "$DIRNAME"

# Write HTML content to the file
cat << 'EOF' > "$FILEPATH"
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>

</body>
</html>
EOF

echo "HTML file '$FILEPATH' created successfully."

Usage Example:


./create_index.sh ./site/pages/index.html

This will:

  • Create the directory ./site/pages/ if it doesn't exist.
  • Generate the index.html file at that path.

Your Bash script is well-written and functions correctly to:

  1. Check if a file path argument is provided.
  2. Create the target directory if it doesn't exist.
  3. Write a basic HTML template into the specified file.

Key Features That Work:

  • Usage message for missing argument.
  • Directory creation with mkdir -p
  • Heredoc (cat << 'EOF') to insert static HTML.
  • Success message after completion.

Suggestions for Enhancement (Optional)

If you're looking to expand or refine the script:

  1. Prevent Overwriting Existing Files

    To avoid accidentally overwriting existing HTML files:

    
if [ -e "$FILEPATH" ]; then
    echo "Error: File '$FILEPATH' already exists."
    exit 1
fi

  2. Make <title> Dynamic

    Allow the user to specify the HTML title as an optional second argument

    
TITLE=${2:-Document}

    Then update the heredoc:

    
<title>$TITLE</title>

    Note: Remove single quotes from 'EOF' if you want variable expansion.

    
cat << EOF > "$FILEPATH"
...
<title>$TITLE</title>
...
EOF

  3. Add Executable Permissions (if needed)

    Make sure the script is executable:

    
chmod +x script.sh
