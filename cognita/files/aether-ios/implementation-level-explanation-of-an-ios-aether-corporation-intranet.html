5. Client (Employee) Profile Settings â€“ iOS Implementation

5.1 Identity Information (Read-Only / Limited Edit)

Purpose

  • Ensure identity consistency with corporate IdP
  • Prevent spoofing

Fields

  • Full Name (read-only)
  • Corporate Email (read-only)
  • Employee ID (read-only)
  • Department (read-only)
  • Profile Photo (editable)

Implementation (Swift)


struct UserProfile {
    let userId: String
    let fullName: String
    let email: String
    let department: String
    let role: UserRole
}
  • Name/email sourced from IdP claims
  • Photo uploaded to secure object storage
  • Changes logged via audit API

5.2 Security & Authentication

Client Capabilities

  • Change password (if local auth allowed)
  • Enable/disable biometric login
  • View active sessions
  • Trigger logout from other devices

iOS Features Used

  • Face ID / Touch ID (LocalAuthentication)
  • Keychain for token storage
  • Secure Enclave

LAContext().evaluatePolicy(.deviceOwnerAuthenticationWithBiometrics)

Session Management

  • Tokens refreshed silently
  • Sessions revoked via API
  • Display last login & device info

6.3 Role & Permission Management

Admins can assign:

  • Client
  • Manager
  • Admin
  • Custom roles

Permissions are policy-based, not hardcoded.


{
    "role": "manager",
    "permissions": [
        "view_reports",
        "approve_requests"
    ]
}

Changes propagate in real time to iOS app via token refresh.

6.4 Policy Enforcement (Critical)

Admins configure:

  • Password policies
  • Session timeouts
  • Device compliance rules

Example:

  • Block access if device is jailbroken
  • Enforce minimum iOS version
  • Force biometric authentication

iOS enforcement:

  • MDM signals
  • App checks on launch
  • API denial if non-compliant

6.5 Audit & Compliance Logs

Admins can view:

  • Login history
  • Profile changes
  • Permission edits
  • Failed access attempts

Audit logs are:

  • Immutable
  • Timestamped
  • Device-tagged

8. Security Best Practices for iOS Aether Intranet

  • JWT stored in Keychain
  • No sensitive data in UserDefaults
  • Encrypted API traffic only
  • Biometric fallback required
  • App-level jailbreak detection
  • Automatic logout on policy violation
