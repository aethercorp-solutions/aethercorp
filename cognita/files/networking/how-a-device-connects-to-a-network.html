How a Device Connects to a Network

Let's break down how a device connects to a network, step by step, from powering it on to being fully online.

1. Physical or Wireless Connection

The device first connects to the network physically or wirelessly:

  • Wired (Ethernet): Connects using a network cable to a switch or router.
  • Wireless (Wi-Fi): Connects via a wireless access point (usually built into a router).

2. Link Layer: Hardware Communication

  • The device's network interface card (NIC) uses a MAC address (unique hardware ID).
  • It checks for a link with the router or switch.
  • If successful, the link becomes active, and the device is now "physically" part of the local network.

3. DHCP Request: Get IP Address

If the device is set to use DHCP (which is the default):

  1. Sends a DHCPDISCOVER broadcast (usually to 255.255.255.255).
  2. The DHCP server (often the router) responds with a DHCPOFFER.
  3. The device replies with DHCPREQUEST to accept the offer.
  4. The server confirms with DHCPACK, assigning:
    • IP address
    • Subnet mask
    • Default gateway
    • DNS server(s)

Now the device has everything it needs to communicate on the network!

4. DNS Resolution & Internet Access

  • When you open a website (e.g., openai.com), the device asks a DNS server to translate the name into an IP address.
  • It then sends a request (like HTTP or HTTPS) through the default gateway (usually your router), out to the Internet.

Summary: End-to-End Steps

Step Action
1 Connects physically or via Wi-Fi
2 Establishes a link-layer connection
3 Requests IP config from DHCP
4 Receives IP, gateway, DNS info
5 Uses DNS to resolve websites
6 Sends/receives data through router to the Internet
