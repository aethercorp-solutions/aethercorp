HTTP header

An HTTP header is a part of the information sent between a client (like a web browser) and a server when making an HTTP request or response. Headers are used to pass metadata‚Äîinformation about the data‚Äîrather than the actual content itself.

üîß How HTTP Headers Work:

When a client sends a request to a server (e.g., when you open a website), it includes request headers. The server responds with response headers, before sending the actual content (HTML, images, JSON, etc.).

üì¶ Structure of an HTTP Header:

Headers are key-value pairs, separated by a colon (:), and are placed before the body of the request or response.

Example (Request Header):


GET /index.html HTTP/1.1
Host: www.example.com
User-Agent: Mozilla/5.0
Accept: text/html

Example (Response Header):


HTTP/1.1 200 OK
Content-Type: text/html
Content-Length: 1234
Cache-Control: no-cache

üß∞ Common HTTP Request Headers

Header Purpose
Host Specifies the domain name of the server
User-Agent Identifies the browser/client making the request
Accept Tells the server what content types the client can handle
Authorization Used for sending credentials (like tokens)
Cookie Sends cookies from client to server

üß∞ Common HTTP Response Headers

Header Purpose
Content-Type Describes the format of the returned content (e.g., text/html, application/json)
Content-Length Size of the response body in bytes
Set-Cookie Sets cookies in the browser
Cache-Control Tells client how to cache the response
Access-Control-Allow-Origin Used in CORS to specify allowed domains

‚úÖ Why HTTP Headers Matter

  • Control caching
  • Handle content negotiation (e.g., language, encoding)
  • Send authentication data
  • Enable cross-origin requests (CORS)
  • Improve security (e.g., via CSP, HSTS headers)

Sure! Here's a basic example of using the fetch() API in JavaScript to send an HTTP request with custom headers.

üì¶ Example: fetch() with custom request headers


fetch('https://api.example.com/data', {
    method: 'GET', // or 'POST', 'PUT', etc.
    headers: {
        'Content-Type': 'application/json',
        'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
        'Accept': 'application/json'
    }
})
.then(response => {
    if (!response.ok) {
        throw new Error('Network response was not ok');
    }
    return response.json(); // Parse the JSON from the response
    })
    .then(data => {
    console.log('Received data:', data);
    })
    .catch(error => {
    console.error('Fetch error:', error);
});

üß† Breakdown:

  • method: Type of HTTP request (GET, POST, etc.)
  • headers: Custom HTTP headers sent with the request
    • Content-Type: Tells the server what format you're sending (for POST, PUT, etc.)
    • Authorization: Often used for API tokens
    • Accept: Tells the server what kind of response you expect (e.g., application/json)
  • response.json(): Converts the response body to a JavaScript object

üîê Example with a POST request and body


fetch('https://api.example.com/login', {
    method: 'POST',
    headers: {
        'Content-Type': 'application/json'
    },
    body: JSON.stringify({
        username: 'john_doe',
        password: 'secret123'
    })
})
    .then(response => response.json())
    .then(data => {
    console.log('Login successful:', data);
    })
    .catch(error => {
    console.error('Login error:', error);
});

Great! Here's how you can do the same thing using Axios, a popular JavaScript library for making HTTP requests.

üì¶ Example: axios.get() with custom headers


import axios from 'axios';

axios.get('https://api.example.com/data', {
  headers: {
    'Authorization': 'Bearer YOUR_ACCESS_TOKEN',
    'Accept': 'application/json'
  }
})
.then(response => {
  console.log('Received data:', response.data);
})
.catch(error => {
  console.error('Error fetching data:', error);
});

üîê Example: axios.post() with body and headers


import axios from 'axios';

axios.post('https://api.example.com/login', {
  username: 'john_doe',
  password: 'secret123'
}, {
  headers: {
    'Content-Type': 'application/json'
  }
})
.then(response => {
  console.log('Login successful:', response.data);
})
.catch(error => {
  console.error('Login error:', error);
});

üß† Axios Response Object

When the request is successful, Axios gives you a response object like:


{
    data: {...},         // The response body
    status: 200,         // HTTP status code
    statusText: 'OK',
    headers: {...},      // Response headers
    config: {...},       // Axios config for the request
    request: {...}       // The request object
}

You can access response headers like this:


axios.get('https://api.example.com/data')
  .then(response => {
    console.log(response.headers); // All response headers
    console.log(response.headers['content-type']); // Specific header
});
