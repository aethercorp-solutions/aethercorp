What a subnet is

I’ll explain subnets from the ground up, then tie it to AWS, since I know you’ve been deep in VPC / serverless land.

1. What a subnet is (plain English)

A subnet (sub-network) is a smaller network carved out of a bigger network.

Think of a city 🏙️

  • VPC = the whole city
  • Subnet = a neighborhood
  • IP address = a house address

You divide the city into neighborhoods so you can:

  • organize traffic
  • control access
  • reduce chaos
  • apply different rules

2. Why subnets exist (the real reasons)

Subnets solve 4 big problems:

1️⃣ IP address management

Instead of one giant pool of IPs, you divide them:

  • Frontend servers
  • Backend servers
  • Databases
  • Internal services

This avoids IP collisions and keeps things clean.

2️⃣ Security boundaries

You can apply different rules per subnet:

  • Public-facing things
  • Private internal things
  • Highly restricted data stores

Security teams love subnets ❤️

3️⃣ Traffic control & performance

Broadcast traffic stays inside the subnet.

Less noise → better performance.

4️⃣ High availability & fault isolation

If one subnet (or zone) goes down, others stay alive.

3. IP addressing basics (CIDR)

Subnets are defined using CIDR notation:


10.0.0.0/16

Breakdown:

  • 10.0.0.0 → network base address
  • /16 → how many bits are for the network

What /16 means

  • Total IPs: 2^(32-16) = 65,536
  • Range:

10.0.0.0 – 10.0.255.255

4. Subnetting a network (real example)

Start with a VPC


VPC: 10.0.0.0/16

You split it into subnets:

Subnet CIDR IP Range
Subnet A 10.0.1.0/24 256 IPs
Subnet B 10.0.2.0/24 256 IPs
Subnet C 10.0.3.0/24 256 IPs

Each /24:

  • 256 total IPs
  • ~251 usable (some are reserved)

5. Public vs Private Subnets (VERY important)

🚨 Myth

“Public subnet means publicly accessible”

❌ Wrong.

✅ Truth

A subnet is public ONLY if:

  • It has a route to an Internet Gateway (IGW)

Public Subnet

  • Route table includes: 
    
0.0.0.0/0 → Internet Gateway
  • Used for:
    • Load balancers
    • Bastion hosts
    • Public APIs

Private Subnet

  • NO route to IGW
  • Often routes to: 
    
0.0.0.0/0 → NAT Gateway
  • Used for:
    • Databases
    • Backend services
    • Lambdas in VPC

6. AWS-specific subnet rules (crucial)

1️⃣ Subnet = ONE Availability Zone

You cannot span a subnet across AZs.

Example:


Subnet 1 → ap-southeast-1a
Subnet 2 → ap-southeast-1b

This is how AWS achieves fault tolerance.

2️⃣ Reserved IPs in AWS

AWS reserves 5 IPs per subnet:

Address Purpose
.0 Network address
.1 VPC router
.2 DNS
.3 Future use
.255 Broadcast

So /24 → 256 − 5 = 251 usable IPs

7. Subnets + Route Tables (who can talk to whom)

Subnets don’t route traffic by themselves.

They use Route Tables.

Example:


Subnet A → Route Table A
Subnet B → Route Table B

Routes decide:

  • internet access
  • VPC-to-VPC
  • on-prem connections
  • NAT access

8. Subnets + Security (layers matter)

Layer 1: Network ACL (stateless)

  • Attached to subnets
  • Allows / denies IP ranges
  • Inbound + outbound rules

Layer 2: Security Groups (stateful)

  • Attached to ENIs / EC2 / Lambda
  • Instance-level protection

Both apply.

Traffic must pass NACL → SG.

9. Subnets in serverless (Lambda, your world 😄)

Since you’ve worked with:

  • Lambda
  • API Gateway
  • MongoDB
  • Auth flows

Here’s the key bit:

Lambda in a VPC

When you attach a Lambda to subnets:

  • Lambda lives inside those subnets
  • It gets ENIs from the subnet IP pool

👉 If the subnet runs out of IPs:

  • Lambda fails to scale

This is why people recommend:

  • /24 or larger
  • multiple subnets across AZs

10. Common subnet design pattern (best practice)

Typical AWS VPC layout


VPC 10.0.0.0/16

Public Subnets
- 10.0.1.0/24 (AZ-a)
- 10.0.2.0/24 (AZ-b)

Private App Subnets
- 10.0.11.0/24 (AZ-a)
- 10.0.12.0/24 (AZ-b)

Private DB Subnets
- 10.0.21.0/24 (AZ-a)
- 10.0.22.0/24 (AZ-b)

Each layer:

  • isolated
  • scalable
  • secure

11. Mental model (remember this)

A subnet is just an IP range + AZ + route table + network rules

That’s it.

Everything else (EC2, Lambda, RDS) just plugs into a subnet.
