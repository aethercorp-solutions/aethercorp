A filesystem is a method an operating system uses to store, organize, manage, and retrieve data on storage devices like hard drives, SSDs, or USB sticks.

You can think of it as a data structure on disk that tells the OS where files are, how big they are, who owns them, etc.

üß± 1. Blocks: Basic Storage Unit

  • The disk is divided into fixed-size blocks (usually 512B, 4KB, etc.).
  • Files are stored as one or more blocks.
  • Even if a file is smaller than a block, it takes up the whole block (‚Üí internal fragmentation).

Example:

  • File A.txt = 6 KB, block size = 4 KB
  • File takes up 2 blocks = 8 KB (with 2 KB unused)

üì¶ 2. Inodes: Metadata of Files

Inodes (Index Nodes) store metadata about files.

Each inode contains:

  • File type (regular file, directory, link)
  • Size
  • Owner & permissions
  • Timestamps (created, modified, accessed)
  • Pointers to data blocks

Inode 1001
  - Size: 15 KB
  - Direct Block Pointers: [12, 13, 14, 15]
  - Indirect Pointers: (if file is large)

üìÅ 3. Directories: Mapping Names to Inodes

Directories are special files that contain mappings:


filename ‚Üí inode number

Example:


dir1/
  - "file.txt" ‚Üí inode 1001
  - "notes.md" ‚Üí inode 1002

When you access dir1/file.txt, the filesystem:

  1. Looks up "dir1"
  2. Opens its inode
  3. Finds the directory entry for "file.txt"
  4. Retrieves inode 1001
  5. Reads data blocks for that file

üìã 4. Superblock: Filesystem Summary

Every filesystem has a superblock, a structure that holds high-level metadata:

  • Total size of filesystem
  • Number of inodes
  • Number of blocks
  • Free blocks/inodes
  • Location of inode table

If the superblock gets corrupted, the whole filesystem may become unreadable.

üìä 5. File Allocation Methods

Files may be stored in blocks in different ways:

‚ñ™Ô∏è Contiguous Allocation

  • Store all blocks of a file next to each other
  • Fast but suffers from fragmentation

üîó Linked Allocation

  • Each block has a pointer to the next
  • Simple, no fragmentation, but slow random access

üóÇÔ∏è Indexed Allocation (used in inodes)

  • Use an index (inode) to point to all data blocks
  • Fast random access, scalable
