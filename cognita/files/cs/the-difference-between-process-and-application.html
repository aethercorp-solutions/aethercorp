The difference between Process and Application

The terms process and application are related but not the same. The key difference is conceptual vs runtime.

1. Application (Program / App)

What it is

An application is a static set of files:

  • Executable binary (.exe, .app, ELF, APK)
  • Libraries
  • Assets (images, configs, resources)

It does nothing by itself until it is launched.

Characteristics

  • Stored on disk
  • Passive (not executing)
  • One application can be launched multiple times
  • Platform-specific packaging

Examples

  • Chrome.app
  • Photoshop.exe
  • MyApp.apk
  • node index.js (the script itself)

2. Process

What it is

A process is a running instance of an application.

Once the OS loads the application into memory and starts executing it, it becomes a process.

Characteristics

  • Lives in memory (RAM)
  • Active (executing instructions)
  • Has:
    • Process ID (PID)
    • Own virtual memory space
    • Open file descriptors
    • Environment variables
    • At least one thread (main thread)

Examples

  • Running Chrome → chrome.exe process
  • Running node index.js → Node.js process
  • Background system services

3. One-to-Many Relationship

Scenario Application Process
App installed but not running ✅ ❌
App launched once 1 1
App launched twice 1 2
Browser with multiple tabs 1 Multiple processes

4. Real-World Analogy

Application = Blueprint

Process = Actual building constructed from the blueprint

You can build:

  • One building from one blueprint
  • Multiple buildings from the same blueprint

5. In Modern Systems (Important Distinction)

Desktop (Windows / macOS / Linux)

  • One application → multiple processes
  • Example: Chrome
    • Browser process
    • Renderer processes
    • GPU process

Mobile (iOS / Android)

  • Typically:
    • One app = one main process
    • System may kill and restart process
  • Android allows:
    • Multiple processes per app (android:process)
  • iOS is stricter (sandboxed, usually single process)

6. Process vs Application vs Thread (Quick Context)

Term Meaning
Application Program on disk
Process Running instance of program
Thread Execution path inside a process

A process must have at least one thread.

7. Why This Matters (for you as a developer)

Given your background in React Native, iOS, Android, AWS Lambda:

  • iOS/Android: OS kills the process, not the app
  • AWS Lambda: Each invocation runs in a process/container
  • Node.js: One app → one process → event loop thread
  • Crash handling: App exists, process died
  • Memory leaks: Live inside a process, not the app

One-sentence summary

An application is a program on disk, while a process is a running instance of that application in memory.
