What is SQL JOIN? (Absolute Beginner Explanation)

In a database, data is often split into multiple tables to keep it organized.

👉 JOIN is how SQL connects related tables together so you can see combined data in one result.

Think of a real-life example 📚

Imagine a school:

Table 1: Students

student_id name
1 Anna
2 Ben
3 Chris

Table 2: Grades

grade_id student_id subject grade
101 1 Math 90
102 1 English 85
103 2 Math 88

👉 The connection between these tables is student_id.

Why we need JOIN

If you query Students, you see names but no grades.

If you query Grades, you see grades but not student names.

JOIN lets you combine them.

Basic JOIN syntax


SELECT columns
FROM table1
JOIN table2
ON condition;

What each part means:

  • SELECT → what columns you want to see
  • FROM → first table
  • JOIN → second table
  • ON → how the tables are connected

1️⃣ INNER JOIN (Most Common)

Meaning:

👉 Show only matching data in both tables.

Example:


SELECT
  Students.name,
  Grades.subject,
  Grades.grade
FROM Students
INNER JOIN Grades
ON Students.student_id = Grades.student_id;

Result:

name subject grade
Anna Math 90
Anna English 85
Ben Math 88

❌ Chris is missing because he has no grades.

Mental picture 🧠

INNER JOIN = overlap only


Students ∩ Grades

2️⃣ LEFT JOIN (Very Important)

Meaning:

👉 Show ALL rows from the left table, even if there’s no match.

Example:


SELECT
  Students.name,
  Grades.subject,
  Grades.grade
FROM Students
LEFT JOIN Grades
ON Students.student_id = Grades.student_id;

Result:

name subject grade
Anna Math 90
Anna English 85
Ben Math 88
Chris NULL NULL

✔ Chris appears, but with NULL values.

Mental picture 🧠

INNER JOIN = overlap only


Students ∩ Grades

2️⃣ LEFT JOIN (Very Important)

Meaning:

👉 Show ALL rows from the left table, even if there’s no match.

Example:


SELECT
  Students.name,
  Grades.subject,
  Grades.grade
FROM Students
LEFT JOIN Grades
ON Students.student_id = Grades.student_id;

Result:

name subject grade
Anna Math 90
Anna English 85
Ben Math 88
Chris NULL NULL

✔ Chris appears, but with NULL values.

Mental picture 🧠

LEFT JOIN = everything on the left


Students ⟵ Grades

3️⃣ RIGHT JOIN

Meaning:

👉 Show ALL rows from the right table, even if no match.


SELECT
  Students.name,
  Grades.subject,
  Grades.grade
FROM Students
RIGHT JOIN Grades
ON Students.student_id = Grades.student_id;

👉 Less commonly used (LEFT JOIN is preferred).

4️⃣ FULL JOIN

Meaning:

👉 Show everything from both tables, matched or not.


SELECT
  Students.name,
  Grades.subject,
  Grades.grade
FROM Students
FULL JOIN Grades
ON Students.student_id = Grades.student_id;

⚠ Not supported in MySQL (works in PostgreSQL, SQL Server).

5️⃣ JOIN with WHERE (Filtering)


SELECT
  Students.name,
  Grades.subject,
  Grades.grade
FROM Students
JOIN Grades
ON Students.student_id = Grades.student_id
WHERE Grades.subject = 'Math';

👉 Shows only Math grades.

Very Common Beginner Mistakes ⚠️

❌ Forgetting the ON condition


JOIN Grades   -- ❌ WRONG

✔ Always use:


ON Students.student_id = Grades.student_id

❌ Confusing JOIN vs WHERE

  • ON → how tables connect
  • WHERE → filtering results

Simple Analogy 🧩

Think of JOIN like matching:

  • Students = people
  • Grades = report cards
  • student_id = ID number

SQL JOIN says:

“Match rows where the ID number is the same.”
