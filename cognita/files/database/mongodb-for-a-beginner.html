1. What is MongoDB?

MongoDB is a NoSQL database. Unlike traditional SQL databases like MySQL or PostgreSQL, MongoDB:

  • Stores data in documents, not tables.
  • Uses JSON-like format (called BSON internally) for data.
  • Is schema-less, meaning each document in a collection can have different fields.
  • Is designed to be scalable, flexible, and high-performance for modern applications.

Think of it as storing objects instead of rows in a table.

2. Key MongoDB Concepts

Concept Explanation
Database A container for collections (like a folder).
Collection Like a table in SQL. A group of documents.
Document A single record, stored in JSON/BSON. Example: { name: "John", age: 30 }
Field Key-value pair inside a document. Example: name: "John"
_id Unique identifier automatically added to every document.
BSON Binary JSON format MongoDB uses internally.

3. MongoDB vs SQL

Feature SQL (MySQL) MongoDB
Schema Fixed schema Flexible schema
Data Format Rows & tables Documents (JSON)
Joins Supports joins Usually avoided, can use $lookup
Scalability Vertical scaling Horizontal scaling
Query Language SQL MongoDB Query Language (MQL)

6. CRUD Operations (Create, Read, Update, Delete)

6.1 Insert Documents


db.users.insertOne({ name: "Alice", age: 25, city: "Manila" })

db.users.insertMany([
  { name: "Bob", age: 30 },
  { name: "Charlie", age: 28 }
])

6.2 Query Documents


// Find all documents
db.users.find()

// Find one document
db.users.findOne({ name: "Alice" })

// Find with condition
db.users.find({ age: { $gt: 25 } }) // greater than 25

6.3 Update Documents


// Update one
db.users.updateOne(
  { name: "Alice" },
  { $set: { age: 26 } }
)

// Update many
db.users.updateMany(
  { age: { $lt: 30 } },
  { $inc: { age: 1 } } // increment age by 1
)

6.4 Delete Documents


db.users.deleteOne({ name: "Charlie" })
db.users.deleteMany({ age: { $lt: 25 } })

7. Indexes

Indexes improve query performance.


db.users.createIndex({ name: 1 }) // 1 for ascending, -1 for descending

8. Relationships in MongoDB

MongoDB supports two types of relationships:

  • Embedded documents (denormalization) 
    
{
    name: "Alice",
    orders: [
        { product: "Laptop", qty: 1 },
        { product: "Mouse", qty: 2 }
    ]
}
  • References (normalization) 
    
// Users
{ _id: 1, name: "Alice" }

// Orders
{ _id: 1, userId: 1, product: "Laptop" }

9. Using MongoDB in Node.js

  1. Install MongoDB driver: 
    
npm install mongodb
  2. Connect and perform operations: 
    
const { MongoClient } = require('mongodb');

const url = 'mongodb://localhost:27017';
const client = new MongoClient(url);

async function main() {
  await client.connect();
  const db = client.db('myDatabase');
  const users = db.collection('users');

  await users.insertOne({ name: "John", age: 30 });
  const allUsers = await users.find().toArray();
  console.log(allUsers);

  await client.close();
}

main();

10. Tips for Beginners

  • Always use _id as a unique identifier.
  • Start with MongoDB Atlas for simplicity.
  • Use Mongoose (ODM) if using Node.js – it simplifies schema and validation.
  • Use find() and findOne() often to explore your data.
  • Learn query operators like $gt, $lt, $in, $set.
